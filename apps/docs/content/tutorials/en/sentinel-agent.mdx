In this tutorial you'll build a **sentinel agent** — a security layer that monitors every tool call made by an AI agent (or an entire AgentTeam), blocks dangerous operations before they execute, detects sensitive data in tool output, and automatically injects remediation tasks into the team's work queue.

The sentinel is implemented entirely with **lifecycle hooks**. There is no separate process or sidecar — you register hook handlers on the session once, and the framework automatically propagates them to every sub-agent at any depth.

<Callout type="info">
  Install the SDK first: `pip install a3s-code` (Python) or `npm install @a3s-lab/code` (Node.js). Configure `~/.a3s/config.hcl` with your LLM provider key.
</Callout>

## How it works

```
Session
  │
  ├── register_hook("sentinel-pre",  pre_tool_use,  handler=block_dangerous)
  └── register_hook("sentinel-post", post_tool_use, handler=detect_sensitive)
                              │
          ┌───────────────────┴────────────────────┐
          │                                        │
     Pre-tool-use                           Post-tool-use
   fires BEFORE execution                  fires AFTER execution
          │                                        │
   ┌──────▼──────┐                        ┌───────▼───────┐
   │ match risky │                        │ match keywords│
   │  patterns?  │                        │ in output?    │
   └──────┬──────┘                        └───────┬───────┘
          │ yes                                   │ yes
          ▼                                       ▼
     { action: "block" }                  record alert
     agent sees error                  inject remediation task
     execution stops                   into AgentTeam board
```

**Hook propagation** — hooks registered once on the parent session are inherited automatically by every sub-agent spawned by the `task` tool, at unlimited nesting depth:

```
Parent session  (sentinel hooks registered here)
  └── task tool spawns Child AgentLoop
        ├── inherits sentinel hook engine
        └── task tool spawns Grandchild AgentLoop
              └── inherits sentinel hook engine
```

### Define risk patterns

Two pattern lists control what the sentinel watches for:

- **`BLOCKED_PATTERNS`** — substrings in bash command arguments that should be blocked outright. The `pre_tool_use` hook fires before the tool executes, so the block takes effect before any damage is done.
- **`SENSITIVE_KEYWORDS`** — keywords in tool output that indicate a data leak. The `post_tool_use` hook fires after the tool returns, so it can only alert and log — not block.

</ScrollyStep>

<ScrollyStep index={3}>

### Sentinel state

A single `SentinelLog` object tracks everything the sentinel observes:

- `blocked` — list of commands that were blocked, with session ID and pattern matched
- `alerts` — list of sensitive-data detections, with keyword and output snippet
- `injected_tasks` — list of remediation tasks posted to the team board
- `task_board` — set this to the `TeamTaskBoard` before running an AgentTeam so the sentinel can inject tasks dynamically

</ScrollyStep>

<ScrollyStep index={4}>

### Hook handlers

Two handler functions, one for each hook event:

**`on_pre_tool_use`** — called before every tool execution. Returns `{"action": "block", "reason": "..."}` to cancel the call, or `None` to allow it. The handler:
1. Extracts the command from `event["payload"]["args"]["command"]`
2. Checks it against `BLOCKED_PATTERNS`
3. If matched: records the block, injects a remediation task, returns block response

**`on_post_tool_use`** — called after every tool execution. Cannot block (the tool already ran). The handler:
1. Extracts the output from `event["payload"]["result"]["output"]`
2. Checks it against `SENSITIVE_KEYWORDS`
3. If matched: records the alert, injects a data-leak review task

</ScrollyStep>

<ScrollyStep index={5}>

### Attach sentinel

`attach_sentinel` registers both hooks on any session with a single function call.

Key parameters:
- **`priority: 1`** — runs before the built-in `SecurityGuard` (priority 1 is the highest)
- **`async_execution: True`** on the post hook — allows the alert and task injection to run without blocking the agent loop

Once attached, the hooks propagate automatically to any sub-agent spawned by the `task` tool — you never need to re-register them.

</ScrollyStep>

<ScrollyStep index={6}>

### Single agent demo

Attach the sentinel to a session, then send three prompts:

1. **Safe command** — `ls /tmp | head -5`. Passes through with no intervention.
2. **Dangerous command** — `rm -rf /tmp/sentinel-test`. The sentinel blocks it before bash runs. The agent receives an error message and can explain why.
3. **Sensitive output** — `echo 'api_key=sk-abc123 token=xyz789'`. The command runs (it's not dangerous), but the post-hook detects the sensitive keywords and records an alert.

</ScrollyStep>

<ScrollyStep index={7}>

### AgentTeam with sentinel

For a team, attach the sentinel to **every member's session** individually, then wire the task board so the sentinel can inject remediation work.

The team has three members:
- **lead** — decomposes the goal into tasks
- **worker** — executes each task (including any dangerous ones the sentinel may block)
- **reviewer** — approves or rejects completed tasks

When the worker tries `rm -rf /tmp/sentinel-test-dir`, the sentinel blocks it and injects a new task: _"Propose a safe alternative to rm -rf ..."_ — the lead or worker can pick it up in the next round.

</ScrollyStep>

```python define risk patterns
# Bash commands the sentinel blocks outright
BLOCKED_PATTERNS = [
    "rm -rf",
    "mkfs",
    "dd if=",
    ":(){:|:&};:",    # fork bomb
    "curl | sh",
    "wget | sh",
    "chmod 777",
    "/etc/shadow",
    "DROP TABLE",
]

# Keywords in tool output that indicate a data leak
SENSITIVE_KEYWORDS = [
    "password=",
    "passwd=",
    "secret=",
    "api_key=",
    "token=",
    "private_key",
    "Authorization:",
    "Bearer ",
]

def _matches_blocked(text: str):
    t = text.lower()
    for pat in BLOCKED_PATTERNS:
        if pat.lower() in t:
            return pat
    return None

def _matches_sensitive(text: str):
    for kw in SENSITIVE_KEYWORDS:
        if kw.lower() in text.lower():
            return kw
    return None
```

```python sentinel state
import threading
from dataclasses import dataclass, field
from typing import Optional

@dataclass
class SentinelLog:
    blocked: list = field(default_factory=list)
    alerts: list = field(default_factory=list)
    injected_tasks: list = field(default_factory=list)
    _lock: threading.Lock = field(default_factory=threading.Lock)
    task_board: Optional[object] = None

    def record_block(self, session_id, tool, cmd, pattern):
        entry = {"session": session_id[:12], "tool": tool,
                 "cmd": cmd[:80], "pattern": pattern}
        with self._lock:
            self.blocked.append(entry)
        print(f"\n  ⛔  BLOCKED [{entry['session']}] {tool}: '{pattern}'")
        print(f"       cmd: {cmd[:80]}")

    def record_alert(self, session_id, tool, keyword, snippet):
        entry = {"session": session_id[:12], "tool": tool,
                 "keyword": keyword, "snippet": snippet[:100]}
        with self._lock:
            self.alerts.append(entry)
        print(f"\n  ⚠️   ALERT [{entry['session']}] {tool}: '{keyword}'")
        print(f"       snippet: {snippet[:100]}")

    def inject(self, description: str) -> Optional[str]:
        if self.task_board is None:
            return None
        task_id = self.task_board.post(description, "sentinel")
        with self._lock:
            self.injected_tasks.append({"id": task_id,
                                        "description": description})
        print(f"  📋  Injected: \"{description[:60]}\"")
        return task_id

LOG = SentinelLog()
```

```python hook handlers
def on_pre_tool_use(event: dict) -> Optional[dict]:
    """Fires before every tool call — can block execution."""
    p = event.get("payload", event)
    tool = p.get("tool", "")
    args = p.get("args", {})
    session_id = p.get("session_id", "<unknown>")

    # Only inspect shell tools
    if tool not in ("bash", "shell"):
        return None

    cmd = args.get("command", "") if isinstance(args, dict) else str(args)
    pattern = _matches_blocked(cmd)
    if pattern is None:
        return None

    LOG.record_block(session_id, tool, cmd, pattern)
    LOG.inject(
        f"SECURITY: '{pattern}' was blocked in session {session_id[:8]}. "
        f"Propose a safe alternative and explain the risk."
    )
    return {"action": "block",
            "reason": f"[sentinel] blocked pattern '{pattern}'"}


def on_post_tool_use(event: dict) -> Optional[dict]:
    """Fires after every tool call — observes output, cannot block."""
    p = event.get("payload", event)
    tool = p.get("tool", "")
    result = p.get("result", {})
    output = result.get("output", "") if isinstance(result, dict) \
             else str(result)
    session_id = p.get("session_id", "<unknown>")

    keyword = _matches_sensitive(output)
    if keyword is None:
        return None

    LOG.record_alert(session_id, tool, keyword, output)
    LOG.inject(
        f"DATA LEAK: '{keyword}' detected in {tool} output "
        f"(session {session_id[:8]}). Sanitize before continuing."
    )
    return None  # post_tool_use cannot block
```

```python attach sentinel
def attach_sentinel(session, label: str = "") -> None:
    tag = f"sentinel-{label}-" if label else "sentinel-"
    session.register_hook(
        f"{tag}pre", "pre_tool_use",
        config={"priority": 1},
        handler=on_pre_tool_use,
    )
    session.register_hook(
        f"{tag}post", "post_tool_use",
        config={"priority": 1, "async_execution": True},
        handler=on_post_tool_use,
    )
    print(f"  ✓ Sentinel attached{' to ' + label if label else ''} "
          f"({session.hook_count()} hooks)")
```

### TypeScript / Node.js

The same sentinel works in TypeScript. The hook handler receives a plain object `{ event_type, payload }`:

```typescript
import { Agent, Team, TeamRunner } from '@a3s-lab/code';

const BLOCKED_PATTERNS = ['rm -rf', 'mkfs', 'dd if=', 'DROP TABLE', 'curl | sh'];
const SENSITIVE_KEYWORDS = ['password=', 'secret=', 'api_key=', 'token=', 'Bearer '];

function matchBlocked(text: string): string | null {
  const t = text.toLowerCase();
  for (const p of BLOCKED_PATTERNS) {
    if (t.includes(p.toLowerCase())) return p;
  }
  return null;
}

function attachSentinel(session: ReturnType<Agent['session']>, label = '') {
  const tag = label ? `sentinel-${label}-` : 'sentinel-';

  session.registerHook(
    `${tag}pre`, 'pre_tool_use', null, { priority: 1 },
    (event) => {
      const p = (event.payload ?? event) as Record<string, unknown>;
      const tool = String(p.tool ?? '');
      if (!['bash', 'shell'].includes(tool)) return null;
      const args = p.args as Record<string, unknown> ?? {};
      const cmd = String(args.command ?? '');
      const pattern = matchBlocked(cmd);
      if (!pattern) return null;
      console.log(`\n  ⛔  BLOCKED [${tool}]: '${pattern}'`);
      return { action: 'block', reason: `[sentinel] blocked pattern '${pattern}'` };
    },
  );

  session.registerHook(
    `${tag}post`, 'post_tool_use', null,
    { priority: 1, asyncExecution: true },
    (event) => {
      const p = (event.payload ?? event) as Record<string, unknown>;
      const result = p.result as Record<string, unknown> ?? {};
      const output = String(result.output ?? '');
      for (const kw of SENSITIVE_KEYWORDS) {
        if (output.toLowerCase().includes(kw.toLowerCase())) {
          console.log(`\n  ⚠️   ALERT: sensitive keyword '${kw}' in output`);
          break;
        }
      }
      return null;
    },
  );
}

// Single agent
const agent = await Agent.create('agent.hcl');
const session = agent.session('/tmp/sentinel', { permissive: true });
attachSentinel(session, 'single');

const r = await session.send('Use bash to run: rm -rf /tmp/test');
console.log(`Agent response: ${r.text.slice(0, 200)}`);

// AgentTeam
const team = new Team('sentinel-team', {
  maxTasks: 20, maxRounds: 6, channelBuffer: 128, pollIntervalMs: 200,
});
team.addMember('lead',     'lead');
team.addMember('worker',   'worker');
team.addMember('reviewer', 'reviewer');

const runner = new TeamRunner(team);
const leadS     = agent.session('/tmp/sentinel-team', { permissive: true });
const workerS   = agent.session('/tmp/sentinel-team', { permissive: true });
const reviewerS = agent.session('/tmp/sentinel-team', { permissive: true });
attachSentinel(leadS,     'lead');
attachSentinel(workerS,   'worker');
attachSentinel(reviewerS, 'reviewer');
runner.bindSession('lead',     leadS);
runner.bindSession('worker',   workerS);
runner.bindSession('reviewer', reviewerS);

const result = await runner.runUntilDone(
  'Lead: post 3 tasks — ls /tmp, df -h /tmp, rm -rf /tmp/test. Worker: execute each.'
);
console.log(`Done: ${result.doneTasks.length}, Blocks: (see above)`);
```

### Rust

For maximum control — implement `HookHandler` directly, with typed access to all event fields:

```rust
use a3s_code_core::{
    Agent, SessionOptions,
    hooks::{Hook, HookConfig, HookEvent, HookEventType, HookHandler, HookMatcher, HookResponse},
};
use std::sync::Arc;

const BLOCKED: &[&str] = &["rm -rf", "mkfs", "dd if=", "DROP TABLE"];

struct SentinelGuard;

impl HookHandler for SentinelGuard {
    fn handle(&self, event: &HookEvent) -> HookResponse {
        let HookEvent::PreToolUse(e) = event else {
            return HookResponse::continue_();
        };
        let cmd = e.args.to_string();
        for pat in BLOCKED {
            if cmd.to_lowercase().contains(&pat.to_lowercase()) {
                eprintln!("⛔ BLOCKED [{}]: '{}'", e.tool, pat);
                return HookResponse::block(
                    format!("[sentinel] blocked pattern '{pat}'")
                );
            }
        }
        HookResponse::continue_()
    }
}

let agent = Agent::new("agent.hcl").await?;
let session = agent.session(".", Some(
    SessionOptions::new().with_permissive_policy()
))?;

session.register_hook(
    Hook::new("sentinel-guard", HookEventType::PreToolUse)
        .with_matcher(HookMatcher::tool("bash"))
        .with_config(HookConfig { priority: 1, ..Default::default() })
);
session.register_hook_handler("sentinel-guard", Arc::new(SentinelGuard));

// The sentinel propagates into every sub-agent spawned by the task tool
let result = session.send("Analyze the repo with multiple sub-agents", None).await?;
println!("{}", result.text);
```

## Key concepts

| Concept | Description |
|---------|-------------|
| `pre_tool_use` hook | Fires before a tool runs. Return `{"action": "block"}` to cancel execution. The agent sees the reason as an error message and can respond accordingly. |
| `post_tool_use` hook | Fires after a tool returns. Cannot block (execution already happened). Use for alerting and logging. |
| Hook priority | Lower numbers run first. Priority `1` runs before the built-in `SecurityGuard`. Use `1–9` for critical blocking hooks, `10+` for logging. |
| Sentinel propagation | Hooks registered on a parent session are automatically inherited by every sub-agent spawned by the `task` tool — at unlimited nesting depth. |
| `async_execution: True` | Runs the hook handler in a background task, so it never delays the agent loop. Use for `post_tool_use` alerts and logging. |
| `task_board.post(desc, "sentinel")` | Injects a new task into the `TeamTaskBoard` at runtime. The team's lead or workers will pick it up in the next round. |
| `TeamTaskBoard` | Obtained from `runner.task_board()` before calling `run_until_done()`. The sentinel holds a reference so it can inject work dynamically. |
