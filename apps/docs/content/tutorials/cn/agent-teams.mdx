# 构建多智能体审查流程

这篇教程使用当前 A3S Code Node SDK 构建代码审查流程：父 session
委派聚焦任务、汇总紧凑的子任务结果，并通过 run replay 检查运行状态。

<Callout type="info">
这篇教程讲的是 **model-driven** delegation——由 LLM 通过 `task` / `tasks` /
auto-delegation 决定委派什么。若需要**开发者编写的、确定性**的 step
（`parallel` / `pipeline` / `parallelResumable`），参见
[可编程编排](/cn/tutorials/orchestration)。
</Callout>

<Callout type="info">
下面的示例使用这些 Node SDK 入口：
`Agent.create`、`agent.session`、`session.registerAgentDir`、
`session.task`、`session.tasks`、`session.runs` 和
`session.runEvents`。
</Callout>

## 改了什么

常规多智能体工作流从普通 session 通过 task delegation 驱动。在启用
`autoDelegation` 后也可以自动启动 subagent。子运行返回紧凑结果，不把完整
transcript 塞回父 prompt。

```text
审查目标
  -> lead session
  -> task / tasks / automatic subagents
  -> 子任务摘要
  -> run replay 与 verification 证据
```

## Agent 配置

使用 ACL，不使用旧 HCL：

```text
default_model = "openai/docs-model"
max_parallel_tasks = 4
auto_parallel = false

providers "openai" {
  apiKey = env("A3S_OPENAI_API_KEY")
  baseUrl = env("A3S_OPENAI_BASE_URL")

  models "docs-model" {
    name = "Docs Model"
    tool_call = true
  }
}

auto_delegation {
  enabled        = true
  auto_parallel  = false
  min_confidence = 0.72
  max_tasks      = 3
}
```

## 可选角色文件

`registerAgentDir()` 会从目录加载角色定义。长期项目角色优先放在 `.a3s/agents`。

```text
.a3s/agents/
  release-reviewer.yaml
```

```yaml
name: release-reviewer
description: Reviews release blockers and separates required fixes from follow-up work.
prompt: |
  Inspect the requested scope. Report blockers first, then risks, then optional improvements.
```

## 协调器

```ts
import { Agent } from '@a3s-lab/code';
import path from 'node:path';

const agent = await Agent.create('agent.acl');
const session = agent.session(process.cwd(), {
  planningMode: 'auto',
  autoDelegation: { enabled: true, maxTasks: 3 },
  autoParallel: false,
});

const registered = session.registerAgentDir(path.join(process.cwd(), '.a3s/agents'));
console.log({ registered });

const summary = await session.task({
  agent: 'general',
  description: 'Review auth release blockers',
  prompt: 'Review src/auth for release blockers. Focus on token validation and error handling.',
  maxSteps: 4,
});

const parallel = await session.tasks([
  {
    agent: 'general',
    description: 'Auth security review',
    prompt: 'Review src/auth for security risks.',
    maxSteps: 3,
  },
  {
    agent: 'general',
    description: 'Auth test review',
    prompt: 'Review src/auth tests for missing edge cases.',
    maxSteps: 3,
  },
]);

const final = await session.send(`
Merge these child review results into a findings-first release note.

Single review:
${summary.output}

Parallel reviews:
${parallel.output}
`);

console.log(final.text);
console.log(final.verificationSummaryText);
console.log(await session.runs());
```

## 检查运行证据

用 replay API 获取运行证据：

```ts
const runs = await session.runs();
for (const run of runs) {
  console.log(run.id, run.status);
  console.log(await session.runEvents(run.id));
}
```

session 空闲时 `currentRun()` 可能是 `null`。读取已完成历史应使用 `runs()`，
取消前先检查 run 的状态。

## 生产边界

`session.bash()`、`session.tool()`、`session.git()` 等 direct host call 是特权
宿主操作。把这个流程暴露给不可信用户前，应在宿主应用层做访问控制。
