在本教程中，你将构建一个**哨兵智能体**——一个安全层，用于监控 AI 智能体（或整个 AgentTeam）发出的每次工具调用，在危险操作执行前将其阻断，检测工具输出中的敏感数据，并自动向团队工作队列中注入修复任务。

哨兵完全通过**生命周期 hooks** 实现，无需独立进程或 sidecar——只需在 session 上注册一次 hook 处理器，框架会自动将其传播到任意深度的所有子 agent。

<Callout type="info">
  请先安装 SDK：`pip install a3s-code`（Python）或 `npm install @a3s-lab/code`（Node.js）。在 `~/.a3s/config.hcl` 中配置 LLM 提供商密钥。
</Callout>

## 工作原理

```
Session
  │
  ├── register_hook("sentinel-pre",  pre_tool_use,  handler=block_dangerous)
  └── register_hook("sentinel-post", post_tool_use, handler=detect_sensitive)
                              │
          ┌───────────────────┴────────────────────┐
          │                                        │
     Pre-tool-use                           Post-tool-use
   工具执行前触发                             工具执行后触发
          │                                        │
   ┌──────▼──────┐                        ┌───────▼───────┐
   │  匹配危险   │                        │  匹配输出中   │
   │   模式？    │                        │  的敏感关键词？│
   └──────┬──────┘                        └───────┬───────┘
          │ 是                                    │ 是
          ▼                                       ▼
     { action: "block" }                  记录告警
     agent 收到错误消息               向 AgentTeam 任务板
     执行停止                         注入修复任务
```

**Hook 传播** — 在父 session 上注册的 hooks 会被 `task` 工具派生的每个子 agent 自动继承，无深度限制：

```
父 session（在此注册哨兵 hooks）
  └── task 工具派生 子 AgentLoop
        ├── 继承哨兵 hook engine
        └── task 工具派生 孙 AgentLoop
              └── 继承哨兵 hook engine
```

</ScrollyStep>

<ScrollyStep index={2}>

### 定义风险模式

两个模式列表控制哨兵监控的内容：

- **`BLOCKED_PATTERNS`** — bash 命令参数中应被直接阻断的子字符串。`pre_tool_use` hook 在工具执行前触发，因此阻断在任何损害发生前生效。
- **`SENSITIVE_KEYWORDS`** — 工具输出中表示数据泄露的关键词。`post_tool_use` hook 在工具返回后触发，只能告警和记录，无法阻断。

</ScrollyStep>

<ScrollyStep index={3}>

### 哨兵状态

单个 `SentinelLog` 对象追踪哨兵观察到的一切：

- `blocked` — 被阻断的命令列表，含 session ID 和匹配的模式
- `alerts` — 敏感数据检测列表，含关键词和输出片段
- `injected_tasks` — 发布到团队任务板的修复任务列表
- `task_board` — 在运行 AgentTeam 前设置此字段，以便哨兵动态注入任务

</ScrollyStep>

<ScrollyStep index={4}>

### Hook 处理器

两个处理器函数，分别对应两个 hook 事件：

**`on_pre_tool_use`** — 在每次工具执行前调用。返回 `{"action": "block", "reason": "..."}` 取消调用，或返回 `None` 放行。处理器：
1. 从 `event["payload"]["args"]["command"]` 提取命令
2. 与 `BLOCKED_PATTERNS` 匹配
3. 若匹配：记录阻断，注入修复任务，返回阻断响应

**`on_post_tool_use`** — 在每次工具执行后调用。无法阻断（工具已运行完毕）。处理器：
1. 从 `event["payload"]["result"]["output"]` 提取输出
2. 与 `SENSITIVE_KEYWORDS` 匹配
3. 若匹配：记录告警，注入数据泄露审查任务

</ScrollyStep>

<ScrollyStep index={5}>

### 附加哨兵

`attach_sentinel` 用一次函数调用在任意 session 上注册两个 hooks。

关键参数：
- **`priority: 1`** — 在内置 `SecurityGuard`（优先级 1，最高）之前运行
- **`async_execution: True`**（post hook）— 允许告警和任务注入在后台运行，不阻塞 agent 循环

一旦附加，hooks 会自动传播到 `task` 工具派生的任何子 agent，无需重新注册。

</ScrollyStep>

<ScrollyStep index={6}>

### 单智能体演示

将哨兵附加到 session，然后发送三条提示：

1. **安全命令** — `ls /tmp | head -5`。通过，无干预。
2. **危险命令** — `rm -rf /tmp/sentinel-test`。哨兵在 bash 运行前阻断。Agent 收到错误消息，可解释原因。
3. **敏感输出** — `echo 'api_key=sk-abc123 token=xyz789'`。命令正常运行（不危险），但 post-hook 检测到敏感关键词并记录告警。

</ScrollyStep>

<ScrollyStep index={7}>

### 带哨兵的 AgentTeam

对于团队，需要**分别**将哨兵附加到每个成员的 session，然后连接任务板以便哨兵注入修复工作。

团队有三个成员：
- **lead** — 将目标分解为任务
- **worker** — 执行每个任务（包括哨兵可能阻断的危险任务）
- **reviewer** — 批准或拒绝已完成的任务

当 worker 尝试 `rm -rf /tmp/sentinel-test-dir` 时，哨兵阻断它并注入一个新任务：_"为 rm -rf 提出安全替代方案..."_ — lead 或 worker 可在下一轮拾取该任务。

</ScrollyStep>

```python 定义风险模式
# 直接阻断的 bash 命令模式
BLOCKED_PATTERNS = [
    "rm -rf",
    "mkfs",
    "dd if=",
    ":(){:|:&};:",    # fork bomb
    "curl | sh",
    "wget | sh",
    "chmod 777",
    "/etc/shadow",
    "DROP TABLE",
]

# 工具输出中表示数据泄露的关键词
SENSITIVE_KEYWORDS = [
    "password=",
    "passwd=",
    "secret=",
    "api_key=",
    "token=",
    "private_key",
    "Authorization:",
    "Bearer ",
]

def _matches_blocked(text: str):
    t = text.lower()
    for pat in BLOCKED_PATTERNS:
        if pat.lower() in t:
            return pat
    return None

def _matches_sensitive(text: str):
    for kw in SENSITIVE_KEYWORDS:
        if kw.lower() in text.lower():
            return kw
    return None
```

```python 哨兵状态
import threading
from dataclasses import dataclass, field
from typing import Optional

@dataclass
class SentinelLog:
    blocked: list = field(default_factory=list)
    alerts: list = field(default_factory=list)
    injected_tasks: list = field(default_factory=list)
    _lock: threading.Lock = field(default_factory=threading.Lock)
    task_board: Optional[object] = None

    def record_block(self, session_id, tool, cmd, pattern):
        entry = {"session": session_id[:12], "tool": tool,
                 "cmd": cmd[:80], "pattern": pattern}
        with self._lock:
            self.blocked.append(entry)
        print(f"\n  ⛔  已阻断 [{entry['session']}] {tool}: '{pattern}'")
        print(f"       命令: {cmd[:80]}")

    def record_alert(self, session_id, tool, keyword, snippet):
        entry = {"session": session_id[:12], "tool": tool,
                 "keyword": keyword, "snippet": snippet[:100]}
        with self._lock:
            self.alerts.append(entry)
        print(f"\n  ⚠️   告警 [{entry['session']}] {tool}: '{keyword}'")
        print(f"       片段: {snippet[:100]}")

    def inject(self, description: str) -> Optional[str]:
        if self.task_board is None:
            return None
        task_id = self.task_board.post(description, "sentinel")
        with self._lock:
            self.injected_tasks.append({"id": task_id,
                                        "description": description})
        print(f"  📋  已注入: \"{description[:60]}\"")
        return task_id

LOG = SentinelLog()
```

```python Hook 处理器
def on_pre_tool_use(event: dict) -> Optional[dict]:
    """工具调用前触发 — 可以阻断执行"""
    p = event.get("payload", event)
    tool = p.get("tool", "")
    args = p.get("args", {})
    session_id = p.get("session_id", "<unknown>")

    # 只检查 shell 类工具
    if tool not in ("bash", "shell"):
        return None

    cmd = args.get("command", "") if isinstance(args, dict) else str(args)
    pattern = _matches_blocked(cmd)
    if pattern is None:
        return None

    LOG.record_block(session_id, tool, cmd, pattern)
    LOG.inject(
        f"安全: '{pattern}' 已在 session {session_id[:8]} 中被阻断。"
        f"请提出安全替代方案并说明原始命令的危险性。"
    )
    return {"action": "block",
            "reason": f"[哨兵] 阻断模式 '{pattern}'"}


def on_post_tool_use(event: dict) -> Optional[dict]:
    """工具调用后触发 — 观察输出，无法阻断"""
    p = event.get("payload", event)
    tool = p.get("tool", "")
    result = p.get("result", {})
    output = result.get("output", "") if isinstance(result, dict) \
             else str(result)
    session_id = p.get("session_id", "<unknown>")

    keyword = _matches_sensitive(output)
    if keyword is None:
        return None

    LOG.record_alert(session_id, tool, keyword, output)
    LOG.inject(
        f"数据泄露: 在 {tool} 输出中检测到 '{keyword}' "
        f"(session {session_id[:8]})。请在继续前清理或脱敏。"
    )
    return None  # post_tool_use 无法阻断
```

```python 附加哨兵
def attach_sentinel(session, label: str = "") -> None:
    tag = f"sentinel-{label}-" if label else "sentinel-"
    session.register_hook(
        f"{tag}pre", "pre_tool_use",
        config={"priority": 1},
        handler=on_pre_tool_use,
    )
    session.register_hook(
        f"{tag}post", "post_tool_use",
        config={"priority": 1, "async_execution": True},
        handler=on_post_tool_use,
    )
    print(f"  ✓ 哨兵已附加{' 到 ' + label if label else ''} "
          f"({session.hook_count()} 个 hooks)")
```

### TypeScript / Node.js

同样的哨兵在 TypeScript 中同样有效。hook 处理器接收纯对象 `{ event_type, payload }`：

```typescript
import { Agent, Team, TeamRunner } from '@a3s-lab/code';

const BLOCKED_PATTERNS = ['rm -rf', 'mkfs', 'dd if=', 'DROP TABLE', 'curl | sh'];
const SENSITIVE_KEYWORDS = ['password=', 'secret=', 'api_key=', 'token=', 'Bearer '];

function matchBlocked(text: string): string | null {
  const t = text.toLowerCase();
  for (const p of BLOCKED_PATTERNS) {
    if (t.includes(p.toLowerCase())) return p;
  }
  return null;
}

function attachSentinel(session: ReturnType<Agent['session']>, label = '') {
  const tag = label ? `sentinel-${label}-` : 'sentinel-';

  session.registerHook(
    `${tag}pre`, 'pre_tool_use', null, { priority: 1 },
    (event) => {
      const p = (event.payload ?? event) as Record<string, unknown>;
      const tool = String(p.tool ?? '');
      if (!['bash', 'shell'].includes(tool)) return null;
      const args = p.args as Record<string, unknown> ?? {};
      const cmd = String(args.command ?? '');
      const pattern = matchBlocked(cmd);
      if (!pattern) return null;
      console.log(`\n  ⛔  已阻断 [${tool}]: '${pattern}'`);
      return { action: 'block', reason: `[哨兵] 阻断模式 '${pattern}'` };
    },
  );

  session.registerHook(
    `${tag}post`, 'post_tool_use', null,
    { priority: 1, asyncExecution: true },
    (event) => {
      const p = (event.payload ?? event) as Record<string, unknown>;
      const result = p.result as Record<string, unknown> ?? {};
      const output = String(result.output ?? '');
      for (const kw of SENSITIVE_KEYWORDS) {
        if (output.toLowerCase().includes(kw.toLowerCase())) {
          console.log(`\n  ⚠️   告警: 输出中检测到敏感关键词 '${kw}'`);
          break;
        }
      }
      return null;
    },
  );
}

// 单智能体
const agent = await Agent.create('agent.hcl');
const session = agent.session('/tmp/sentinel', { permissive: true });
attachSentinel(session, 'single');

const r = await session.send('使用 bash 运行: rm -rf /tmp/test');
console.log(`Agent 响应: ${r.text.slice(0, 200)}`);

// AgentTeam
const team = new Team('sentinel-team', {
  maxTasks: 20, maxRounds: 6, channelBuffer: 128, pollIntervalMs: 200,
});
team.addMember('lead',     'lead');
team.addMember('worker',   'worker');
team.addMember('reviewer', 'reviewer');

const runner = new TeamRunner(team);
const leadS     = agent.session('/tmp/sentinel-team', { permissive: true });
const workerS   = agent.session('/tmp/sentinel-team', { permissive: true });
const reviewerS = agent.session('/tmp/sentinel-team', { permissive: true });
attachSentinel(leadS,     'lead');
attachSentinel(workerS,   'worker');
attachSentinel(reviewerS, 'reviewer');
runner.bindSession('lead',     leadS);
runner.bindSession('worker',   workerS);
runner.bindSession('reviewer', reviewerS);

const result = await runner.runUntilDone(
  'Lead: 发布 3 个任务——ls /tmp、df -h /tmp、rm -rf /tmp/test。Worker: 逐一执行。'
);
console.log(`已完成: ${result.doneTasks.length}`);
```

### Rust

最大控制力——直接实现 `HookHandler`，可类型安全地访问所有事件字段：

```rust
use a3s_code_core::{
    Agent, SessionOptions,
    hooks::{Hook, HookConfig, HookEvent, HookEventType, HookHandler, HookMatcher, HookResponse},
};
use std::sync::Arc;

const BLOCKED: &[&str] = &["rm -rf", "mkfs", "dd if=", "DROP TABLE"];

struct SentinelGuard;

impl HookHandler for SentinelGuard {
    fn handle(&self, event: &HookEvent) -> HookResponse {
        let HookEvent::PreToolUse(e) = event else {
            return HookResponse::continue_();
        };
        let cmd = e.args.to_string();
        for pat in BLOCKED {
            if cmd.to_lowercase().contains(&pat.to_lowercase()) {
                eprintln!("⛔ 已阻断 [{}]: '{}'", e.tool, pat);
                return HookResponse::block(
                    format!("[哨兵] 阻断模式 '{pat}'")
                );
            }
        }
        HookResponse::continue_()
    }
}

let agent = Agent::new("agent.hcl").await?;
let session = agent.session(".", Some(
    SessionOptions::new().with_permissive_policy()
))?;

session.register_hook(
    Hook::new("sentinel-guard", HookEventType::PreToolUse)
```

## 核心概念

| 概念 | 说明 |
|------|------|
| `pre_tool_use` hook | 工具运行前触发。返回 `{"action": "block"}` 取消执行。Agent 将阻断原因作为错误消息接收，并可据此响应。 |
| `post_tool_use` hook | 工具返回后触发。无法阻断（执行已完成）。用于告警和日志记录。 |
| Hook 优先级 | 数字越小越先执行。优先级 `1` 在内置 `SecurityGuard` 之前运行。关键阻断 hooks 使用 `1–9`，日志 hooks 使用 `10+`。 |
| 哨兵传播 | 在父 session 上注册的 hooks 会被 `task` 工具派生的每个子 agent 自动继承——深度无限制。 |
| `async_execution: True` | 在后台任务中运行 hook 处理器，不延迟 agent 循环。适用于 `post_tool_use` 告警和日志记录。 |
| `task_board.post(desc, "sentinel")` | 运行时向 `TeamTaskBoard` 注入新任务。团队的 lead 或 worker 将在下一轮拾取该任务。 |
| `TeamTaskBoard` | 在调用 `run_until_done()` 前通过 `runner.task_board()` 获取。哨兵持有该引用以动态注入工作。 |
