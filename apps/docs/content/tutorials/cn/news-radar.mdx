# 构建消息雷达流程

这篇教程展示消息流程中的 A3S Code 部分：注册 stdio MCP server、直接调用工具，
并把返回内容流式合成为简报。

<Callout type="warn">
第三方 fetch server、复杂网页抓取、RSS 语义和搜索新鲜度都属于应用层责任。把
这些 live sources 当作生产输入前，应先在你的应用中测试。
</Callout>

## 构件

- `session.toolNames()` 包含 `web_search` 与 `web_fetch`。
- `session.addMcpServer()` 可以启动 stdio MCP server。
- `session.mcpStatus()` 可以报告已注册 MCP server。
- `session.tool('mcp__server__tool', args)` 可以调用已注册 MCP tool。
- `session.stream()` 可以合成简报；消费方式使用 `.next()`。

## Session

```ts
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.acl');
const session = agent.session(process.cwd(), {
  builtinSkills: true,
  planningMode: 'enabled',
});

console.log(session.toolNames());
```

## 注册 Stdio MCP Server

这个示例使用一个最小本地 stdio MCP server，并调用 `mcp__echo__echo` 与
`mcp__echo__get_secret` 调用。
下面的示例假设 workspace 中存在
`tools/mcp_echo_server.mjs`；替换成自己的 stdio MCP server 前，应先测试那个
server。

```ts
await session.addMcpServer('echo', 'stdio', process.execPath, [
  'tools/mcp_echo_server.mjs',
  'secret',
]);

console.log(await session.mcpStatus());

const echo = await session.tool('mcp__echo__echo', {
  message: 'news-radar check',
});

console.log(echo.output);
```

## Live Fetch 边界

`web_search` 和 `web_fetch` 已出现在 tool registry 中。这里不断言真实网络结果、
新鲜度或来源质量。把 production news-fetching pipeline 接入前，应先测试计划使用
的 live sources。

## 流式生成简报

```ts
const stream = await session.stream(`
Create a concise news briefing from these source excerpts.
Separate confirmed facts from uncertain or missing data.

Source excerpt:
${echo.output}
`);

while (true) {
  const { value: event, done } = await stream.next();
  if (done) break;
  if (event?.text) process.stdout.write(event.text);
}
```

## 移除 MCP Server

```ts
await session.removeMcpServer('echo');
console.log(await session.mcpStatus());
```

第三方 fetch server、browser automation server 或定时生产抓取，只有在你的环境中
测试通过后，才应纳入这套流程。
