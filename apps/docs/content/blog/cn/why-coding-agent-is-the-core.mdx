# A3S Code v2.1.0：Coding Agent 运行时

A3S Code 是可嵌入的 coding-agent runtime。宿主应用创建 agent，打开绑定
workspace 的 session，然后通过 prompt、tools、确定性程序、委派任务、replay 与
verification evidence 驱动工作。

```ts
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.acl');
const session = agent.session(process.cwd(), {
  builtinSkills: true,
  planningMode: 'enabled',
});

const result = await session.send(
  'Find where authentication errors are handled and summarize the flow.',
);

console.log(result.text);
console.log(result.verificationSummaryText);
```

## 运行时表面

- 使用 `providers` 与 `models` block 的 ACL 配置。
- OpenAI-compatible 的 `apiKey`/`baseUrl` 与 `api_key`/`base_url` alias。
- session options：model override、builtin skills、prompt slots、planning、
  file-backed memory/session stores、auto-save、compaction、continuation 与
  permission-policy shape。
- Direct tools：read、glob、grep、bash、git、generic `tool`、skill search 与
  tool registry inspection。
- 通过 `session.program()` 和核心 `program` tool 使用 Programmatic Tool Calling。
- 通过 `delegateTask` 与 `parallelTask` 做任务委派。
- Run replay、trace events、verification summaries 与 run-scoped inspection。
- Live stdio MCP 注册与 tool invocation。
- Queue inspection 与 external-task primitives。
- 通过 `save()` 和 `agent.resumeSession()` 做 session persistence。

## Streaming

使用 `.next()` 消费 stream：

```ts
const stream = await session.stream('Explain the release risk in one paragraph.');

while (true) {
  const { value: event, done } = await stream.next();
  if (done) break;
  if (event?.text) process.stdout.write(event.text);
}
```

流式消费方式以你安装的 SDK 版本文档为准。

## 为什么 Coding Agent 仍然是核心

Coding agent 的价值在于它能通过客观反馈闭环：读代码、修改、运行命令、检查
失败、再尝试。A3S Code v2.1.0 把这条闭环暴露成可嵌入运行时：sessions、
tools、PTC、delegation、memory、replay 与 verification evidence。

完整 API 表面见 [API Contract](/cn/docs/code/api-contract)。
