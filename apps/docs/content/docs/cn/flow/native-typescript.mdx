# Native TypeScript Workflows

A3S Flow 是 Rust SDK。`NativeTsRuntime` 是可选 runtime adapter，适用于希望 workflow
authors 写 TypeScript、但仍由 Rust 掌控 run creation、event storage、replay、workers、
scheduling 和 inspection 的 hosts。

这不是 standalone TypeScript SDK。TypeScript source 会被编译成 native artifact，并通过
versioned JSON protocol 调用。

## Compiler Contract

`NativeTsRuntime` 期望 compiler executable 支持这个命令形状：

```bash
a3s-flow-native-compiler compile <entrypoint.ts> -o <artifact>
```

生成的 artifact 必须：

- 可由 host 执行，
- 接受 `--a3s-flow-runtime`，
- 从 stdin 读取一个 `NativeRuntimeRequest` JSON object，
- 向 stdout 写出一个 `NativeRuntimeResponse` JSON object，
- 把 workflow requests dispatch 到配置的 export，
- 按 `payload.step_name` dispatch step requests。

## Configure The Runtime

```rust
use a3s_flow::{NativeTsRuntime, NativeTsRuntimeConfig};

let runtime = NativeTsRuntime::new(NativeTsRuntimeConfig::new(
    "/path/to/a3s-flow-native-compiler",
    ".a3s-flow/native-ts",
    ".",
));
```

Examples 也可以从环境变量读取 compiler path：

```bash
A3S_FLOW_NATIVE_TS_COMPILER=/path/to/a3s-flow-native-compiler \
  cargo run --example native_ts_greeting

A3S_FLOW_NATIVE_TS_COMPILER=/path/to/a3s-flow-native-compiler \
  cargo run --example native_ts_preflight
```

未设置该变量时，examples 会打印缺失 prerequisite 并成功退出，使常规 Rust example suite
保持可移植。

## Preflight

Host 想要提前 validation 时调用 `preflight(&spec)`：

```rust
let preflight = runtime.preflight(&spec).await?;

println!("entrypoint={}", preflight.entrypoint.display());
println!("artifact={}", preflight.artifact.display());
println!("source_hash={}", preflight.source_hash);
println!("cache_hit={}", preflight.cache_hit);
```

Preflight 会校验 spec、解析 entrypoint、hash source 和 runtime identity，仅在 artifact
cache cold 时编译，并在 compilation failure 时把 compiler stderr 作为 runtime error 返回。
