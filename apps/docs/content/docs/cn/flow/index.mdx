# A3S Flow

A3S Flow 是为智能体平台设计的工作流引擎，执行 JSON 定义的 DAG，支持并发波次调度、可插拔节点类型和完整生命周期控制。为 A3S 生态系统构建，提供 Dify 兼容的工作流执行，支持暂停/恢复功能和可扩展的节点注册表。

## 架构

```
  外部调用者
  ┌─────────────────────────────────────────────────────────────┐
  │                       FlowEngine                            │
  │                                                             │
  │  node_types() → Vec<String>   列出已注册的节点类型           │
  │  start(def, vars) → Uuid      解析 DAG，生成任务，返回       │
  │  pause(id)                    在波次边界发出暂停信号          │
  │  resume(id)                   解除暂停的执行                 │
  │  terminate(id)                通过 CancellationToken 取消   │
  │  state(id) → ExecutionState   快照当前状态                  │
  │                                                             │
  │  ┌───────────────────────────────────────────────────────┐  │
  │  │               ExecutionState (每个 Uuid)              │  │
  │  │                                                       │  │
  │  │            ┌──────────┐                               │  │
  │  │   start() ►│ Running  │◄─── resume()                  │  │
  │  │            └────┬─────┘                               │  │
  │  │    pause() ─────┤  terminate()                        │  │
  │  │                 │  ├──────────────► Terminated        │  │
  │  │                 ▼  │  节点错误   ► Failed(msg)        │  │
  │  │            ┌────────┴─┐  全部完成 ► Completed(result) │  │
  │  │            │  Paused  │                               │  │
  │  │            └──────────┘                               │  │
  │  └───────────────────────────────────────────────────────┘  │
  └────────────────────────────┬────────────────────────────────┘
                               │  生成后台 Tokio 任务
                               │  (watch::Receiver + CancellationToken)
                               ▼
  ┌─────────────────────────────────────────────────────────────┐
  │                       FlowRunner                            │
  │                  (每个执行一个任务)                          │
  │                                                             │
  │  波次 1 │ fetch                    无依赖 → 立即运行         │
  │         │  └─ outputs["fetch"]                             │
  │  波次 2 │ summarize                fetch 完成 → 立即运行     │
  │         │  └─ outputs["summarize"]                         │
  │  波次 3 │ branch_a  branch_b       都就绪 → 并发执行        │
  │         │  └─ outputs["branch_a"], outputs["branch_b"]     │
  │  波次 4 │ notify                   扇入汇聚 → 立即运行       │
  │         │  └─ outputs["notify"]                            │
  └─────────────────────────────────────────────────────────────┘
```

## 核心特性

- **JSON 原生工作流** — 纯 JSON 对象（`{ nodes, edges }`），无需 YAML 或 DSL
- **并发波次调度** — 无相互依赖的节点通过 Tokio JoinSet 并行运行
- **完整生命周期控制** — 在波次边界暂停、恢复或中途取消执行
- **可插拔节点类型** — 实现 `Node` trait 添加自定义节点（LLM、HTTP、MCP 等）
- **Dify 兼容** — 支持 Dify 工作流格式，内置 14+ 节点类型
- **共享可变上下文** — 跨节点状态共享，用于对话上下文和元数据
- **环检测** — 在执行前的解析时验证 DAG 结构
- **条件执行** — `run_if` 守卫实现条件节点执行
- **子流程支持** — `iteration` 和 `loop` 节点支持嵌套工作流执行
- **事件流** — 订阅执行事件（节点开始/完成、波次进度）
- **Rust、Python、TypeScript SDK** — 多语言支持，API 一致
- **零拷贝执行** — 通过 Arc 和 async/await 实现高效内存使用

## 使用场景

- **智能体工作流** — 多步骤智能体任务，包含 LLM 调用、工具使用和决策树
- **数据管道** — ETL 工作流，包含 HTTP 获取、CSV 解析和转换
- **编排** — 协调多个服务，支持条件路由和错误处理
- **自动化** — 定时任务，支持循环、重试和通知
- **集成** — 通过自定义节点连接外部 API、数据库和消息队列
