# Rust API

安装 Rust crate：

```toml
[dependencies]
a3s-acl = "0.2.2"
```

## Parse And Generate

```rust
use a3s_acl::{generate, parse};

let doc = parse(r#"
default_model = "provider/model-id"

providers "provider" {
  api_key = env("PROVIDER_API_KEY")
}
"#)?;

let text = generate(&doc);
# Ok::<(), a3s_acl::ParseError>(())
```

Public crate exports：

| API | 作用 |
| --- | --- |
| `parse` / `parse_acl` | 把 ACL text 解析成 `Document`。 |
| `parse_with_limits`、`ParseLimits` | 使用显式 document、nesting、collection、token 和 diagnostic limits 解析 untrusted input。 |
| `collect_diagnostics`、`collect_diagnostics_with_limits` | 在不构造 partial AST 的前提下收集 bounded diagnostics。 |
| `DiagnosticCode`、`ParseError`、`DiagnosticReport` | 返回 stable codes、UTF-8 byte spans、redacted messages 和明确的 truncation state。 |
| `generate` / `generate_acl` | 从 `Document` 生成 ACL text。 |
| `generate_from_map` | 从 `HashMap<String, Value>` 生成 ACL text。 |
| `canonical_bytes`、`canonical_digest` | 生成 stable cross-SDK bytes 和带 algorithm prefix 的 SHA-256 digest。 |
| `CanonicalError` | 拒绝 unsupported programmatic values 且不回显值。 |
| `Lexer` / `Parser` | 高级 tokenization 和 parsing entry points。 |

## Bounded Admission

```rust
use a3s_acl::{parse_with_limits, ParseLimits};

let input = "limits { max_tasks = 10 }";
let document = parse_with_limits(
    input,
    ParseLimits {
        max_document_bytes: 64 * 1024,
        max_nesting_depth: 32,
        max_collection_items: 1_000,
        max_token_bytes: 16 * 1024,
        max_diagnostics: 20,
    },
)?;
```

Document 和 token sizes 使用 UTF-8 bytes。`ParseError` 提供 stable
`DiagnosticCode`、带 UTF-8 byte offsets 的 complete source span，以及
compatibility `line` / `column` fields。Messages 只标识 token kinds，不包含
source token values 或 snippets。

Parse APIs 仍然是 fail-fast。CLI 和 editor integrations 可以使用 bounded
budget 收集多个错误：

```rust
use a3s_acl::{collect_diagnostics_with_limits, ParseLimits};

let report = collect_diagnostics_with_limits(
    "first = ]\nsecond = ]",
    ParseLimits {
        max_diagnostics: 20,
        ..ParseLimits::default()
    },
);
```

发生 syntax error 后，collection 会从下一 source line 继续。
Resource-limit diagnostics 仍然是 fatal。Report 最多保存
`max_diagnostics` 个 errors，只有在继续观察到超出 budget 的 error 后才会把
`truncated` 设为 true。

## AST

| Type | Fields |
| --- | --- |
| `Document` | `blocks: Vec<Block>` |
| `Block` | `name`、`labels`、`blocks`、`attributes` |
| `Value` | `String`、`Number`、`Bool`、`List`、`Object`、`Null`、`Call` |

`Value` 还暴露 `as_str`、`as_number`、`as_bool`、`is_null`、`is_string` 等 helpers。

## Builders

```rust
use a3s_acl::builder::{boolean, call, string, BlockBuilder, DocumentBuilder};

let provider = BlockBuilder::new("providers")
    .label("provider")
    .attr("api_key", call("env", vec![string("PROVIDER_API_KEY")]))
    .attr("enabled", boolean(true))
    .build();

let doc = DocumentBuilder::new().block(provider).build();
```

Builder helpers 包括 `string`、`number`、`integer`、`boolean`、`null`、`list`、`call`。

## Native Generation

```rust
use a3s_acl::{Generator, GeneratorConfig};

let generator = Generator::with_config(GeneratorConfig {
    indent: "    ",
    comments: false,
});
let text = generator.generate(&doc);
```

Generation 只有一个 native ACL contract：block labels 保留在 block headers。
不存在 label-as-attribute compatibility mode。

## Canonical Bytes And Digests

```rust
use a3s_acl::{canonical_bytes, canonical_digest};

let bytes = canonical_bytes(&doc)?;
let digest = canonical_digest(&doc)?;
assert!(digest.starts_with("sha256:"));
```

签名、durable identity 和 cross-SDK comparisons 应使用 canonical APIs。
Canonical bytes 会保留 ordered ACL constructs，同时对 semantically
unordered attribute 和 object keys 排序。

## Notes

- Generated attributes 会按 key 排序，方便稳定输出。
- Strings 会 escape quotes、backslashes、newlines、carriage returns 和 tabs。
- ACL parse/generation 不解析 secrets，也不执行 function calls。
