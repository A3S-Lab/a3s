# Node SDK

安装 Node package：

```bash
npm install @a3s-lab/acl
```

仓库中的 package version 是 `0.2.2`。

## Parse And Generate

```typescript
import {
  BlockBuilder,
  DocumentBuilder,
  call,
  generate,
  parse,
  string,
} from '@a3s-lab/acl';

const doc = parse(`
providers "provider" {
  api_key = env("PROVIDER_API_KEY")
}
`);

const provider = new BlockBuilder('providers')
  .label('provider')
  .attr('api_key', call('env', [string('PROVIDER_API_KEY')]))
  .build();

const generated = generate(new DocumentBuilder().block(provider).build());
```

## Public API

| API | 作用 |
| --- | --- |
| `parse(input, limits?)` | 使用默认或显式 resource limits，把 ACL text 解析成 `Document`。 |
| `collectDiagnostics(input, limits?)` | 在不构造 partial AST 的前提下收集 bounded diagnostics。 |
| `DEFAULT_PARSE_LIMITS` | Document bytes、nesting、collection items、token bytes 和 diagnostic count 的 shared immutable defaults。 |
| `ParseError`、`DIAGNOSTIC_CODES`、`DiagnosticReport` | Stable codes、complete spans、UTF-8 byte offsets、redacted messages 和明确的 truncation state。 |
| `generate(doc)` | 生成 labels 位于 block headers 的 native ACL text。 |
| `canonicalBytes(doc)`、`canonicalDigest(doc)` | 生成 stable cross-SDK bytes 和带 algorithm prefix 的 SHA-256 digest。 |
| `CanonicalError` | 对 non-finite numbers、non-scalar strings 和 non-portable identifiers 返回不回显值的错误。 |
| `string`、`number`、`boolean`、`bool`、`nullValue`、`list`、`call` | Value constructors。 |
| `BlockBuilder`、`DocumentBuilder` | Structured document builders。 |
| `Lexer`、`Parser` | Advanced tokenization/parsing classes。 |

Package 在 `src/index.d.ts` 中提供 TypeScript definitions。

## Bounded Admission

在 API boundary 传入显式 limits：

```typescript
import {ParseError, parse} from '@a3s-lab/acl';

const source = 'limits { max_tasks = 10 }';

try {
  const document = parse(source, {
    maxDocumentBytes: 64 * 1024,
    maxNestingDepth: 32,
    maxCollectionItems: 1_000,
    maxTokenBytes: 16 * 1024,
    maxDiagnostics: 20,
  });
} catch (error) {
  if (error instanceof ParseError) {
    console.error(error.code, error.span);
  }
}
```

Document 和 token sizes 使用 UTF-8 bytes。Diagnostic messages 只标识 token
kinds，不包含 source token values 或 snippets。

Parse API 仍然是 fail-fast。CLI 和 editor integrations 可以使用 bounded
budget 收集多个错误：

```typescript
import {collectDiagnostics} from '@a3s-lab/acl';

const report = collectDiagnostics('first = ]\nsecond = ]', {
  maxDiagnostics: 20,
});
```

发生 syntax error 后，collection 会从下一 source line 继续。
Resource-limit diagnostics 仍然是 fatal。Report 最多保存
`maxDiagnostics` 个 errors，只有在继续观察到超出 budget 的 error 后才会把
`truncated` 设为 true。

## Native And Canonical Output

`generate` 只有一个 native ACL output contract。Labels 保留在 block headers；
alternate generator modes 和 generator options 均不支持。

签名、durable identity 和 cross-SDK comparisons 应使用 `canonicalBytes` 或
`canonicalDigest`。Canonical bytes 会保留 ordered ACL constructs，同时对
semantically unordered attribute 和 object keys 排序。

## 边界

SDK 是 parser/generator 和 AST builder。它不会加载环境变量、拉取远程配置、验证 provider
credentials，也不会执行 `env(...)` 等函数。
