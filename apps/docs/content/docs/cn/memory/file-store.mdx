# FileMemoryStore

默认的持久化后端。每条记忆对应一个 JSON 文件，外加一个轻量索引用于快速搜索。

## 目录结构

```
memory/
  index.json          ← 轻量索引（id、tags、importance、timestamp）
  items/
    {uuid}.json       ← 完整记忆内容，原子写入
```

## 使用方式

```rust
use a3s_memory::{FileMemoryStore, MemoryItem, MemoryStore};

let store = FileMemoryStore::new("./memory").await?;

store.store(MemoryItem::new("通过 `just release` 完成部署").with_importance(0.9)).await?;

let results = store.search("部署", 5).await?;
let recent  = store.get_recent(10).await?;
let top     = store.get_important(0.7, 10).await?;
```

## 关键特性

- **原子写入** — 通过 `.tmp` → rename 写入，崩溃不会导致数据损坏
- **内存索引** — 启动时加载，`search()` 和 `search_by_tags()` 不访问磁盘
- **路径穿越防护** — 记忆 ID 在用作文件名前会被净化
- **持久化** — 进程重启后仍然保留，重新构建实例即可加载

## 索引重建

`index.json` 被删除或损坏时，无需数据损失即可恢复：

```rust
let store = FileMemoryStore::new("./memory").await?;

// 扫描 items/ 目录，从 item 文件重建 index.json
let recovered = store.rebuild_index().await?;
println!("已恢复 {} 条记忆", recovered);
```
