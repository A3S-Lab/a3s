# A3S Box

A3S Box 用 libkrun MicroVM 运行 Linux OCI 工作负载。它提供类 Docker CLI、本地镜像仓库、卷、TCP 端口发布、可选 TEE 流程，以及实验性的 Kubernetes CRI 服务。

最重要的边界：**A3S Box 不是 Docker 或 Kubernetes 的完全替代品。它是一个能力边界明确的 MicroVM 运行时。**

## 当前状态

| 领域 | 当前真实状态 |
| --- | --- |
| CLI 运行时 | 当前核心产品面。目标主机是 macOS Apple Silicon/HVF 与 Linux/KVM 类环境。 |
| 镜像 | 已实现 pull、load、save、tag、inspect、history、remove、本地缓存解析、push 与 cosign 验证/签名路径。依赖 registry 的路径需要联网验证。 |
| Dockerfile 构建 | 支持明确子集。`RUN` 使用 Linux `chroot`，要求可 root 的 Linux；macOS 默认拒绝 `RUN`，除非显式开启 unsafe host run。 |
| 网络 | 支持默认 TSI、TCP 端口发布、自定义 bridge 网络和 peer `/etc/hosts` 发现；NAT 能力按平台不同。 |
| Compose | 支持常用本地开发子集，不等同完整 Docker Compose。 |
| TEE | 支持 SEV-SNP 方向的证明、RA-TLS、密封、密钥注入和模拟模式。硬件信任取决于目标主机。TDX 未产品化。 |
| Kubernetes CRI | 实验性。可用于评估与测试，不是当前核心完成目标。Windows CRI 不支持。 |
| Windows | 存在原生运行时代码路径，但当前文档和 smoke 覆盖聚焦 macOS/Linux。 |

## 心智模型

```text
Host
  a3s-box CLI
    images / boxes / volumes / networks / audit state
    VmManager
      |
      | shim process + libkrun
      v
MicroVM
  guest-init (PID 1)
  exec server / PTY server / attestation server
  OCI image workload
```

宿主机准备 rootfs 并启动 shim；shim 配置 libkrun；VM 内的 guest-init 启动控制服务并运行工作负载。

## 今天适合使用的能力

- 前台、后台、交互式运行标准 Linux OCI 镜像。
- `create` / `start` 持久化流程，包含命令、entrypoint、环境变量、工作目录、用户、hostname、hosts 配置。
- `exec`、PTY、attach、logs、wait、stats、inspect、pause/unpause、restart 与清理流程。
- 镜像 pull、load、save、tag、inspect、history、prune、remove，以及部分 push/signing 工作流。
- 卷、`cp`、`diff`、`export`、`commit` 和配置/文件系统级 snapshot。
- 支持平台上的 TCP 端口发布和 bridge peer 发现。
- 面向本地多服务开发的 Compose 子集。

## 当前边界

- 没有 Docker daemon API。
- 没有完整 Dockerfile/buildx 兼容；不支持 multi-platform image index。
- 不支持运行中网络热插拔；需要先停止 box。
- strict/custom 网络策略尚未执行；当前会拒绝，避免伪安全。
- macOS bridge 模式不支持 outbound NAT；需要外网访问时用默认 TSI。
- 不支持 Windows CRI。
- TDX 还不是产品化路径。
