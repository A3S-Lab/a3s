# 镜像与构建

A3S Box 使用 Linux OCI 镜像。本地镜像仓库由 `pull`、`run`、`build`、`load`、`save`、`tag` 和 `image-inspect` 共享。

## 引用与缓存

| 输入 | 含义 |
| --- | --- |
| `alpine` | Docker Hub `docker.io/library/alpine:latest`。 |
| `alpine:3.20` | Docker Hub `docker.io/library/alpine:3.20`。 |
| `ghcr.io/org/app:v1` | 显式 registry、repository 和 tag。 |
| `image@sha256:<digest>` | digest 引用。 |
| `sha256:<prefix>` | 本地唯一匹配时按 digest 前缀查找。 |

Docker Hub alias 在无歧义时会复用同一缓存镜像。

## Pull 和认证

```bash
a3s-box login ghcr.io -u USER -p TOKEN
a3s-box pull ghcr.io/org/app:v1
a3s-box logout ghcr.io
```

registry 操作需要网络；相关端到端测试不会混入纯单元测试。

## 签名验证

```bash
# cosign 公钥验证
a3s-box pull --verify-key cosign.pub ghcr.io/org/app:v1

# cosign keyless 验证
a3s-box pull \
  --verify-issuer https://accounts.google.com \
  --verify-identity user@example.com \
  ghcr.io/org/app:v1
```

keyless 验证需要 registry 与 Sigstore/Fulcio/Rekor 相关网络访问。

## Inspect、Save、Load、Tag

```bash
a3s-box images
a3s-box image-inspect alpine:latest
a3s-box history alpine:latest
a3s-box tag alpine:latest local/alpine:test
a3s-box save -o alpine.tar alpine:latest
a3s-box load -i alpine.tar --tag local/alpine:test
a3s-box rmi local/alpine:test
```

`image-inspect` 会展示 entrypoint、cmd、env、workdir、user、exposed ports、volumes、labels、stop signal、healthcheck 和 ONBUILD metadata。

## Dockerfile 构建子集

```bash
a3s-box build -t app:dev .
a3s-box build -t app:dev -f Containerfile .
a3s-box build -t app:dev --build-arg VERSION=1.2.3 --platform linux/amd64 .
```

支持的指令：`FROM`（含 `scratch`）、shell-form `RUN`、shell-form `COPY`/`ADD`、`WORKDIR`、`ENV`、`ENTRYPOINT`、`CMD`、`EXPOSE`、`LABEL`、`USER`、`ARG`、`SHELL`、`STOPSIGNAL`、`HEALTHCHECK`、`ONBUILD` metadata triggers、`VOLUME`。

边界：

- `RUN` 使用 Linux `chroot`，要求可 root 的 Linux。
- macOS 默认拒绝 `RUN`；`A3S_BOX_UNSAFE_HOST_RUN=1` 仅用于 unsafe 本地实验。
- `COPY --chown`、`ADD --chown` 等不支持 flag 会明确失败。
- `--platform` 只接受一个目标平台，不生成 multi-platform index。

## Push

```bash
a3s-box tag app:dev registry.example/org/app:dev
a3s-box push registry.example/org/app:dev
```

Push 需要可访问的 OCI registry。配置 `--sign-key` 时可走 push 后签名路径。
