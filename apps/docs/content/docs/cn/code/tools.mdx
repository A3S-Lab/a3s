## 文件操作

### `read`

读取带行号的文件内容，支持大文件的偏移和行数限制。

```json
{
  "file_path": "src/main.rs",
  "offset": 0,
  "limit": 100
}
```

### `write`

创建或覆盖文件，自动创建父目录。

```json
{
  "file_path": "src/config.rs",
  "content": "pub struct Config {\n    pub port: u16,\n}\n"
}
```

### `edit`

替换文件中的指定字符串，适合精准修改。

```json
{
  "file_path": "src/main.rs",
  "old_string": "fn main() {",
  "new_string": "fn main() -> Result<()> {"
}
```

### `patch`

应用统一 diff 补丁，适合复杂多行编辑。

```json
{
  "file_path": "src/lib.rs",
  "diff": "--- a/src/lib.rs\n+++ b/src/lib.rs\n@@ -1,3 +1,4 @@\n+use anyhow::Result;\n use std::io;\n"
}
```

## 搜索

### `grep`

使用 ripgrep 模式搜索文件内容，支持正则、glob 和上下文行数。

```json
{
  "pattern": "fn.*async",
  "path": "src/",
  "glob": "*.rs",
  "context": 2
}
```

### `glob`

查找匹配 glob 模式的文件。

```json
{
  "pattern": "src/**/*.rs"
}
```

### `ls`

列出目录内容及文件大小和类型。

```json
{
  "path": "src/"
}
```

## Shell

### `bash`

执行 Shell 命令，命令在会话工作区目录中运行。

```json
{
  "command": "cargo test --lib -- test_name"
}
```

**安全：** 使用[权限系统](/cn/docs/code/security)控制 bash 访问。

## 网络

### `web_fetch`

从 URL 获取内容，返回从 HTML 中提取的文本。

```json
{
  "url": "https://docs.rs/tokio/latest/tokio/"
}
```

### `web_search`

使用多个搜索引擎搜索网络。

```json
{
  "query": "rust async trait object",
  "engine": "google"
}
```

## 直接工具调用

不经过 LLM，直接在会话上调用工具：

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// 便捷封装
const content = await session.readFile('src/main.rs');
const output = await session.bash('cargo test --lib');
const files = await session.glob('**/*.rs');
const matches = await session.grep('fn main');

// 通用工具调用
const result = await session.tool('edit', {
  file_path: 'src/main.rs',
  old_string: 'fn main() {',
  new_string: 'fn main() -> Result<()> {',
});
console.log(`[${result.exitCode}] ${result.output}`);
```
</Tab>
<Tab value="Python">
```python
# 便捷封装
content = session.read_file("src/main.rs")
output = session.bash("cargo test --lib")
files = session.glob("**/*.rs")
matches = session.grep("fn main")

# 通用工具调用
result = session.tool("edit", {
    "file_path": "src/main.rs",
    "old_string": "fn main() {",
    "new_string": "fn main() -> Result<()> {"
})
print(f"[{result['exit_code']}] {result['output']}")
```
</Tab>

## Git

### `git_worktree`

管理 Git 工作树，实现并行工作区隔离。支持 `status`、`create`、`list`、`remove` 子命令。

```json
{
  "command": "create",
  "branch": "feature-auth",
  "path": "/repo/wt-feature-auth"
}
```

## 子智能体委托

### `task`

将工作委托给专门的子智能体，子智能体在独立会话中运行，拥有独立的对话历史。

```json
{
  "description": "为 auth 模块编写单元测试",
  "prompt": "为 src/auth.rs 创建覆盖所有边界情况的完整测试"
}
```

### `parallel_task`

并发将多个任务委托给子智能体，每个任务并行运行，拥有独立的智能体实例。

```json
{
  "tasks": [
    {
      "agent": "general",
      "prompt": "分析 auth 模块的安全性"
    },
    {
      "agent": "general",
      "prompt": "审查数据库层的性能"
    }
  ]
}
```

### `run_team`

协调 Lead → Worker → Reviewer 团队，处理需要分解、执行和质量审查的目标。

```json
{
  "goal": "审计 authentication 模块的安全问题",
  "lead_agent": "general",
  "worker_agent": "general",
  "reviewer_agent": "general",
  "max_steps": 10
}
```

## 沙箱路由（可选）

没有独立的 `sandbox` 工具。当会话配置了 `BashSandbox` 句柄时，`bash` 命令会透明地路由到 A3S Box MicroVM：

```rust
SessionOptions::new().with_sandbox_handle(Arc::new(my_box_impl))
```

工作区挂载在 VM 内的 `/workspace`。详见 [隔离 → MicroVM 沙箱](/cn/docs/code/isolation)。

## 文件版本历史

所有文件修改工具（`write`、`edit`、`patch`）在修改前自动捕获快照。你可以：

- 查看版本间的 diff
- 恢复任意历史版本
- 追踪会话中的所有修改

每个会话最多维护 500 个快照。

## 扩展自定义工具

除内置工具外，A3S Code 支持通过以下方式集成外部工具：

- **[MCP](/cn/docs/code/mcp)** — 通过模型上下文协议连接外部工具服务器（JSON-RPC 2.0, stdio + HTTP+SSE）
- **自定义 Tool Trait** — 在 Rust 中实现 `Tool` trait 并注册到 `ToolRegistry`

详见 [MCP](/cn/docs/code/mcp) 了解外部工具集成示例。
