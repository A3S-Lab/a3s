# Provider 配置

A3S Code 从 ACL 读取运行时配置。配置源可以是 `.acl` 文件路径，也可以是内联 ACL 字符串。JSON 和旧 HCL 配置不再支持。

## 基本结构

```text
default_model = "provider/model-id"
max_parallel_tasks = 8
auto_parallel = false

providers "provider" {
  apiKey = env("PROVIDER_API_KEY")
  baseUrl = env("PROVIDER_BASE_URL")

  models "model-id" {
    name = "Human readable model name"
    tool_call = true
  }
}

storage_backend = "file"
```

`apiKey` 与 `api_key` 都可用，`baseUrl` 与 `base_url` 都可用。密钥建议只通过环境变量注入，仓库里只提交模板或非敏感默认值。

## 委派控制

```text
agent_dirs = ["./.a3s/agents"]

auto_delegation {
  enabled        = true
  auto_parallel  = false
  min_confidence = 0.72
  max_tasks      = 4
}
```

`max_parallel_tasks` 限制有边界的 sibling fan-out。`auto_delegation.enabled`
开启自动 subagent 委派。顶层 `auto_parallel = false` 会覆盖
`auto_delegation.auto_parallel`，只关闭自动并行子 agent fan-out；手动
`parallel_task` 仍可用。

## Anthropic

```text
default_model = "anthropic/claude-sonnet-4-20250514"

providers "anthropic" {
  apiKey = env("ANTHROPIC_API_KEY")
}
```

## OpenAI 兼容服务

任何兼容 OpenAI Chat Completions 的服务都可以通过 `openai` provider 接入：

```text
default_model = "openai/gpt-4o"

providers "openai" {
  apiKey = env("A3S_OPENAI_API_KEY")
  baseUrl = env("A3S_OPENAI_BASE_URL")
}
```

MiniMax 也使用同一个 provider：

```text
default_model = "openai/MiniMax-M2.7-highspeed"

providers "openai" {
  apiKey = env("A3S_OPENAI_API_KEY")
  baseUrl = env("A3S_OPENAI_BASE_URL")

  models "MiniMax-M2.7-highspeed" {
    name = "MiniMax M2.7 Highspeed"
    tool_call = true
  }
}
```

当 `A3S_OPENAI_*` 未设置时，测试脚本会把 `MINIMAX_API_KEY` 与 `MINIMAX_BASE_URL` 映射过去。

## 存储

```text
storage_backend = "memory"
storage_backend = "file"
sessions_dir = ".a3s/sessions"
```

短测试使用 memory；通过 ACL 加载的可恢复本地 session 使用 `sessions_dir`。
`storage_url` 会被解析为自定义存储元数据，但它本身不会创建文件型 session
store。SDK 宿主也可以直接传 `sessionStore: new FileSessionStore(...)` /
`opts.session_store = FileSessionStore(...)`。

## 私有 Provider 检查

验证私有 provider 时，应使用临时 env-injected ACL。发布脚本设计上不会修改真实 `.a3s/config.acl`：它们会提取 provider 值，写入使用 `env(...)` 的临时 ACL，再把这些值注入当前进程环境。

```bash
A3S_CONFIG_FILE=/path/to/.a3s/config.acl \
  A3S_OPENAI_API_KEY="$A3S_OPENAI_API_KEY" \
  A3S_OPENAI_BASE_URL="$A3S_OPENAI_BASE_URL" \
  scripts/real_config_env_integration.sh
```

SDK 对齐由单独的真实 provider 检查脚本覆盖：

```bash
A3S_CONFIG_FILE=/path/to/.a3s/config.acl \
  scripts/sdk_real_config_env_integration.sh
```

把实际 provider 证据保存在发布记录里。完整本地 API 表面见 [API Contract](/cn/docs/code/api-contract)。
