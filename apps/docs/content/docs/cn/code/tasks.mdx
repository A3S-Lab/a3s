# 任务

A3S Code 提供统一的任务系统，支持 LLM 驱动的规划、目标追踪和并行执行。任务系统将复杂请求分解为结构化的执行计划。

<Callout type="info">
**核心功能：** LLM 规划器 | Wave 调度器 | 目标追踪 | 依赖图 | 并行执行
</Callout>

## 概述

当启用规划时，Agent 会：

<Steps>
<Step>**LlmPlanner** 将任务分解为带依赖关系的步骤</Step>
<Step>**WaveScheduler** 将独立步骤分组为 wave</Step>
<Step>每个 wave 通过 `tokio::JoinSet` 并行执行</Step>
<Step>依赖步骤等待其前置步骤完成</Step>
</Steps>

```
Wave 1: [分析认证模块] + [分析数据库 schema]  ← 并行
Wave 2: [实现 JWT 集成]                       ← 等待 wave 1
Wave 3: [编写测试]                             ← 等待 wave 2
```

## 目标追踪

启用目标追踪后，Agent 会维护一个任务列表并在流式事件中报告进度：

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const session = agent.session('/project', {
  planning: true,
  goalTracking: true,
});

const stream = await session.stream('Build a REST API with auth');
while (true) {
  const { value, done } = await stream.next();
  if (done || !value) break;
  if (value.type === 'task_updated') {
    for (const task of value.tasks) {
      const icon = { pending: '⏳', in_progress: '🔄', completed: '✅', failed: '❌' }[task.status];
      console.log(`${icon} ${task.description}`);
    }
  }
}
```
</Tab>
<Tab value="Python">
```python
session = agent.session("/project", planning=True, goal_tracking=True)

for event in session.stream("Build a REST API with auth"):
    if event.event_type == "task_updated":
        for task in event.tasks:
            icons = {"pending": "⏳", "in_progress": "🔄", "completed": "✅", "failed": "❌"}
            print(f"{icons[task.status]} {task.description}")
```
</Tab>
</Tabs>

## 执行计划结构

LlmPlanner 生成 JSON 结构化的执行计划：

```json
{
  "steps": [
    {
      "id": "step-1",
      "description": "Analyze current auth implementation",
      "dependencies": [],
      "tools": ["read", "grep", "glob"]
    },
    {
      "id": "step-2",
      "description": "Analyze database schema",
      "dependencies": [],
      "tools": ["read", "grep"]
    },
    {
      "id": "step-3",
      "description": "Implement JWT middleware",
      "dependencies": ["step-1", "step-2"],
      "tools": ["write", "edit"]
    },
    {
      "id": "step-4",
      "description": "Write integration tests",
      "dependencies": ["step-3"],
      "tools": ["write", "bash"]
    }
  ]
}
```

## Wave 调度

WaveScheduler 将独立步骤分组为 wave 并行执行：

```
依赖图:
  step-1 ──┐
            ├──→ step-3 ──→ step-4
  step-2 ──┘

Wave 分组:
  Wave 1: [step-1, step-2]  ← 无依赖，并行执行
  Wave 2: [step-3]          ← 依赖 step-1 和 step-2
  Wave 3: [step-4]          ← 依赖 step-3
```

每个 wave 中的步骤通过 `tokio::JoinSet` 并行执行。
