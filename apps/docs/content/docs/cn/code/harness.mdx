# Agent 驾驭协议（AHP）

**Agent Harness Protocol（AHP）** 是一种语言无关的协议，用于从外部进程监督 A3S Code 的 agent 会话。[生命周期 Hooks](/docs/cn/code/hooks) 允许你在进程内（Rust、Python、TypeScript）拦截事件；而 AHP 允许你用**任意语言**做同样的事——只需派生一个子进程，通过 stdio 上的换行分隔 JSON-RPC 2.0 进行通信。

AHP 采用与 MCP 相同的传输原则：A3S Code 宿主派生一个驾驭服务器进程，通过 stdin 发送事件，从 stdout 读取决策。驾驭服务器可用 Go、Java、Shell 脚本或任意其他语言实现。

## 架构

```
A3S Code 宿主
  └── AgentSession
        └── AhpHookExecutor（实现 HookExecutor trait）
              ├── pre_tool_use / pre_prompt  →  JSON-RPC 请求  →  等待响应
              └── 其他所有事件               →  JSON-RPC 通知  →  不等待

驾驭服务器（子进程，任意语言）
  ├── 从 stdin 读取换行分隔的 JSON
  └── 将 JSON-RPC 响应写入 stdout
```

驾驭服务器的 stderr 由宿主继承，所有服务器端日志输出将显示在宿主终端中。

## 协议

### 传输层

**当前实现：** 事件通过子进程的 stdin/stdout 以换行分隔的 JSON 发送。每条消息是一个以换行符（`\n`）结尾的 JSON 对象。

stdio 传输层是当前版本中唯一的内置传输层。此设计优先考虑：
- **简单性** — 无需网络配置、端口或身份验证
- **进程隔离** — 驾驭服务器作为独立的操作系统进程运行
- **语言无关** — 任何能读取 stdin 和写入 stdout 的语言都可以工作

**未来扩展性：** 协议本身与传输层无关。未来版本可能支持：
- HTTP/SSE（类似 MCP 的 HTTP 传输）
- WebSocket（用于远程驾驭服务器）
- Unix domain sockets（用于本地 IPC，开销更低）
- gRPC（用于高性能部署）

JSON-RPC 2.0 消息格式保持不变——只有传输层会改变。

### 请求（阻塞事件）

宿主在需要执行前获取决策时发送此消息：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "harness/event",
  "params": {
    "event_type": "pre_tool_use",
    "payload": {
      "session_id": "s1",
      "tool": "Bash",
      "args": { "command": "ls -la" },
      "working_directory": "/workspace",
      "recent_tools": []
    },
    "meta": { "depth": 0 }
  }
}
```

驾驭服务器必须在默认超时（10 秒）内返回 JSON-RPC 结果：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "action": "continue",
    "reason": null,
    "modified": null,
    "retry_delay_ms": null
  }
}
```

### 通知（即发即忘事件）

宿主为不需要决策的可观测事件发送此消息：

```json
{
  "jsonrpc": "2.0",
  "method": "harness/event",
  "params": {
    "event_type": "post_tool_use",
    "payload": { "session_id": "s1", "tool": "Bash", ... },
    "meta": { "depth": 0 }
  }
}
```

通知不需要也不会收到任何响应。

## 编写驾驭服务器

驾驭服务器是任意能从 stdin 读取换行分隔 JSON 并将 JSON-RPC 响应写入 stdout 的进程。所有日志必须写入 stderr，以免干扰协议通信。

### 参考实现

仓库提供了多种语言的参考服务器：

**Python:** `core/examples/ahp_server.py`
- 从每条消息中解析 `event_type` 和 `payload`
- 读取 `meta.depth` 以实现深度感知决策
- 通过正则表达式阻止危险的 Bash 命令
- 在 `post_tool_use`（即发即忘）中检测敏感输出模式
- 将所有日志写入 stderr

**TypeScript:** `core/examples/ahp_server.ts`
- 完整的 TypeScript 实现，具有类型安全
- 深度感知策略执行（对子智能体应用更严格的规则）
- 基于模式的命令阻断
- 工具输出中的敏感数据检测
- 全面的事件处理

### Python 参考服务器

核心结构：

```python
import json, sys

def on_pre_tool_use(payload, depth):
    tool = payload.get("tool", "")
    command = payload.get("args", {}).get("command", "")
    # 深度感知：对子智能体应用更严格的策略
    if depth > 0 and tool == "Bash" and is_dangerous(command):
        return {"action": "block", "reason": f"在深度 {depth} 处被阻断"}
    return {"action": "continue"}

for raw_line in sys.stdin:
    msg = json.loads(raw_line.strip())
    params = msg.get("params", {})
    event_type = params.get("event_type", "")
    payload    = params.get("payload", {})
    depth      = params.get("meta", {}).get("depth", 0)
    req_id     = msg.get("id")          # 通知时为 None

    if req_id is None:
        dispatch_notification(event_type, payload, depth)
        continue

    result = dispatch_request(event_type, payload, depth) or {"action": "continue"}
    print(json.dumps({"jsonrpc": "2.0", "id": req_id, "result": result}), flush=True)
```


### TypeScript 参考服务器

完整实现位于 `core/examples/ahp_server.ts`。核心模式：

```typescript
import * as readline from 'readline';

interface AhpResponse {
  action: 'continue' | 'block' | 'skip' | 'retry';
  reason?: string;
}

function handlePreToolUse(payload: any, depth: number): AhpResponse {
  const tool = payload.tool || '';
  const command = payload.args?.command || '';

  // 深度感知：对子智能体应用更严格的策略
  if (tool === 'Bash' && isDangerous(command)) {
    const reason = `在深度 ${depth} 处阻断危险命令`;
    console.error(`[BLOCK] ${reason}`);
    return { action: 'block', reason };
  }

  return { action: 'continue' };
}

// 主循环
const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
  terminal: false,
});

rl.on('line', (line: string) => {
  const msg = JSON.parse(line);
  const { event_type, payload, meta } = msg.params;
  const depth = meta?.depth ?? 0;
  const reqId = msg.id;

  if (reqId === undefined) {
    handleNotification(event_type, payload, depth);
  } else {
    const result = handleRequest(event_type, payload, depth);
    console.log(JSON.stringify({ jsonrpc: '2.0', id: reqId, result }));
  }
});
```

### 最简 Shell 驾驭服务器

允许一切操作的 Shell 脚本（可作为起点使用）：

```bash
#!/usr/bin/env bash
# harness.sh — 全放行 AHP 服务器
while IFS= read -r line; do
    id=$(echo "$line" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('id',''))")
    if [ -n "$id" ]; then
        echo "{"jsonrpc":"2.0","id":$id,"result":{"action":"continue"}}"
    fi
done
```

## 子智能体深度传播

当挂载了驾驭服务器的 session 通过 `task` 工具派生子智能体时，`AhpHookExecutor` 会自动传播——同一个驾驭服务器进程接收来自所有子智能体的事件。`meta.depth` 字段随嵌套层级递增，使服务器能对更深层的 agent 应用更严格的策略。

```
父 session（depth: 0）
  └── task 工具派生子智能体（depth: 1）
        └── task 工具派生孙子智能体（depth: 2）
              └── ...
```

所有层级的事件都被复用到同一个 stdin/stdout 流中。驾驭服务器可通过 `session_id`（每个 agent 唯一）和 `depth` 区分事件来源。

<Callout type="info">
  AHP 深度传播使用 `ToolContext.hook_executor` 机制，`AhpHookExecutor` 在其中注册并在每个嵌套层级继承。
</Callout>
