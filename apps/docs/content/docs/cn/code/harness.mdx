# 智能体驾驭协议 (AHP) v2.0

**智能体驾驭协议（Agent Harness Protocol, AHP）** 是一个通用的、框架无关的协议，用于监督、控制和审计自主AI智能体。与传统的与特定框架紧密耦合的监控方案不同，AHP提供了一个适用于任何智能体系统的通用接口。

<Callout type="info">
  **v2.0 新特性：** 传输层无关设计、能力协商、双向查询、批处理支持和增强的安全模型。查看[迁移指南](#从-v1x-迁移)了解如何从 v1.x 升级。
</Callout>

## 概述

### 什么是"驾驭层"（Harness）？

在AI智能体的语境中，**驾驭层**是一个外部监督层，它可以：

- **拦截**智能体在执行前的动作（执行前钩子）
- **观察**智能体的行为和输出（执行后钩子）
- **执行**策略和约束（阻断、限流、资源配额）
- **审计**所有智能体活动，用于合规和调试
- **修改**智能体行为（提示词注入、参数调优）

驾驭层位于智能体和其环境**之间**，充当一个透明代理，可以检查、修改或阻止任何交互。

## 核心特性

- **框架无关** — 适用于任何智能体系统（编码智能体、研究智能体、机器人等）
- **语言中立** — 驾驭服务器可以用任何语言编写（Python、Go、Rust、Java等）
- **传输灵活** — 支持 stdio、HTTP、WebSocket、gRPC、Unix socket 和自定义传输
- **双向通信** — 智能体可以向驾驭层查询指导，而不仅仅是接收命令
- **有状态** — 驾驭层可以维护会话状态、跟踪历史并做出上下文感知的决策
- **可扩展** — 可以添加新的事件类型、动作和元数据，而不破坏兼容性
- **安全** — 内置认证、加密和沙箱支持
- **可观测** — 完整的追踪、指标和日志记录

use a3s_code_core::{Agent, SessionOptions};
use a3s_ahp::{AhpClient, Transport};

// stdio 传输（本地驾驭）
let ahp = AhpClient::new(Transport::Stdio {

        SessionOptions::new().with_ahp_client(ahp)

## 编写驾驭服务器

驾驭服务器是任何实现AHP协议的进程。这是一个最小的Python示例：

```python
import json, sys

def handle_event(event):
    event_type = event["params"]["event_type"]
    payload = event["params"]["payload"]

    if event_type == "pre_action":
        # 应用策略
        command = payload.get("arguments", {}).get("command", "")
        if "rm -rf" in command:
            return {"decision": "block", "reason": "危险命令"}
        return {"decision": "allow"}

    # 默认：允许
    return {"decision": "allow"}

# 主循环（stdio 传输）
for line in sys.stdin:
    event = json.loads(line)
    req_id = event.get("id")

    if req_id:  # 请求（阻塞）
        result = handle_event(event)
        response = {"jsonrpc": "2.0", "id": req_id, "result": result}
        print(json.dumps(response), flush=True)
    else:  # 通知（即发即弃）
        handle_event(event)
```
