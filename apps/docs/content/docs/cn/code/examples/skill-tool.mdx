# Skill Tool - 权限隔离

**Skill 工具**（v1.3.4 引入）允许将 skills 作为一等工具调用，并在执行期间临时授予权限。这强制执行基于 skill 的访问模式，防止 agent 绕过 skills 直接访问底层工具。

<Callout type="info">
**GitHub Issue:** [#8 - Skill Tool 机制用于强制基于 skill 的工具访问](https://github.com/A3S-Lab/Code/issues/8)
</Callout>

## 概述

传统的 skills 将指令注入到系统提示中，但 agent 仍然可以绕过它们直接调用工具。Skill 工具通过以下方式解决了这个问题：

1. **使 skills 可调用** — LLM 可以将 `Skill("skill-name")` 作为工具调用
2. **临时权限授予** — 在执行期间授予 skill 的 `allowed-tools` 权限
3. **自动撤销** — 执行后撤销权限（RAII 模式）
4. **权限隔离** — 父 agent 无法绕过 skills 访问工具

## 工作原理

```
┌─────────────────────────────────────────────────────────────┐
│ 用户提示: "使用 data-processor 读取 README.md"              │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ LLM (Kimi/Claude/GPT-4)                                     │
│ 调用: Skill("data-processor", prompt="Read README.md")     │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ SkillTool.execute()                                         │
│ 1. 从 SkillRegistry 获取 skill                             │
│ 2. 从 allowed-tools 创建 PermissionPolicy                  │
│ 3. 使用 skill 权限创建新的 AgentLoop                       │
│ 4. 使用临时权限执行 skill                                  │
│ 5. 返回结果                                                 │
│ 6. 权限自动撤销（RAII）                                    │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 结果返回给 LLM                                              │
└─────────────────────────────────────────────────────────────┘
```

## 基本用法

### 步骤 1：定义 Skill

创建带有 `allowed-tools` 的 skill 文件：

```markdown title="skills/data-processor.md"
---
name: data-processor
description: 处理和分析数据文件
allowed-tools: read(*), grep(*)
---

# Data Processor Skill

你是一个数据处理专家。你可以：
- 使用 read 工具读取文件
- 使用 grep 搜索模式
- 分析和总结文件内容

你不能：
- 写入文件
- 执行 bash 命令
- 编辑文件
```

## 权限隔离

Skill 工具的关键特性是**权限隔离** — 父 agent 只能通过批准的 skills 访问工具。

### 示例：受限 Agent

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
import { Agent, SessionOptions, PermissionPolicy, PermissionRule } from '@a3s-lab/code';

const agent = await Agent.create('agent.hcl');

// Agent 只能使用 Skill 工具，不能直接使用 read/grep
const session = agent.session('.', {
  skillsDir: './skills',
  permissionPolicy: new PermissionPolicy({
    allow: [new PermissionRule('Skill(*)')],  // 只允许 Skill 工具
    deny: [
      new PermissionRule('read(*)'),          // 拒绝直接 read
      new PermissionRule('grep(*)')           // 拒绝直接 grep
    ],
    defaultDecision: 'deny'
  })
});

// ✅ 这个可以 - agent 调用 Skill 工具
const result1 = await session.send(
  '使用 data-processor 读取 file.txt'
);

// ❌ 这个失败 - agent 尝试直接调用 read
const result2 = await session.send(
  '读取 file.txt'
);
// 错误: 工具 'read' 被权限策略阻止
```
</Tab>
<Tab value="Python">
```python
from a3s_code import Agent, SessionOptions, PermissionPolicy, PermissionRule

agent = Agent.create("agent.hcl")

# Agent 只能使用 Skill 工具，不能直接使用 read/grep
session = agent.session(".", SessionOptions(
    skills_dir="./skills",
    permission_policy=PermissionPolicy(
        allow=[PermissionRule("Skill(*)")],  # 只允许 Skill 工具
        deny=[
            PermissionRule("read(*)"),        # 拒绝直接 read
            PermissionRule("grep(*)")         # 拒绝直接 grep
        ],
        default_decision="deny"
    )
))

# ✅ 这个可以 - agent 调用 Skill 工具
result1 = session.send("使用 data-processor 读取 file.txt")

# ❌ 这个失败 - agent 尝试直接调用 read
result2 = session.send("读取 file.txt")
# 错误: 工具 'read' 被权限策略阻止
```
</Tab>
</Tabs>

## 使用场景

### 1. 受限环境

允许 agent 仅通过批准的 skills 使用强大工具：

```yaml
# Skill: safe-executor
allowed-tools: bash(npm test), bash(cargo test)
```

Agent 只能通过 skill 运行批准的 bash 命令。

### 2. 审计追踪

跟踪调用了哪些 skills 以及它们使用了哪些工具：

```typescript
// Hook 到 skill 调用
session.on('tool_start', (event) => {
  if (event.name === 'Skill') {
    console.log(`Skill 被调用: ${event.args.skill_name}`);
  }
});
```

### 3. Skill 组合

通过链式调用 skill 构建复杂工作流：

```markdown
# Skill: data-pipeline
allowed-tools: Skill(data-processor), Skill(data-validator)
```

一个 skill 可以调用其他 skills。

### 4. 安全边界

在 skill 级别强制执行安全策略：

```yaml
# Skill: production-deployer
allowed-tools: bash(kubectl apply -f *), bash(helm upgrade *)
```

只允许特定的部署命令。

## 真实示例

### 场景：数据分析 Agent

你想要一个可以分析数据文件但不能修改它们或执行任意命令的 agent。

**步骤 1：创建 Skills**

```markdown title="skills/data-reader.md"

name: data-reader
description: 读取和解析数据文件
allowed-tools: read(*), grep(*)
---

# Data Reader

读取和解析数据文件。你可以：
- 读取 CSV、JSON 和文本文件
- 搜索模式
- 提取特定数据
```

```markdown title="skills/data-analyzer.md"
# Data Analyzer

分析数据文件并生成洞察。你可以：
- 使用 data-reader 加载数据
- 计算统计数据
```

## 最佳实践

### 1. 最小权限原则

只授予 skills 所需的最小工具：

```yaml
# ❌ 过于宽松
allowed-tools: "*"

# ✅ 只允许特定工具
allowed-tools: "read(*), grep(*)"
```

### 2. Skill 组合

将复杂任务分解为专注的 skills：

```yaml
# ❌ 一个 skill 做所有事情
allowed-tools: "read(*), write(*), bash(*), grep(*)"

# ✅ 多个专注的 skills
# Skill 1: data-reader (read, grep)
# Skill 2: data-writer (write)
# Skill 3: data-processor (Skill(data-reader), Skill(data-writer))
```

### 3. 清晰的描述

编写清晰的 skill 描述，让 LLM 知道何时使用它们：

```markdown
# ❌ 模糊
description: 处理数据

# ✅ 具体
description: 读取和分析 CSV/JSON 数据文件，提取统计数据，识别趋势
```

### 4. 权限策略

将 Skill 工具与权限策略结合实现纵深防御：

```typescript
// 第 1 层：权限策略（只允许 Skill 工具）
permissionPolicy: new PermissionPolicy({
  allow: [new PermissionRule('Skill(*)')],
  defaultDecision: 'deny'
})

// 第 2 层：Skill allowed-tools（只允许 read/grep）
allowed-tools: "read(*), grep(*)"
```

## 故障排除

### Skill 未找到

**错误：** `Skill 'data-processor' not found`

**解决方案：** 确保 skill 已加载：

```typescript
const session = agent.session('.', {
  skillsDir: './skills'  // 包含 skill 文件的目录
});
```

### 权限被拒绝

**错误：** `Tool 'read' is blocked by permission policy`

**解决方案：** 检查 skill 的 `allowed-tools`：

```yaml
# Skill 必须允许该工具
allowed-tools: "read(*), grep(*)"
```

### 嵌套 Skill 调用

**问题：** 一个 skill 可以调用另一个 skill 吗？

**答案：** 可以！在 `allowed-tools` 中使用 `Skill(*)`：

```yaml
# Skill: orchestrator
allowed-tools: "Skill(data-reader), Skill(data-writer)"
```
