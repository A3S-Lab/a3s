# Agentic Parse

`agentic_parse` 用于在结构化文档内容上做总结、分析和问答。富文档格式会统一走共享的
`CompositeDocumentParser` 流水线，因此 PDF、Office、HTML、EPUB、图片和邮件都可以落到同一套
面向 block 的元数据模型上。

## Runtime Metadata

当实际触发 OCR 时，`agentic_parse` 会暴露运行态元数据，例如 OCR provider、model、
`dpi` 以及归一化后的 parser 配置。

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const { parseDocumentRuntime } = require('@a3s-lab/code')

const tool = await session.tool('agentic_parse', { path: 'docs/scanned.pdf' })
console.log(tool.documentRuntime)

const runtime = parseDocumentRuntime(tool)
if (runtime?.ocr) {
  console.log(runtime.ocr.provider, runtime.ocr.model, runtime.ocr.dpi)
}
```

**源码:** [`crates/code/sdk/node/examples/test-document-ocr-provider.js`](https://github.com/A3S-Lab/Code/blob/main/crates/code/sdk/node/examples/test-document-ocr-provider.js)
</Tab>
<Tab value="Python">
```python
tool = session.tool("agentic_parse", {"path": "docs/scanned.pdf"})
print(tool.document_runtime)

runtime = tool.document_runtime_info
if runtime and runtime.ocr:
    print(runtime.ocr.provider, runtime.ocr.model, runtime.ocr.dpi)
```

**源码:** [`crates/code/sdk/python/examples/test_document_runtime_info.py`](https://github.com/A3S-Lab/Code/blob/main/crates/code/sdk/python/examples/test_document_runtime_info.py)
</Tab>
</Tabs>

## LLM Block Selection

当 `agentic_parse` 携带 `query` 运行时，tool metadata 会记录实际被选入 LLM 输入的
结构块。这样既能调试选块逻辑，也能在 SDK helper 中保留页码 / 章节 locator。

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const { parseAgenticParseLlmBlocks } = require('@a3s-lab/code')

const tool = await session.tool('agentic_parse', {
  path: 'docs/scanned.pdf',
  query: 'overview',
})

for (const block of tool.agenticParseLlmBlocks ?? parseAgenticParseLlmBlocks(tool) ?? []) {
  console.log(block.index, block.kind, block.label, block.location?.display)
}
```

**源码:** [`crates/code/sdk/node/examples/test-agentic-parse-llm-blocks.js`](https://github.com/A3S-Lab/Code/blob/main/crates/code/sdk/node/examples/test-agentic-parse-llm-blocks.js)
</Tab>
<Tab value="Python">
```python
tool = session.tool(
    "agentic_parse",
    {"path": "docs/scanned.pdf", "query": "overview"},
)

for block in tool.agentic_parse_llm_blocks_info:
    location = block.location.display if block.location else None
    print(block.index, block.kind, block.label, location)
```

**源码:** [`crates/code/sdk/python/examples/test_document_runtime_info.py`](https://github.com/A3S-Lab/Code/blob/main/crates/code/sdk/python/examples/test_document_runtime_info.py)
</Tab>
</Tabs>

<Callout type="info">
`agentic_search` 和 `agentic_parse` 共享同一个 parser registry。搜索结果里的
match locator 与解析结果里的 LLM block locator 都来自同一份 `CompositeDocumentParser`
block 图。
</Callout>

## OCR Provider 注入

如果宿主应用需要自定义 OCR 后端，可以在创建 session 时注入 provider。注入的
provider 优先级高于内置的本地工具 fallback。

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const session = agent.session('.', {
  documentOcrProvider: {
    name: 'js-mock-ocr',
    formats: ['pdf', 'image'],
    model: 'openai/gpt-4.1-mini',
    async run(request) {
      console.log(request.path, request.format)
      return 'Scanned invoice total: 42 USD'
    },
  },
})
```

**源码:** [`crates/code/sdk/node/examples/test-document-ocr-provider.js`](https://github.com/A3S-Lab/Code/blob/main/crates/code/sdk/node/examples/test-document-ocr-provider.js)
</Tab>
<Tab value="Python">
```python
opts.document_ocr_provider = DocumentOcrProvider(
    "python-mock-ocr",
    ocr_callback,
    formats=["pdf", "image"],
    model="openai/gpt-4.1-mini",
)
session = agent.session(".", opts)
```

**源码:** [`crates/code/sdk/python/examples/test_document_ocr_provider.py`](https://github.com/A3S-Lab/Code/blob/main/crates/code/sdk/python/examples/test_document_ocr_provider.py)
</Tab>
</Tabs>
