# 安全

A3S Code 提供两个内置安全层：**权限策略** 和 **人机协同（HITL）确认**。可通过 `SecurityProvider` trait 和 `HookEngine` 实现额外的安全逻辑。

## 安全架构

```
用户提示 → Agent 循环
  ├─ 权限策略          ← Deny → Allow → Ask → Default
  ├─ HITL 确认         ← 独立于权限系统
  ├─ 工具执行
  └─ SecurityProvider  ← 通过 trait 可插拔（污点追踪、清洗等）
```

## 权限策略

权限系统控制 agent 可以使用哪些工具。规则按以下顺序评估：**Deny → Allow → Ask → Default**。

### 评估顺序

规则按以下顺序评估：

<Steps>
<Step>**Deny 规则** — 如果任何 deny 规则匹配，立即阻止</Step>
<Step>**Allow 规则** — 如果任何 allow 规则匹配，允许执行</Step>
<Step>**Ask 规则** — 如果任何 ask 规则匹配，触发 HITL 确认</Step>
<Step>**默认策略** — 如果没有规则匹配，使用默认策略（permissive 或 strict）</Step>
</Steps>

## 人机协同（HITL）

HITL 确认**独立于**权限系统。即使工具被权限 `allow`，HITL 仍然可以要求用户确认。

### 配置

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const session = agent.session('/project', {
  confirmationPolicy: {
    enabled: true,
    timeoutMs: 30000,
    timeoutAction: 'reject',
  },
});
```
</Tab>
<Tab value="Python">
```python
from a3s_code import ConfirmationPolicy

session = agent.session("/project",
    confirmation_policy=ConfirmationPolicy(
        enabled=True,
        timeout_ms=30000,
        timeout_action="reject",
    ))
```
</Tab>
</Tabs>

### 处理确认事件

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const stream = await session.stream('Clean up old logs');
while (true) {
  const { value, done } = await stream.next();
  if (done || !value) break;
  if (value.type === 'confirmation_required') {
    console.log(`🔒 Approve '${value.toolName}'?`, value.toolArgs);
    await session.respondConfirmation(value.toolId, { approved: true });
  } else if (value.type === 'permission_denied') {
    console.log(`🚫 ${value.toolName} blocked: ${value.reason}`);
  } else if (value.type === 'text_delta') {
    process.stdout.write(value.text);
  }
}
```
</Tab>
<Tab value="Python">
```python
for event in session.stream("Clean up old logs"):
    if event.event_type == "confirmation_required":
        print(f"🔒 Approve '{event.tool_name}'? {event.tool_args}")
        session.respond_confirmation(event.tool_id, approved=True)
    elif event.event_type == "permission_denied":
        print(f"🚫 {event.tool_name} blocked: {event.reason}")
    elif event.event_type == "text_delta":
        print(event.text, end="", flush=True)
```
</Tab>
</Tabs>

## 工作区边界

所有文件操作限制在会话的工作区目录内。尝试访问工作区外的文件会被阻止。

## 内置安全提供者

A3S Code 默认不启用任何 `SecurityProvider`。调用 `with_default_security()` 即可激活内置的 `DefaultSecurityProvider`，无需任何额外配置。

### 启用方式

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
import { DefaultSecurityProvider } from '@a3s-lab/code';

// 默认安全配置
const session = agent.session('/project', {
  securityProvider: new DefaultSecurityProvider(),
});

// 自定义配置
const session = agent.session('/project', {
  securityConfig: {
    enableTaintTracking: true,
    enableOutputSanitization: true,
    enableInjectionDetection: true,
    customPatterns: [
      { name: 'internal_id', pattern: 'INT-\\d{6}', label: 'REDACTED:ID' },
    ],
  },
});
```
</Tab>
<Tab value="Python">
```python
from a3s_code import DefaultSecurityProvider

# 默认安全配置
session = agent.session("/project",
    security_provider=DefaultSecurityProvider())

# 自定义配置
session = agent.session("/project", security_config={
    "enable_taint_tracking": True,
    "enable_output_sanitization": True,
    "enable_injection_detection": True,
    "custom_patterns": [
        {"name": "internal_id", "pattern": r"INT-\d{6}", "label": "REDACTED:ID"},
    ],
})
```

### 提示注入检测

`DefaultSecurityProvider` 在 `GenerateStart` hook 中对用户输入进行注入检测，识别以下模式：

| 模式 | 示例 |
|------|------|
| 忽略先前指令 | "Ignore all previous instructions..." |
| 无视上下文 | "Disregard all prior context..." |
| 模式切换 | "You are now in developer mode..." |
| 遗忘指令 | "Forget everything you learned..." |
| 新指令注入 | "New instructions: ..." |
| 系统提示覆盖 | "System prompt override" |

检测到注入时，通过 `OnError` hook 上报，并在 agent 事件流中发出 `SecurityAlert`。污点追踪同时扫描工具输出，防止间接注入（通过读取被污染文件或网页内容）。

## 最佳实践

<Steps>
<Step>**从严格策略开始** — 使用 `PermissionPolicy::strict()` 并仅显式允许需要的工具</Step>
<Step>**阻止危险模式** — 始终拒绝破坏性 bash 命令：`bash(rm -rf:*)`、`bash(curl:*|sh)`</Step>
<Step>**为写操作启用 HITL** — 要求确认文件修改和 bash 执行</Step>
<Step>**使用工作区边界** — 永远不要禁用工作区限制</Step>
<Step>**实现 SecurityProvider** — 为敏感数据添加自定义污点追踪和输出清洗</Step>
<Step>**审计日志** — 使用 hooks 记录所有工具执行以供安全审查</Step>
</Steps>
