# Agent Orchestrator

Agent Orchestrator 是一个主子智能体协调器，基于 a3s-event 实现统一的事件总线，支持实时监控所有子智能体的行为、规划和执行，并提供动态控制能力。

<Callout type="info">
  **版本要求**: v1.1.0+
</Callout>

## 核心特性

- **真实 AgentLoop 执行** - 通过 `create_with_agent()` 让 SubAgent 运行真实 LLM + 工具调用
- **实时监控** - 监控所有 SubAgent 的生命周期、状态变更、进度和工具执行
- **任务列表查询** - 实时查看所有 SubAgent 的任务列表和当前活动
- **活动跟踪** - 追踪每个 SubAgent 的当前活动（调用工具、请求 LLM、等待控制）
- **动态控制** - 暂停、恢复、取消 SubAgent，调整参数，注入新提示词
- **事件驱动** - 基于 a3s-event 的统一事件总线
- **可插拔通讯** - 默认内存事件通讯，支持自定义 NATS provider
- **并发管理** - 控制最大并发 SubAgent 数量

## 架构

```text
AgentOrchestrator (主智能体)
    ↓ 订阅事件
┌───────────────────────────────┐
│   a3s-event EventBus          │
│ (Memory / NATS / Custom)      │
└───────────────────────────────┘
    ↑ 发布事件    ↓ 控制信号
SubAgentWrapper (子智能体)
```

## 快速开始

<Tabs items={['TypeScript', 'Python']}>
<Tab value="TypeScript">

```typescript
import { Orchestrator, SubAgentConfig } from '@a3s-lab/code';

// 创建 orchestrator
const orch = Orchestrator.create();

// 创建 SubAgent 配置
const config: SubAgentConfig = {
  agentType: 'general',
  prompt: '使用 glob 查找 TypeScript 文件',
  description: '查找 TypeScript 文件',
  permissive: true,
  maxSteps: 10,
};

// 启动 SubAgent
const handle = orch.spawnSubagent(config);
console.log(`SubAgent ID: ${handle.id}`);

// 监控状态
for (let i = 0; i < 10; i++) {
  await new Promise(resolve => setTimeout(resolve, 500));
  const state = handle.state();
  const active = orch.activeCount();
  console.log(`[${i * 0.5}s] Active: ${active}, State: ${state}`);

  // 控制操作
  if (i === 3) {
    handle.pause();
    console.log('已暂停');
  }
  if (i === 5) {
    handle.resume();
    console.log('已恢复');
  }
}

// 等待完成
const result = await handle.wait();
console.log(`结果: ${result}`);
```

## 真实执行

`Orchestrator.create()` 默认使用内部占位符——仅用于测试和演示。要运行**真实的 LLM + 工具调用**，用 `AgentOrchestrator::from_agent(agent)`（Rust）或 `create_with_agent()`（Python/TypeScript）挂载真实的 AgentLoop。

<Tabs items={['TypeScript', 'Python']}>
<Tab value="TypeScript">

```typescript
import { Agent, Orchestrator } from '@a3s-lab/code';

const agent = await Agent.create('agent.hcl');

// 将 orchestrator 连接到真实 AgentLoop 执行
const orch = Orchestrator.createWithAgent(agent, process.cwd());

const handle = orch.spawnSubagent({
  agentType: 'general',
  prompt: '使用 glob 工具查找所有 TypeScript 文件，并报告找到了多少个。',
  permissive: true,
  maxSteps: 5,
});

// 等待所有完成
orch.waitAll();
```

result = handle.wait()
print(result)

### 执行模式对比

| 模式 | API | 子智能体运行方式 |
|------|-----|----------------|
| 占位符（默认） | `Orchestrator.create()` | 内部 stub，用于测试和演示 |
| 真实执行 |  · Python/TS: `Orchestrator.create_with_agent(agent, ws)` | 完整 AgentLoop，含真实 LLM + 工具 |

### 真实执行架构

```text
AgentOrchestrator::from_agent(agent)
    └── SubAgentWrapper  （已挂载 Agent）
            ├── AgentRegistry   — 查找 agent 类型定义
            ├── ToolExecutor    — 内置工具（不含 Task 工具，防止循环嵌套）
            └── AgentLoop       — 真实 LLM 轮次 + 工具执行
                    │ 事件
                    ▼
        OrchestratorEvent::SubAgentInternalEvent { id, event }
```

<Callout type="info">
- **超时**：`timeout_ms` 超时后自动取消 AgentLoop 任务并返回错误
- **暂停**：标记 `Paused` 并停止接受控制信号，当前 LLM 轮次完成后才停止；`cancel()` 立即中止
- **`duration_ms`**：报告真实耗时，非硬编码零
</Callout>

## 事件类型

| 事件 | 描述 |
|------|------|
| `SubAgentStarted` | SubAgent 启动 |
| `SubAgentCompleted` | SubAgent 完成 |
| `SubAgentStateChanged` | 状态变更 |
| `SubAgentProgress` | 进度更新 |
| `SubAgentInternalEvent` | 内部事件（来自 AgentLoop） |
| `PlanningStarted` | 规划开始 |
| `PlanningCompleted` | 规划完成 |
| `ToolExecutionStarted` | 工具执行开始 |
| `ToolExecutionCompleted` | 工具执行完成 |
| `ControlSignalReceived` | 控制信号接收 |
| `ControlSignalApplied` | 控制信号应用 |
| `ExternalTaskPending` | 工具调用等待外部 worker 处理 |
| `ExternalTaskCompleted` | 外部任务已完成（成功或失败） |

## 控制信号

| 方法 | 说明 |
|------|------|
| `handle.pause().await?` | 暂停执行，可稍后恢复 |
| `handle.resume().await?` | 恢复已暂停的 SubAgent |
| `handle.cancel().await?` | 立即终止，不可恢复 |
| `handle.adjust_params(Some(20), Some(60000)).await?` | 动态调整最大步数和超时 |
| `handle.inject_prompt("...").await?` | 向运行中的 SubAgent 注入新提示 |

Python/TypeScript 使用相同方法名，去掉 `.await?`。

## 实时监控 API

<Tabs items={['TypeScript', 'Python']}>
<Tab value="TypeScript">

```typescript
// 获取所有 SubAgent 信息列表
const subagents = orch.listSubagents();
for (const info of subagents) {
  console.log(`SubAgent: ${info.id}`);
  console.log(`  Type: ${info.agentType}`);
  console.log(`  State: ${info.state}`);
  console.log(`  Description: ${info.description}`);

  if (info.currentActivity) {
    console.log(`  Current Activity: ${info.currentActivity.activityType}`);
  }
}

// 获取特定 SubAgent 的详细信息
const info = orch.getSubagentInfo('subagent-1');
if (info) {
  console.log(`Created at: ${info.createdAt}`);
  console.log(`Updated at: ${info.updatedAt}`);
}

// 获取所有活跃 SubAgent 的当前活动
const activities = orch.getActiveActivities();
for (const entry of activities) {
  if (entry.activity.activityType === 'calling_tool') {
    console.log(`${entry.id} is calling a tool`);
  } else if (entry.activity.activityType === 'requesting_llm') {
    console.log(`${entry.id} is requesting LLM`);
  }
}

// 获取所有 SubAgent 状态
const states = orch.getAllStates();
for (const entry of states) {
  console.log(`${entry.id}: ${entry.state}`);
}

// 获取活跃数量
const count = orch.activeCount();
console.log(`Active SubAgents: ${count}`);
```

### SubAgent 活动类型

SubAgent 在执行过程中会有以下活动类型：

| 活动类型 | 描述 | 数据 |
|---------|------|------|
| `Idle` | 空闲状态 | - |
| `CallingTool` | 正在调用工具 | 工具名称和参数 |
| `RequestingLlm` | 正在请求 LLM | 消息数量 |
| `WaitingForControl` | 等待控制信号 | 等待原因 |

### 监控 API 列表

| API | 描述 |
|-----|------|
| `list_subagents()` | 获取所有 SubAgent 信息列表 |
| `get_subagent_info(id)` | 获取特定 SubAgent 详细信息 |
| `get_active_activities()` | 获取所有活跃 SubAgent 的当前活动 |
| `get_all_states()` | 获取所有 SubAgent 状态 |
| `active_count()` | 获取活跃 SubAgent 数量 |
| `pause_subagent(id)` | 暂停指定 SubAgent |
| `resume_subagent(id)` | 恢复指定 SubAgent |
| `cancel_subagent(id)` | 取消指定 SubAgent |
| `wait_all()` | 等待所有 SubAgent 完成 |

## 最佳实践

1. **生产环境使用 `create_with_agent`** - 仅使用 `create()` 只会运行 stub，需要挂载真实 agent 才能执行
2. **合理设置并发限制** - 根据系统资源设置 `max_concurrent_subagents`
3. **使用事件过滤** - 订阅特定 SubAgent 的事件以减少噪音
4. **处理错误状态** - 监听 `SubAgentStateChanged` 事件并处理错误
5. **设置超时** - 为长时间运行的任务设置 `timeout_ms`，超时后任务会被自动取消
6. **清理资源** - 使用 `cancel()` 清理不再需要的 SubAgent
