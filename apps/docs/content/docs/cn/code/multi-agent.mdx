# 多智能体编排

A3S Code 通过 `AgentOrchestrator` 提供统一的多智能体编排层，支持两种执行模式：**独立子代理**（`spawn`）和**团队工作流**（`run_team`）。两者共用同一个声明类型 `AgentSlot`。

## 架构

```text
┌─────────────────────────────────────────────────────────┐
│                    AgentOrchestrator                    │
│                                                         │
│  ┌──────────────────┐      ┌─────────────────────────┐  │
│  │  spawn(AgentSlot) │      │ run_team(goal, ws, slots)│  │
│  │  role = None      │      │  role = Lead/Worker/     │  │
│  │                   │      │         Reviewer         │  │
│  └────────┬─────────┘      └──────────────┬──────────┘  │
│           │                               │              │
│  SubAgentWrapper                    AgentTeam +          │
│  ├─ AgentRegistry                   TeamRunner           │
│  ├─ ToolExecutor                    ├─ Lead 分解目标     │
│  └─ AgentLoop (真实 LLM)            ├─ Worker 执行任务   │
│                                     └─ Reviewer 审批     │
└─────────────────────────────────────────────────────────┘
              ↑
        AgentRegistry
   (内置: general / explore / plan)
   (自定义: ~/.a3s/agents/*.yaml)
```

**三个概念的关系：**

| 概念 | 职责 |
|------|------|
| `AgentDefinition` / `AgentRegistry` | 静态 agent 定义（YAML/Markdown），描述 agent 的提示词、权限和工具访问 |
| `AgentSlot` | 运行时声明类型，指定要启动的 agent 类型和角色 |
| `AgentOrchestrator` | 执行引擎，通过 `spawn` 或 `run_team` 驱动 AgentLoop |

## AgentSlot — 统一声明类型

`AgentSlot` 是描述"如何启动一个 agent"的统一声明类型，同时用于独立子代理和团队成员：

```rust
pub struct AgentSlot {
    pub agent_type: String,             // "general" | "explore" | "plan" | 自定义名称
    pub role: Option<TeamRole>,         // None = 独立子代理; Some(Lead/Worker/Reviewer) = 团队成员
    pub description: String,
    pub prompt: String,
    pub permissive: bool,               // true = 跳过人工确认
    pub max_steps: Option<usize>,
    pub timeout_ms: Option<u64>,
    pub workspace: String,
    pub agent_dirs: Vec<String>,        // 额外的自定义 agent 目录
    // ... 其他字段
}
```

`role` 字段决定 slot 的执行路径：
- `role = None` → `spawn()` 作为独立子代理运行
- `role = Some(Lead)` / `Some(Worker)` / `Some(Reviewer)` → `run_team()` 组建团队

## 路径一：独立子代理 (spawn)

`spawn(AgentSlot)` 在隔离的子会话中运行单个 agent，支持实时监控和动态控制。

### 快速开始

<Tabs items={['Rust', 'TypeScript', 'Python']}>
<Tab value="Rust">

```rust
use a3s_code_core::{Agent, orchestrator::{AgentOrchestrator, AgentSlot}};
use std::sync::Arc;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let agent = Arc::new(Agent::new("agent.hcl").await?);
    let orch = AgentOrchestrator::from_agent(agent);

    let slot = AgentSlot::new("general", "查找所有 Rust 源文件")
        .with_description("Find Rust files")
        .with_permissive(true)
        .with_max_steps(10);

    let handle = orch.spawn(slot).await?;
    let result = handle.wait().await?;
    println!("{result}");
    Ok(())
}
```

### 子代理活动类型

| 活动 | 描述 |
|------|------|
| `Idle` | 空闲 |
| `CallingTool { tool_name, args }` | 正在执行工具调用 |
| `RequestingLlm { message_count }` | 正在向 LLM 发送请求 |
| `WaitingForControl { reason }` | 等待控制信号 |

### 事件类型

| 事件 | 描述 |
|------|------|
| `SubAgentStarted` | 子代理启动 |
| `SubAgentCompleted` | 子代理完成 |
| `SubAgentStateChanged` | 状态变更 |
| `SubAgentProgress` | 进度更新 |
| `SubAgentInternalEvent` | 内部事件（来自 AgentLoop） |
| `ToolExecutionStarted/Completed` | 工具执行 |
| `ControlSignalReceived/Applied` | 控制信号 |

## 路径二：团队工作流 (run_team)

`run_team(goal, workspace, slots)` 自动组建 Lead → Worker → Reviewer 团队，将目标分解成任务并协调执行。

<Callout type="info">
  **前提**：需要使用 `Orchestrator.create(agent)` 挂载真实 Agent。
</Callout>

### 快速开始

<Tabs items={['Rust', 'TypeScript', 'Python']}>
<Tab value="Rust">

```rust
use a3s_code_core::{Agent, orchestrator::{AgentOrchestrator, AgentSlot, TeamRole}};
use std::sync::Arc;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let agent = Arc::new(Agent::new("agent.hcl").await?);
    let orch = AgentOrchestrator::from_agent(agent);

    let slots = vec![
        AgentSlot::new("general", "").with_role(TeamRole::Lead).with_max_steps(5),
        AgentSlot::new("general", "").with_role(TeamRole::Worker).with_max_steps(5),
        AgentSlot::new("general", "").with_role(TeamRole::Reviewer).with_max_steps(3),
    ];

    let result = orch.run_team(
        "列举 3 种常见的 Rust 数据结构并简要说明",
        ".",
        slots,
    ).await?;

    println!("完成任务: {}", result.done_tasks.len());
    println!("拒绝任务: {}", result.rejected_tasks.len());
    println!("轮数: {}", result.rounds);
    for task in &result.done_tasks {
        println!("  [{}] {}", task.id, task.result.as_deref().unwrap_or(""));
    }
    Ok(())
}
```

### 工作流程

```
1. Lead 分解
   ├─ 接收目标，输出 JSON: {"tasks": ["任务1", "任务2", ...]}
   └─ 为每个任务调用 board.post()

2. Worker 执行（并发）
   ├─ 循环 claim() → 认领任务 → session.send(任务) → board.complete()
   └─ 无任务时等待 poll_interval_ms

3. Reviewer 审批
   ├─ 对每个 InReview 任务，检查结果是否符合要求
   ├─ 输出 "APPROVED: <原因>" → board.approve() → 任务进入 Done
   └─ 输出 "REJECTED: <反馈>" → board.reject() → 任务重新入队

4. 终止条件
   └─ 全部任务完成，或达到 max_rounds 上限
```

### TeamRole 枚举

| 角色 | Rust | TypeScript/Python | 职责 |
|------|------|-------------------|------|
| Lead | `TeamRole::Lead` | `"lead"` | 分解目标，发布任务 |
| Worker | `TeamRole::Worker` | `"worker"` | 认领并执行任务 |
| Reviewer | `TeamRole::Reviewer` | `"reviewer"` | 审批任务结果 |

## 任务板（高级用法）

对于需要完全控制编排逻辑的场景，可以直接操作 `TeamTaskBoard`：

```rust
use a3s_code_core::agent_teams::{AgentTeam, TeamConfig, TeamRole, TeamRunner};

let config = TeamConfig { max_tasks: 50, max_rounds: 10, ..Default::default() };
let mut team = AgentTeam::new("my-team", config);
team.add_member("lead",     TeamRole::Lead);
team.add_member("worker-1", TeamRole::Worker);
team.add_member("reviewer", TeamRole::Reviewer);

let board = team.task_board();

// 手动操作任务板
let id = board.post("重构认证模块", "lead", None).unwrap();
let task = board.claim("worker-1").unwrap();
board.complete(&task.id, "已完成重构");
board.approve(&task.id);

// 统计
let (open, in_progress, in_review, done, rejected) = board.stats();
```

### 任务状态机

```
post()      claim()       complete()
  Open ──► InProgress ──► InReview
                               │
                   ┌───────────┴──────────┐
              approve()              reject()
                   │                      │
                 Done           Rejected ──► (重新 claim())
```

### 点对点消息传递

```rust
// 向指定成员发送消息
team.send_message("lead", "worker-1", "重点关注 JWT 验证", None).await;

// 广播给所有成员
team.broadcast("lead", "开始审核阶段", None).await;

// Worker 读取收件箱
let mut rx = team.take_receiver("worker-1").unwrap();
while let Some(msg) = rx.recv().await {
    println!("{} → {}: {}", msg.from, msg.to, msg.content);
}
```

```yaml
name: security-reviewer
description: Reviews code for security vulnerabilities
mode: subagent
max_steps: 25

permissions:
  allow: [read, grep, glob, ls]
  deny:  [write, edit, bash]
```

```markdown

name: db-migration
description: Database migration specialist
mode: subagent

permissions:
  allow: [read, write, edit, bash, grep, glob]
  deny:  [web_fetch, web_search]
```

## Orchestrator 创建模式

| 模式 | API | 子代理运行方式 |
|------|-----|---------------|
| 占位符（测试） | `Orchestrator.create()` / `AgentOrchestrator::new_memory()` | 内部 stub，不调用 LLM |

## 两种路径的选择

| 场景 | 推荐路径 |
|------|---------|
| 运行单个专用任务 | `spawn(AgentSlot)` |
| 需要实时监控和暂停/恢复控制 | `spawn(AgentSlot)` |
| 目标需要分解成多个子任务并行执行 | `run_team(goal, ws, slots)` |
| 需要 Lead/Worker/Reviewer 协作和质量审批 | `run_team(goal, ws, slots)` |
| 需要完全控制任务板和编排逻辑 | `AgentTeam` + `TeamRunner` 直接操作 |
