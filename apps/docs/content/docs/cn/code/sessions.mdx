# 会话

每个会话绑定到一个工作区目录。Agent 通过 `agent.session(workspace, options?)` 创建会话。会话持有自己的 LLM 客户端、对话历史和工具上下文。

生成 API — `send()` 和 `stream()` — 将提示发送给 LLM 并返回响应。Agent 循环自动处理工具执行。

## 系统提示词自定义（插槽式）

在不覆盖核心 Agent 能力的前提下自定义 Agent 行为。四个命名插槽允许注入自定义个性，同时保留工具使用策略、自主行为和完成标准：

| 插槽 | 位置 | 行为 |
|------|------|------|
| `role` | 核心提示词之前 | 替换默认的 "You are A3S Code..." 身份行 |
| `guidelines` | 核心提示词之后 | 作为 `## Guidelines` 章节追加 |
| `response_style` | 替换章节 | 替换默认的 `## Response Format` 章节 |
| `extra` | 提示词末尾 | 自由格式指令（向后兼容旧版 system_prompt） |

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const session = agent.session('/my-project', {
  role: 'You are a senior Rust developer.',
  guidelines: 'Use clippy. No unwrap(). Prefer Result.',
  responseStyle: 'Be concise. Use bullet points.',
  extra: 'This project uses tokio and axum.',
});
```
</Tab>
<Tab value="Python">
```python
from a3s_code import SessionOptions
```

## 错误恢复与弹性

三层错误恢复机制保护长时间运行的会话：

<Accordions>
<Accordion title="解析错误恢复">
当 LLM 返回格式错误的工具参数（`__parse_error`）时，错误会作为工具结果反馈给模型以便自我修正。连续失败 `max_parse_retries` 次后循环中止。
</Accordion>
<Accordion title="工具执行超时">
每个工具调用都包装在 `tokio::time::timeout` 中。超时的工具会产生错误消息反馈给 LLM；会话继续运行。
</Accordion>
<Accordion title="熔断器">
瞬态 LLM API 故障（网络错误、速率限制）在非流式模式下会以短指数退避重试最多 `circuit_breaker_threshold` 次。在流式模式下，任何故障都是致命的（事件无法重放）。
</Accordion>
</Accordions>

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// Individual controls
const session = agent.session('.', {
  maxParseRetries: 3,        // bail after 3 consecutive parse errors
  toolTimeoutMs: 30000,      // 30s per tool
  circuitBreakerThreshold: 5 // retry LLM up to 5 times
});
```
</Tab>
<Tab value="Python">
```python

  max_parse_retries=3,        # bail after 3 consecutive parse errors

  circuit_breaker_threshold=5 # retry LLM up to 5 times
)
```

## 流式

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// Returns an EventStream — use for await...of or call .next() manually
const stream = await session.stream('Refactor the auth module');
for await (const event of stream) {
  if (event.type === 'text_delta') process.stdout.write(event.text);
  if (event.type === 'tool_start') console.log(`\n🔧 ${event.toolName}`);
  if (event.type === 'tool_end') console.log(`  → ${event.toolOutput?.slice(0, 100)}`);
}

// Or iterate manually with .next()
const stream2 = await session.stream('Explain src/main.rs');
while (true) {
  const { value, done } = await stream2.next();
  if (done) break;
  if (value.type === 'text_delta') process.stdout.write(value.text);
}
```
</Tab>
<Tab value="Python">
```python
# Sync iteration (works without an event loop)
for event in session.stream("Refactor the auth module"):
    if event.event_type == "text_delta":
        print(event.text, end="", flush=True)
    elif event.event_type == "tool_start":
        print(f"\n🔧 {event.tool_name}")
    elif event.event_type == "tool_end":
        print(f"  → {event.tool_output[:100]}")

# Async iteration (inside async def)
async for event in session.stream("Refactor the auth module"):
    if event.event_type == "text_delta":
        print(event.text, end="", flush=True)
    elif event.event_type == "end":
        print(f"\nDone — {event.total_tokens} tokens")
        break
```
</Tab>

## 斜杠指令

会话指令在发送给 LLM 之前被拦截处理。在任意会话中输入 `/help` 可列出所有可用指令。

| 指令 | 描述 |
|------|------|
| `/help` | 列出可用指令 |
| `/model [provider/model]` | 显示或切换当前模型 |
| `/cost` | 显示 token 使用量和估算费用 |
| `/history` | 显示对话轮次数和 token 统计 |
| `/clear` | 清空对话历史 |
| `/compact` | 手动触发上下文压缩 |
| `/tools` | 列出已注册的工具 |
| `/mcp` | 列出已连接的 MCP 服务器及其工具 |
| `/loop [interval] <prompt>` | 调度循环提示（默认间隔：10m） |
| `/cron-list` | 列出所有已调度的循环提示 |
| `/cron-cancel <id>` | 按 ID 取消定时任务 |

### 自定义指令

注册在 LLM 之前处理的自定义斜杠指令：

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
function statusHandler(args: string, ctx: CommandContext): string {
  console.log(`Model: ${ctx.model}, History: ${ctx.historyLen} msgs, args: ${args}`);
  return `Status: OK`;  // 注意：返回值会被记录但当前实现中不会被捕获
}

session.registerCommand("status", "显示会话状态", statusHandler);

const result = await session.send("/status hello");
console.log(result.text);
// Command 'status' executed successfully
```

处理函数接收：
- `args` — 指令名称之后的所有内容（字符串）
- `ctx` — 包含 `sessionId`、`model`、`historyLen`、`totalTokens`、`totalCost`、`toolNames` 和 `mcpServers` 的对象

**注意**：由于 napi-rs 的限制，处理函数的返回值当前无法被捕获。命令会成功执行，您可以在处理函数中使用 `console.log()` 或其他副作用。未来版本将支持返回值捕获。

</Tab>
<Tab value="Python">
```python
def status_handler(args: str, ctx: dict) -> str:

session.register_command("status", "Show session status", status_handler)

print(result.text)
```

处理函数接收：
- `args` — 指令名称之后的所有内容（字符串）

## 定时任务

会话通过 `/loop` 斜杠指令或程序化调度器 API 支持循环提示。定时任务在每次 `send()` 调用返回后、其间隔时间到期时触发。后台 ticker 在第一次 `send()` 时懒启动，会话销毁时自动停止。

### 调度循环提示

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// 通过 /loop 斜杠指令
const r = await session.send('/loop 30s check deployment status');
console.log(r.text);
// Scheduled [a1b2c3d4]: "check deployment status" — fires every 30s

// 程序化（间隔单位为秒）
const taskId = session.scheduleTask('summarize recent commits', 300);  // 每 5 分钟
console.log(`Scheduled: ${taskId}`);
```
</Tab>
<Tab value="Python">
```python
# 通过 /loop 斜杠指令
r = session.send('/loop 30s check deployment status')
print(r.text)
# Scheduled [a1b2c3d4]: "check deployment status" — fires every 30s

# 程序化（间隔单位为秒）
task_id = session.schedule_task('summarize recent commits', 300)  # 每 5 分钟
print(f'Scheduled: {task_id}')
```
</Tab>
</Tabs>

**间隔语法：**

| 格式 | 示例 | 含义 |
|------|------|------|
| 秒 | `30s` | 每 30 秒 |
| 分钟 | `5m` | 每 5 分钟 |
| 小时 | `2h` | 每 2 小时 |
| 天 | `1d` | 每天 |
| 前置 | `/loop 5m <prompt>` | 间隔在提示词前 |
| 后置 | `/loop <prompt> every 5m` | `every` 子句在末尾 |
| 默认 | `/loop <prompt>` | 每 10 分钟 |
