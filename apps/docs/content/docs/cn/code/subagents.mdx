# 子智能体

A3S Code 支持将工作委托给**子智能体** — 运行独立工具循环并将结果返回给父智能体的独立 agent 实例。三个内置工具负责委托：

| 工具 | 模式 | 适用场景 |
|---|---|---|
| `task` | 单个子智能体 | 一项专注的工作 |
| `parallel_task` | N 个并发子智能体 | 可并行展开的独立任务 |
| `run_team` | Lead → Worker → Reviewer | 需要分解和质量审查的复杂目标 |

所有委托工具均为内置工具，始终可用 — 无需挂载插件。

## `task` — 单个子智能体

将提示委托给一个子智能体。子智能体在独立会话中运行，拥有独立的对话历史，不继承父智能体的历史。

```json
{
  "agent": "general",
  "prompt": "为 src/auth.rs 编写覆盖所有边界情况的单元测试",
  "workspace": "/可选/路径/覆盖"
}
```

  type={{
    agent: { type: 'string', required: true, description: '委托的 agent 名称（如 "general"）' },
    prompt: { type: 'string', required: true, description: '子智能体的任务提示' },
    workspace: { type: 'string', description: '工作区路径覆盖（默认: 继承父工作区）' },

### 示例

<Tabs groupId="lang" items={['TypeScript', 'Python']}>

```typescript
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.hcl');
const session = agent.session('.');

console.log(result.text);
```

## `parallel_task` — 并发子智能体

并发运行多个任务，每个任务拥有独立的 agent 实例。结果统一收集后返回。

```json
{
  "tasks": [
    { "agent": "general", "prompt": "审计 src/auth.rs 的安全问题" },
    { "agent": "general", "prompt": "审计 src/db.rs 的 SQL 注入风险" },
    { "agent": "general", "prompt": "审计 src/api.rs 的输入校验缺口" }
  ]
}
```

<TypeTable
  type={{
    tasks: { type: 'array', required: true, description: '要并行运行的 {agent, prompt, workspace?} 对象数组' },
  }}
/>

<Callout type="info">
`parallel_task` 中的每个任务独立运行，不共享状态，也不互相通信。
</Callout>

## `run_team` — Lead → Worker → Reviewer

为需要分解、并行执行和质量审查的目标协调三角色团队：

1. **Lead（领导）** — 将目标拆解为任务列表
2. **Workers（执行者）** — 并行认领并执行任务
3. **Reviewer（审查者）** — 对每个结果进行审批或拒绝

```json
{
  "goal": "审计 authentication 模块的安全问题",
  "lead_agent": "general",
  "worker_agent": "general",
  "reviewer_agent": "general",
  "max_steps": 10
}
```

<TypeTable
  type={{
    goal: { type: 'string', required: true, description: '团队要完成的目标' },
    lead_agent: { type: 'string', default: '"general"', description: 'Lead 角色的 agent 类型' },
    worker_agent: { type: 'string', default: '"general"', description: 'Worker 角色的 agent 类型' },
    reviewer_agent: { type: 'string', default: '"general"', description: 'Reviewer 角色的 agent 类型' },
    max_steps: { type: 'number', description: '每个团队成员的最大工具轮次' },
  }}
/>

## 如何选择

| 场景 | 推荐工具 |
|---|---|
| 分派单项专注任务 | `task` |
| 同时审计多个模块 | `parallel_task` |
| 需要规划 + 审查的大型目标 | `run_team` |
| 并行探索多个假设 | `parallel_task` |
| 一次完成重构 + 测试 + 文档 | `run_team` |

## 子智能体隔离

每个子智能体运行在独立会话中：

- **独立对话历史** — 子智能体看不到父智能体的消息
- **默认共享工作区** — 子智能体对文件的修改对父智能体可见
- **工作区覆盖** — 通过 `workspace` 参数隔离文件写入
- **相同配置** — 子智能体继承相同的 `agent.hcl` 和模型设置

## 编排子智能体（Rust API）

对运行中的 agent 进行高级控制，使用 `AgentOrchestrator`：

```rust
use a3s_code_core::orchestrator::AgentOrchestrator;

let orchestrator = AgentOrchestrator::new();

// 暂停 / 恢复 / 取消运行中的子智能体
orchestrator.pause_agent(&agent_id).await?;
orchestrator.resume_agent(&agent_id).await?;
orchestrator.cancel_agent(&agent_id).await?;

// 在执行过程中注入提示
orchestrator.inject_prompt(&agent_id, "停下来，总结你目前做了什么").await?;
```

详见 [编排器](/cn/docs/code/orchestrator)。
