# A3S Code

A3S Code v2.3.0 是一个驾驭式编程智能体运行时。本节文档聚焦当前 Node SDK 能力面。

## v2.3.0 新特性

| 领域 | v2.3.0 新内容 |
| --- | --- |
| SDK API | 紧凑的对象形式方法：`send()`、`run()`、`stream()`、`task()`、`tasks()`、`git()`、`addMcp()`、`removeMcp()`、`mcps()` 用于长期集成。 |
| 可观测性 | 通过活跃工具快照和更丰富的 run replay API 实现实时 run/tool 可观测性，覆盖 Rust、Node.js 和 Python SDK。 |
| Python SDK | Worker agents、HITL 确认策略/控制、session-for-worker、实时 worker 注册和 session close 的完整功能对等。 |
| AHP 集成 | 单一驾驭/咨询/控制平面，具有更丰富的事件上下文、心跳状态、运行时状态快照和决策映射。 |
| 架构 | 将大型 agent 和 session API 实现文件拆分为聚焦的运行时模块，提升可维护性。 |
| 文档 | 在 `manual/SDK_API_DESIGN.md` 下添加持久的 SDK API 设计契约。 |
| 清理 | 移除过时的 sidecar/copilot/BTW/strategize/BTE 机制及相关 prompts、docs、configs 和 examples。 |

### 继承自 v2.0-2.2

| 领域 | 特性 |
| --- | --- |
| 配置 | 只支持 ACL 配置，使用带标签的 `providers` 和 `models` 块；`env(...)` 是标准凭证注入方式。 |
| 委派 | `task` 和 `parallel_task` 是默认多运行路径，委派子运行返回紧凑摘要而不是完整 transcript。 |
| 工具表面 | 核心工具保持小而稳定；web、git、batch、skill 管理和 MCP 工具按意图启用。`generate_object` 提供 schema 校验的结构化输出。 |
| PTC | Programmatic Tool Calling 在内嵌 QuickJS 中运行有边界的 JavaScript；Node SDK 通过 `session.program(...)` 对齐核心 `program` 工具。 |
| 持久化 | file/memory store 支持可恢复 session、显式 `save()`、`autoSave`、memory recall 和 session ID。 |

当前执行形态是：

```text
Agent / AgentSession
  -> 上下文组装
  -> 工具选择
  -> 确定性任务走 PTC
  -> 已配置的权限与确认策略
  -> 执行
  -> 追踪与验证证据
  -> 压缩
```

## 安装

```bash
npm install @a3s-lab/code
```

## 配置

v2.3.0 只使用 ACL。配置文件使用 `.acl` 后缀和带标签的块；旧的 HCL 配置不再支持。

```text
default_model = "anthropic/claude-sonnet-4-20250514"

providers "anthropic" {
  apiKey = env("ANTHROPIC_API_KEY")
}
```

MiniMax 等 OpenAI 兼容服务使用 `openai` provider 和 `baseUrl`：

```text
default_model = "openai/MiniMax-M2.7-highspeed"

providers "openai" {
  apiKey = env("A3S_OPENAI_API_KEY")
  baseUrl = env("A3S_OPENAI_BASE_URL")

  models "MiniMax-M2.7-highspeed" {
    name = "MiniMax M2.7 Highspeed"
    tool_call = true
  }
}

storage_backend = "file"
```

发布集成脚本也接受 `MINIMAX_API_KEY` 和 `MINIMAX_BASE_URL`，并把它们映射成 OpenAI 兼容 provider 使用的环境变量。

## 使用

```ts
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.acl');
const session = agent.session('/my-project', {
  builtinSkills: true,
  planningMode: 'enabled',
});

const result = await session.send('定位认证错误处理逻辑并总结流程');
console.log(result.text);
console.log(result.verificationSummaryText);
```

## 当前 v2.3.0 能力面

### 核心 API

- **紧凑 SDK 方法**：`send()`、`run()`、`stream()` 用于对话；`task()`、`tasks()` 用于委派；`git()` 用于 git 操作；`addMcp()`、`removeMcp()`、`mcps()` 用于 MCP 管理。
- **可观测性**：`runs()`、`runSnapshot()`、`runEvents()`、`activeTools()`、`cancelRun()` 用于实时 run/tool 追踪。
- **配置**：`.acl` 是唯一支持的格式；`Agent.create()` 接受 `.acl` 文件或内联 ACL 字符串。
- **Planning**：明确的 `auto` / `enabled` / `disabled` 契约；`auto` 使用结构化预分析而非关键词门控。
- **PTC**：Programmatic Tool Calling 通过 `session.program(...)` 将确定性工具链移到宿主代码执行。
- **委派**：`task()` 和 `tasks()`（原 `parallel_task`）返回紧凑子摘要而非完整 transcript。

### Python SDK 功能对等

- Worker agents、HITL 确认策略/控制、`session_for_worker()`、实时 worker 注册和 `close()`。
- 与 Node.js SDK 完全功能对等，适用于生产集成。

### 基础设施

- **AHP 集成**：单一驾驭/咨询/控制平面，具有更丰富的事件上下文、心跳状态和决策映射。
- **Hooks**：注册、计数和卸载是公开 session API。
- **权限**：显式规则列表加默认决策。
- **Sessions**：默认不启用队列；lane queue 是显式的外部或混合分发基础设施。
- **结果**：通过 `session.traceEvents()`、`session.verificationReports()` 和 run replay API 获取 token 统计、验证状态和验证摘要。

建议从 [API 契约](/cn/docs/code/api-contract)、[会话](/cn/docs/code/sessions)和[Provider 配置](/cn/docs/code/providers)开始。
