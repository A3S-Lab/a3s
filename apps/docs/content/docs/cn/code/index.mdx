# A3S Code

A3S Code v3.4.0 是一个驾驭式编程智能体运行时。运行时保持 agent loop 小而可观测，驾驭层负责上下文组装、工具执行、权限策略、任务委派、验证证据与 run replay。

## v3.4.0 重点

| 领域 | 当前能力 |
| --- | --- |
| 自动委派 | 可根据显式 `@agent` 提及、高置信任务描述和自定义 agent 的 proactive 描述自动触发 subagent。 |
| 内置 subagent | `explore`、`plan`、`general` / `general-purpose`、`verification`、`review` 可通过 `task`、`parallel_task`、`session.task(...)`、`session.tasks(...)` 和自动委派使用。 |
| Agent 目录 | 自定义 Markdown/YAML agents 会从配置的 `agent_dirs`、项目/用户 `.a3s/agents` 以及 Claude 兼容的 `.claude/agents` 迁移路径加载。`.a3s/agents` 是 A3S 原生位置，并在同一作用域同名时优先。 |
| 并行控制 | `max_parallel_tasks` 限制 sibling fan-out。`auto_parallel = false` / `autoParallel: false` 只全局关闭自动并行子 agent fan-out；手动 `parallel_task` 仍可用。 |
| 可编程编排 | [`session.parallel`](/cn/docs/code/orchestration)、`session.pipeline`、`session.parallelResumable` 表达确定性 fan-out、无屏障 pipeline 以及可恢复/可迁移工作流，与模型驱动的 `task` / `tasks` 委派互补。 |
| SDK 对齐 | Rust、Node.js 和 Python 暴露同一组核心委派、run replay、typed tool errors、workspace backend 与自动委派控制。 |
| Workspace 栈 | 本地文件系统、S3 兼容对象存储和 HTTP/JSON remote-git backend 共享端到端 typed `ToolErrorKind`。 |

当前执行形态：

```text
Agent / AgentSession
  -> 上下文组装
  -> 可选 planning
  -> 配置启用时自动 subagent 委派
  -> 工具选择或 programmatic tool calling
  -> 权限与确认策略
  -> 执行
  -> trace、artifact 与验证证据
  -> 压缩
```

## 安装

```bash
npm install @a3s-lab/code
pip install a3s-code
cargo add a3s-code-core
```

## 配置

A3S Code 使用 ACL 配置。推荐 `.acl` 文件和带标签的块；JSON 和旧 HCL 配置不再支持。

```text
default_model = "openai/MiniMax-M2.7-highspeed"
max_parallel_tasks = 8
auto_parallel = false

providers "openai" {
  apiKey = env("A3S_OPENAI_API_KEY")
  baseUrl = env("A3S_OPENAI_BASE_URL")

  models "MiniMax-M2.7-highspeed" {
    name = "MiniMax M2.7 Highspeed"
    tool_call = true
  }
}

agent_dirs = ["./.a3s/agents"]

auto_delegation {
  enabled        = true
  auto_parallel  = false
  min_confidence = 0.72
  max_tasks      = 4
}

storage_backend = "file"
```

顶层 `auto_parallel` 是自动并行 fan-out 的全局开关。它不会移除 `parallel_task` 或 SDK 的 `session.tasks(...)` helper。

## 使用

```ts
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.acl');
const session = agent.session('/my-project', {
  builtinSkills: true,
  planningMode: 'auto',
  autoDelegation: { enabled: true, maxTasks: 4 },
  maxParallelTasks: 8,
  autoParallel: false,
});

const result = await session.send(
  '使用 @review 检查认证相关变更，并总结发布阻塞项',
);

console.log(result.text);
console.log(result.verificationSummaryText);
```

## 当前能力面

- **对话**：`send()`、`run()` 和 `stream()` 用于同步与流式回合。
- **委派**：`task()` 和 `tasks()` 封装核心 `task` / `parallel_task` 工具，并返回紧凑子任务摘要。
- **编排**：`session.parallel(...)`、`session.pipeline(...)` 和 `session.parallelResumable(...)` 在 `AgentExecutor` seam 之上表达可编程 fan-out、无屏障 pipeline 以及可恢复/可迁移工作流。
- **自动 subagent**：通过 `autoDelegation` 或 `auto_delegation` 启用，由 `maxParallelTasks` / `max_parallel_tasks` 限流，可用 `autoParallel: false` / `auto_parallel = false` 关闭自动并行。
- **可观测性**：`runs()`、`runSnapshot()`、`runEvents()`、`activeTools()`、`cancelRun()` 用于实时 run/tool 追踪。
- **PTC**：`session.program(...)` 在 QuickJS 中运行有边界的 JavaScript 工具链。
- **结构化输出**：`generate_object` 生成带修复重试的 schema 校验 JSON。
- **配置**：`Agent.create()` 接受 `.acl` 文件路径或内联 ACL 字符串。
- **持久化**：file/memory store 支持可恢复 session、显式 `save()`、`autoSave`、memory recall 和 session ID。

建议从 [API 契约](/cn/docs/code/api-contract)、[会话](/cn/docs/code/sessions)、[任务](/cn/docs/code/tasks) 和 [Provider 配置](/cn/docs/code/providers)开始。
