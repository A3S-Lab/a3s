# Agentic Search Tool

A3S Code 新增了 `agentic_search` 工具，这是一个受 Sirchmunk 启发的智能代码搜索工具，实现了多阶段搜索管道。

<Callout type="info">
与传统的 `grep` 工具不同，`agentic_search` 接受自然语言查询，自动提取关键词，并使用 IDF 加权相关性评分对结果进行排序。
</Callout>

## 核心特性

- **自然语言查询** — 输入 "JWT token validation" 而不是正则表达式
- **自动关键词提取** — 移除停用词，提取多粒度关键词
- **并行搜索** — 同时搜索文件内容和结构
- **相关性排序** — IDF 加权评分，匹配更多关键词的文件排名更高
- **文件类型感知** — 代码文件获得 1.2x 提升，配置文件 1.0x，文档 0.9x
- **蒙特卡洛采样** — DEEP 模式使用高斯重要性采样进行深度分析

## 搜索管道

### FAST 模式（默认）

```
Phase 1: 关键词提取
  ↓ 移除停用词，提取 2-3 词组合和单词

Phase 2: 并行搜索
  ├─ 内容搜索：使用 ripgrep 风格的正则匹配
  └─ 结构搜索：文件名和路径匹配

Phase 3: 相关性评分
  ↓ relevance = (match_count / sqrt(file_lines)) * idf_boost * file_type_boost

Phase 4: 结果格式化
  ↓ 返回带上下文的匹配行
```

### DEEP 模式（蒙特卡洛采样）

```
Phase 1: 初始搜索
  ↓ 获取 2x max_results 作为采样池

Phase 2: 蒙特卡洛证据采样
  ├─ Act 1: 模糊锚定 — 识别高分区域
  ├─ Act 2: 高斯重要性采样 — 在锚点周围采样（sigma 衰减）
  └─ Act 3: Top-K 选择 — 返回最高证据分数的区域

Phase 3: 证据合成
  ↓ 将采样结果组织为连贯的证据区域

Phase 4: 增强格式化
  ↓ 显示证据分数和高斯权重
```

### 搜索模式

#### FAST 模式（默认）

最常用的模式，2-5 秒返回结果：

```typescript
agentic_search({
  query: "how does authentication work",
  max_results: 5,
  context_lines: 2
})
```

#### DEEP 模式（蒙特卡洛采样）

深度分析模式，使用高斯重要性采样，10-30 秒返回结果：

```typescript
agentic_search({
  query: "explain the entire authentication flow",
  mode: "deep",
  max_results: 3
})
```

**DEEP 模式特性：**
- 初始搜索获取 2x 结果作为采样池
- 使用高斯分布在高分匹配周围采样
- sigma 根据匹配密度自适应衰减（5.0 → 2.0）
- 计算证据分数：`relevance * sum(weights) / sqrt(sampled_lines)`
- 将采样行组织为连贯的证据区域

#### FILENAME_ONLY 模式

快速文件发现，只搜索文件名：

```typescript
agentic_search({
  query: "auth",
  mode: "filename_only",
  max_results: 20
})
```

## 相关性评分算法

```rust
// 基础分数：匹配数量 / sqrt(文件总行数)
let base_score = (match_count as f32) / (total_lines as f32).sqrt();

// IDF 提升：匹配的唯一关键词数 / 总关键词数
let idf_boost = (unique_keywords_matched as f32) / (total_keywords as f32);

// 文件类型提升
let type_boost = match file_type {
    Code => 1.2,
    Config => 1.0,
    Documentation => 0.9,
    Other => 0.8,
};

// 最终相关性
let relevance = base_score * idf_boost * type_boost;
```

这确保：
- 匹配更多关键词的文件排名更高
- 较小、更聚焦的文件比大文件排名更高
- 代码文件优先于文档文件

## 对比：agentic_search vs grep

| 特性 | agentic_search | grep |
|------|----------------|------|
| 输入 | 自然语言 | 正则表达式 |
| 关键词提取 | 自动 | 手动 |
| 相关性排序 | 是（IDF 加权） | 否（按文件顺序） |
| 文件类型感知 | 是 | 否 |
| 停用词过滤 | 是 | 否 |
| 最适合 | 探索性搜索 | 精确模式匹配 |

**使用建议：**
- 使用 `agentic_search` 进行初始发现
- 使用 `grep` 进行精确的后续搜索

## 最佳实践

### 1. 使用自然语言

```typescript
// ✅ 好
agentic_search({ query: "how does authentication work" })

// ❌ 不好
agentic_search({ query: "auth" })
```

### 2. 具体化查询

```typescript
// ✅ 好
agentic_search({ query: "JWT token validation logic" })

// ❌ 不好
agentic_search({ query: "tokens" })
```

### 3. 结合其他工具

```typescript
// 1. 使用 agentic_search 发现相关文件
const search = await agentic_search({ query: "authentication" });

// 2. 使用 read 检查最相关的文件
const content = await read({ path: "src/auth.rs" });

// 3. 使用 grep 在发现的文件中进行精确搜索
const matches = await grep({
  pattern: "verify_token",
  path: "src/auth.rs"
});
```

### 4. 调整上下文行数

```typescript
// 概览：0-1 行
agentic_search({ query: "auth", context_lines: 0 })

// 标准：2-3 行（默认）
agentic_search({ query: "auth", context_lines: 2 })

// 详细：4-5 行
agentic_search({ query: "auth", context_lines: 5 })
```

### 5. 使用 include 模式

```typescript
// 只搜索 Rust 文件
agentic_search({
  query: "database",
  include: "*.rs"
})

// 搜索配置文件
agentic_search({
  query: "port",
  include: "*.{toml,yaml,json}"
})
```

## 性能

在典型的中型项目（~50k 行代码）上：

| 操作 | 时间 | 说明 |
|------|------|------|
| 单关键词搜索 | ~100ms | 搜索 "authentication" |
| 多关键词搜索 | ~150ms | 搜索 "JWT token validation" |
| 文件名搜索 | ~50ms | filename_only 模式 |
| 大型代码库（500k 行） | ~800ms | 使用适当的 include 模式 |

<Callout type="info">
性能取决于代码库大小、文件数量和磁盘速度。使用 SSD 和适当的 include 模式可获得最佳性能。
</Callout>

## 故障排除

### 搜索太慢

1. 使用 `include` 模式限制文件类型
2. 减少 `max_results`
3. 使用 `filename_only` 模式进行快速发现

### 找不到预期结果

1. 尝试更具体的关键词
2. 检查是否被 `.gitignore` 排除
3. 验证文件大小是否 < 1MB（自动跳过大文件）
4. 尝试使用 `grep` 进行精确匹配

### 结果太多

1. 增加查询的具体性
2. 使用 `include` 模式过滤文件类型
3. 减少 `max_results`
4. 减少 `context_lines`
