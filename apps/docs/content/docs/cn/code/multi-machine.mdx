# 多机器

多机器架构通过显式外部或混合分发实现。

```text
coordinator session
  -> task 或 lane dispatch
  -> external worker
  -> result, changed files, verification evidence
  -> coordinator merge
```

Coordinator 拥有对话、最终合并、应用策略和发布决策。Worker 接收有边界的任务，返回摘要、检查或修改过的文件、命令、验证状态和错误信息。HTTP、消息队列、任务系统或内部 RPC 都可以作为传输层。
