# 架构

A3S Code v2.1.0 围绕小运行时内核和控制执行循环的驾驭层组织。

```text
Prompt
  -> context assembly
  -> tool selection
  -> LLM generation
  -> programmatic tool calling
  -> 已配置的 permission/security integration
  -> tool execution
  -> observation
  -> verification
  -> trace storage
  -> compaction
```

内核负责维护 session、调用 provider、执行工具、记录 observation 并返回结构化事件。驾驭层协调上下文预算、工具可见性、permission/hooks/AHP/security-provider 集成点、输出摘要、完成验证，以及面向宿主应用暴露 run snapshot、TaskList 状态和 trace event。

## PTC 路径

Programmatic Tool Calling 是确定性工作流的驾驭路径。`program` 工具会在内嵌 QuickJS VM 中运行 JavaScript 脚本，并只暴露一个受控 `ctx` 对象。默认情况下，脚本可以调用除 `program` 之外的所有已注册工具；调用方也可以用 `allowed_tools` 收窄工具面。脚本可以组合 `readFile`、`read`、`glob`、`grep`、`ls`、`bash`、`git` 和通用 `tool(name, args)`，并把一个结构化结果交给 agent。

适合使用 PTC 的场景包括：

- 仓库搜索与代码片段收集
- 测试发现与验证命令选择
- 包元数据检查
- 发布 preflight 证据收集
- 日志解析与摘要
- trace 索引

当下一步需要判断时，让模型调用工具；当动作序列已经确定时，用 PTC 执行，再让模型消费结果。脚本可以通过内联 `source` 传入，也可以通过 workspace 相对 `.js` 或 `.mjs` 文件路径载入；VM 本身没有直接文件系统、网络、子进程或环境变量访问权限。
