# AGENTS.md 自动加载

A3S Code 会自动从工作区根目录加载 `AGENTS.md` 文件中的项目特定指令，类似于 Claude Code 的 `CLAUDE.md` 机制。

## 概述

当你创建会话时，A3S Code 会自动：
1. 检查工作区根目录是否存在 `AGENTS.md`
2. 如果找到则加载其内容
3. 将其作为"项目指令"注入到系统提示词中

这允许你提供项目特定的指南、约定和要求，agent 会自动遵循这些规则。

## 使用方法

### 创建 AGENTS.md

在项目根目录放置 `AGENTS.md` 文件：

```markdown
# 项目指令

这是一个使用 Node.js 和 Express 的 TypeScript 项目。

## 代码风格
- 使用 TypeScript 严格模式
- 优先使用 async/await 而不是回调
- 使用 ESLint 和 Prettier
- 使用 Jest 编写测试

## 架构
- 遵循 MVC 模式
- 使用依赖注入
- 保持控制器精简
- 业务逻辑放在服务层

## 测试
- 所有服务都要有单元测试
- API 端点要有集成测试
- 最低 80% 代码覆盖率

## 安全
- 验证所有用户输入
- 使用参数化查询
- 永远不要记录敏感数据
```

### 创建会话

创建会话时，agent 会自动加载 `AGENTS.md`：

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.hcl');

// AGENTS.md 会自动从 /my-project/AGENTS.md 加载
const session = agent.session('/my-project', {
  builtinSkills: true,
});

// agent 现在会遵循你的项目指令
const result = await session.send('创建一个新的用户注册端点');
console.log(result.text);
```

## 行为

### 加载优先级

`AGENTS.md` 内容被注入到系统提示词的 `extra` 插槽中，这意味着：

1. **核心 agentic 行为**始终保留
2. **AGENTS.md** 在技能指令之前加载
3. **自定义 `extra` 插槽**内容（如果提供）在 AGENTS.md 之前

### 日志记录

当 `AGENTS.md` 被加载时，你会看到日志消息：

```
INFO Auto-loaded AGENTS.md from workspace root: /my-project/AGENTS.md
```

如果文件存在但为空，则跳过：

```
DEBUG AGENTS.md exists but is empty — skipping
```

如果读取失败，会记录警告：

```
WARN Failed to read AGENTS.md — skipping: Permission denied
```

### 错误处理

- **文件未找到**：静默跳过（无错误）
- **空文件**：静默跳过（无错误）
- **读取错误**：记录警告，会话创建继续

## 与 Claude Code 的比较

| 功能 | A3S Code | Claude Code |
|------|----------|-------------|
| 文件名 | `AGENTS.md` | `CLAUDE.md` |
| 位置 | 工作区根目录 | 项目根目录 |
| 自动加载 | ✅ 是 | ✅ 是 |
| 格式 | Markdown | Markdown |
| 作用域 | 每个会话 | 每个项目 |

<Callout type="info">
**注意：** `AGENTS.md` 是按会话加载的，不是全局的。每个工作区可以有自己的指令。
