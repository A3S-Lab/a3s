# Cloud 0.1.0 Status and Compatibility

Status: **Verified**

Snapshot: `cloud-0.1.0`

This is versioned documentation for the E0 integration snapshot. It is not a
production support declaration and no matching GitHub release has been
published. The evidence boundary is the clean-host E0 gate described in the
[pinned Cloud development plan](https://github.com/A3S-Lab/Cloud/blob/5d7307cc7bf5b155f708ca9f5bd3e5d06745b68d/docs/development-plan.md).

## Capability status

| Label | Meaning in this snapshot |
| --- | --- |
| **Verified** | The exact locked revisions passed the named executable gate. E0 covers one outbound Linux/Docker node, deployment A, managed TLS, ordered logs, update to B, cloned-A rollback, durable stop, and cleanup. |
| **Experimental** | Implemented or documented with tests, but not wired through a released Cloud operator surface or production gate. The signed node/Gateway updater is in this category. |
| **Optional** | A supported provider choice that is not required by the local profile. NATS and S3-compatible log storage are examples. |
| **Planned** | No shipped contract may be inferred. Cloud CLI, management MCP, managed backups, multi-node evacuation, HA, and production disaster-recovery objectives remain planned. |
