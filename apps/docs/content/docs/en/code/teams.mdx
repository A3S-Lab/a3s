# Teams

A3S Code v2.1.0 no longer exposes legacy public team APIs. Teams are now a harness pattern built from named agent definitions plus the `task` and `parallel_task` tools.

## Recommended Shape

1. Define roles as agent files in an `agents/` directory.
2. Create a parent session with `agentDirs`.
3. Let the parent delegate bounded work through `task`.
4. Use `parallel_task` for independent lanes.
5. Merge child summaries, evidence references, and risks in the parent.

```ts
const session = agent.session('/repo', {
  agentDirs: ['./agents'],
  builtinSkills: true,
  planningMode: 'enabled',
});

await session.send(`
Build a release-readiness team:
- explorer: find risky changed areas
- tester: identify missing verification
- security: review side-effect paths

Use parallel_task and return a single prioritized report.
`);
```

SDK callers can also call `session.delegateTask(...)` or
`session.parallelTask(...)` directly when the host already knows the lanes.

## Worker Agents

Register disposable worker agents with `workerAgents` or `registerWorkerAgent()`:

```ts
const session = agent.session('/repo', {
  workerAgents: [
    {
      name: 'frontend-cow',
      description: 'Small verified frontend fixes',
      kind: 'implementer',
      model: 'openai/gpt-4o',
      maxSteps: 24,
      confirmationInheritance: 'auto_approve',
    },
  ],
});
```

### Confirmation Inheritance

Control how child runs resolve Ask decisions with `confirmationInheritance`:

- `'auto_approve'` (default): Child runs auto-approve all Ask decisions
- `'deny_on_ask'`: Child runs fail immediately when encountering an Ask
- `'inherit_parent'`: Child runs inherit the parent's confirmation policy

```ts
const session = agent.session('/repo', {
  workerAgents: [
    {
      name: 'restricted-writer',
      description: 'Write files with parent confirmation',
      kind: 'implementer',
      confirmationInheritance: 'inherit_parent',  // requires parent approval
    },
  ],
});
```

Use `'inherit_parent'` when child runs need human approval for sensitive operations,
or `'deny_on_ask'` to ensure child runs never block on confirmation prompts.

## Runtime State

Legacy lifecycle control-plane APIs are removed. Applications should observe
streaming events and run replay snapshots, then cancel active Node work with
`cancelRun(runId)` when needed.
