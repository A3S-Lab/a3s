# A3S Code

A3S Code v2.3.0 is a harness-driven runtime for coding agents. This section
focuses on the current Node SDK surface.

## v2.3.0 Highlights

| Area | New in v2.3.0 |
| --- | --- |
| SDK API | Compact, object-shaped methods: `send()`, `run()`, `stream()`, `task()`, `tasks()`, `git()`, `addMcp()`, `removeMcp()`, `mcps()` for long-lived integrations. |
| Observability | Live run/tool observability through active tool snapshots and richer run replay APIs across Rust, Node.js, and Python SDKs. |
| Python SDK | Full parity for worker agents, HITL confirmation policy/control, session-for-worker, live worker registration, and session close. |
| AHP Integration | Single harness/advisory/control plane with richer event context, heartbeat state, runtime state snapshots, and decision mapping. |
| Architecture | Split large agent and session API implementation files into focused runtime modules for maintainability. |
| Documentation | Added durable SDK API design contract under `manual/SDK_API_DESIGN.md`. |
| Cleanup | Removed obsolete sidecar/copilot/BTW/strategize/BTE mechanism and related prompts, docs, configs, and examples. |

### Inherited from v2.0-2.2

| Area | Features |
| --- | --- |
| Configuration | ACL-only config with labeled `providers` and `models` blocks; `env(...)` is the standard credential path. |
| Delegation | `task` and `parallel_task` are the default multi-agent path, returning compact child summaries instead of full transcripts. |
| Tool surface | Core tools stay small; web, git, batch, skill administration, and MCP tools are intent-gated. |
| PTC | Programmatic Tool Calling runs bounded JavaScript in embedded QuickJS; the Node SDK exposes `session.program(...)` over the core `program` tool. |
| Persistence | File and memory stores support resumable sessions, explicit `save()`, `autoSave`, memory recall, and session IDs. |

The current execution shape is:

```text
Agent / AgentSession
  -> context assembly
  -> tool selection
  -> programmatic tool calling when deterministic
  -> permission and confirmation policy where configured
  -> execution
  -> trace and verification evidence
  -> compaction
```

## Install

```bash
npm install @a3s-lab/code
```

## Configure

v2.3.0 uses ACL only. Use `.acl` files and labeled blocks; legacy HCL files are not supported.

```text
default_model = "anthropic/claude-sonnet-4-20250514"

providers "anthropic" {
  apiKey = env("ANTHROPIC_API_KEY")
}
```

OpenAI-compatible services, including MiniMax, use the `openai` provider with `baseUrl`:

```text
default_model = "openai/MiniMax-M2.7-highspeed"

providers "openai" {
  apiKey = env("A3S_OPENAI_API_KEY")
  baseUrl = env("A3S_OPENAI_BASE_URL")

  models "MiniMax-M2.7-highspeed" {
    name = "MiniMax M2.7 Highspeed"
    tool_call = true
  }
}

storage_backend = "file"
```

The release integration scripts also accept `MINIMAX_API_KEY` and `MINIMAX_BASE_URL` as aliases for the OpenAI-compatible environment variables.

## Use

```ts
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.acl');
const session = agent.session('/my-project', {
  builtinSkills: true,
  planningMode: 'enabled',
});

const result = await session.send(
  'Find where authentication errors are handled and summarize the flow',
);

console.log(result.text);
console.log(result.verificationSummaryText);
```

## Current v2.3.0 Surface

### Core API

- **Compact SDK methods**: `send()`, `run()`, `stream()` for conversation; `task()`, `tasks()` for delegation; `git()` for git operations; `addMcp()`, `removeMcp()`, `mcps()` for MCP management.
- **Observability**: `runs()`, `runSnapshot()`, `runEvents()`, `activeTools()`, `cancelRun()` for live run/tool tracking.
- **Configuration**: `.acl` is the only supported format; `Agent.create()` accepts `.acl` files or inline ACL strings.
- **Planning**: Explicit `auto` / `enabled` / `disabled` contract; `auto` uses structured pre-analysis instead of keyword gates.
- **PTC**: Programmatic Tool Calling moves deterministic tool chains into host code via `session.program(...)`.
- **Delegation**: `task()` and `tasks()` (formerly `parallel_task`) return compact child summaries instead of full transcripts.

### Python SDK Parity

- Worker agents, HITL confirmation policy/control, `session_for_worker()`, live worker registration, and `close()`.
- Full feature parity with Node.js SDK for production integrations.

### Infrastructure

- **AHP Integration**: Single harness/advisory/control plane with richer event context, heartbeat state, and decision mapping.
- **Hooks**: Registration, counting, and unregistration are public session APIs.
- **Permissions**: Explicit rule lists plus a default decision.
- **Sessions**: Queue-free by default; lane queues are explicit infrastructure for external or hybrid dispatch.
- **Results**: Token counts, verification status, and verification summaries via `session.traceEvents()`, `session.verificationReports()`, and run replay APIs.
