# A3S Code

A3S Code v2.1.0 is a harness-driven runtime for coding agents. The model reasons; the harness controls context, tool visibility, permissions, execution, verification evidence, traces, artifacts, and compaction.

## v2.1.0 Highlights

| Area | New in v2.1.0 |
| --- | --- |
| Configuration | ACL-only config with labeled `providers` and `models` blocks; `env(...)` is the standard credential path. |
| Provider smoke | Real OpenAI-compatible provider checks support `A3S_OPENAI_*` and MiniMax aliases `MINIMAX_API_KEY` / `MINIMAX_BASE_URL`. |
| Delegation | `task` and `parallel_task` are the default multi-agent path, returning compact child summaries instead of full transcripts. |
| SDK parity | Rust, Node, and Python expose explicit planning mode, run replay, tool schemas, program execution, and delegation helpers. |
| Harness control | AHP 2.4 transports, hooks, explicit permission policy, and security providers all feed one effective action decision. |
| Verification | `AgentResult` exposes verification status, counts, structured summaries, token usage, trace data, and artifact references. |
| Tool surface | Core tools stay small; web, git, batch, skill administration, and MCP tools are intent-gated. |
| PTC | Programmatic Tool Calling runs bounded JavaScript in embedded QuickJS; Node/Python expose `session.program(...)` helpers over the core `program` tool. |
| Persistence | File and memory stores support resumable sessions, explicit `save()`, `autoSave`, memory recall, and session IDs. |
| Infrastructure | Lane queues are optional and explicit for external/hybrid dispatch; ordinary sessions stay queue-free. |

The current execution shape is:

```text
Agent / AgentSession
  -> context assembly
  -> tool selection
  -> programmatic tool calling when deterministic
  -> permission and confirmation policy
  -> execution
  -> trace, artifacts, verification evidence
  -> compaction
```

## Install

```bash
pip install a3s-code
npm install @a3s-lab/code
```

Rust applications depend on `a3s-code-core`.

## Configure

v2.1.0 uses ACL only. Use `.acl` files and labeled blocks; legacy HCL files are not supported.

```text
default_model = "anthropic/claude-sonnet-4-20250514"

providers "anthropic" {
  apiKey = env("ANTHROPIC_API_KEY")
}
```

OpenAI-compatible services, including MiniMax, use the `openai` provider with `baseUrl`:

```text
default_model = "openai/MiniMax-M2.7-highspeed"

providers "openai" {
  apiKey = env("A3S_OPENAI_API_KEY")
  baseUrl = env("A3S_OPENAI_BASE_URL")

  models "MiniMax-M2.7-highspeed" {
    name = "MiniMax M2.7 Highspeed"
    tool_call = true
  }
}

storage_backend = "file"
```

The release integration scripts also accept `MINIMAX_API_KEY` and `MINIMAX_BASE_URL` as aliases for the OpenAI-compatible environment variables.

## Use

```ts
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.acl');
const session = agent.session('/my-project', {
  builtinSkills: true,
  planningMode: 'enabled',
});

const result = await session.send(
  'Find where authentication errors are handled and summarize the flow',
);

console.log(result.text);
console.log(result.verificationSummaryText);
```

```py
from a3s_code import Agent

agent = Agent.create("agent.acl")
session = agent.session("/my-project", builtin_skills=True, planning_mode="enabled")

result = session.send("Find where authentication errors are handled and summarize the flow")
print(result.text)
```

## Current v2.1.0 Surface

- `.acl` is the supported configuration format.
- `Agent.create()` accepts `.acl` files or inline ACL strings; JSON config is not supported.
- OpenAI-compatible providers use `apiKey`, `baseUrl`, and model blocks; MiniMax release tests are covered by env injection.
- Planning is an explicit `auto` / `enabled` / `disabled` contract; `auto` uses structured pre-analysis instead of keyword gates.
- Run tracking is public through `runs`, `runSnapshot`, `runEvents`, `currentRun`, and run-scoped cancellation.
- PTC moves deterministic tool chains such as search, read, rank, summarize, and verify into host code instead of repeated LLM tool loops.
- Routine delegation uses the `task` and `parallel_task` tools.
- Node/Python expose `delegateTask` / `delegate_task`, `parallelTask` / `parallel_task`, `program`, and `toolDefinitions` / `tool_definitions`.
- Legacy lifecycle control-plane APIs are removed; use streaming events, run replay, and run-scoped cancellation for application state.
- Hooks registered on a parent session propagate to delegated child runs created by `task`.
- AHP 2.4 supports stdio, HTTP, WebSocket, and Unix socket transports; Unix sockets are Unix-target only.
- Sessions are queue-free by default; lane queues are explicit infrastructure for external or hybrid dispatch.
- Permission policy is explicit: rule lists plus a default decision.
- Results expose token counts, verification status, verification summaries, trace events, and artifact references.
