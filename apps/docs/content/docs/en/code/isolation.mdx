# Isolation

Isolation in v2.1.0 is mostly a harness concern: each session binds to one workspace, each delegated child run receives bounded context, and side effects go through permission checks.

## Workspace Boundary

```ts
const session = agent.session('/repo');
```

Relative file, search, shell, and git operations are evaluated from the session workspace. Security providers and hooks can tighten access further.

## Delegated Context

`task` and `parallel_task` isolate child reasoning. The parent receives compact results instead of full transcripts. This avoids prompt pollution and makes evidence review easier.

## Storage Isolation

Use separate memory and session store directories for different products, tenants, or test suites:

```ts
import { FileMemoryStore, FileSessionStore } from '@a3s-lab/code';

const session = agent.session('/repo', {
  memoryStore: new FileMemoryStore('./.a3s/memory'),
  sessionStore: new FileSessionStore('./.a3s/sessions'),
});
```

## External Harness

AHP can enforce organization-level boundaries. Hooks can enforce local process rules. Permission policy keeps final tool execution auditable.
