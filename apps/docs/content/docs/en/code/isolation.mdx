# Isolation Strategies

A3S Code provides four isolation layers to prevent agent operations from affecting files or system resources outside the intended scope.

```
Isolation Layers
  ├─ Workspace Boundary   ← File ops confined to the session directory (always on)
  ├─ File Version History ← Auto-snapshot before writes; diff and restore any version
  ├─ Git Worktree         ← Agent works on an isolated branch; discard with one command
  └─ MicroVM Sandbox      ← bash runs inside an isolated VM (requires sandbox feature)
```

## Workspace Boundary

All file operations (`read`, `write`, `edit`, `glob`, `grep`) are confined to the directory specified when creating the session. Attempts to access paths outside the workspace are intercepted immediately — no retry, no path disclosure to the LLM.

This boundary is **always active** and cannot be disabled through configuration.

## File Version History

Whenever the agent executes `write`, `edit`, or `patch`, A3S Code automatically captures a snapshot of the file before the modification. Snapshots are stored in session memory and support version listing, diff generation, and restore.

### Snapshot Triggers

| Tool | Snapshot Taken |
|------|---------------|
| `write` | Before overwriting |
| `edit` | Before string replacement |
| `patch` | Before applying patch |

### Using File History

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// List all versions of a file
const versions = await session.listFileVersions('src/auth.rs');
for (const v of versions) {
  console.log(`v${v.version} — ${v.toolName} — ${v.timestamp}`);
}

// Generate diff
const diff = await session.diffFileVersions('src/auth.rs', 0, 1);
console.log(diff);

// Restore to version 0
await session.restoreFileVersion('src/auth.rs', 0);
```
</Tab>
<Tab value="Python">
```python
# List all versions of a file
versions = session.list_file_versions("src/auth.rs")
```

</Tab>

## Git Worktree

The `git_worktree` tool lets the agent work on an isolated Git worktree (branch). All agent changes stay on the isolated branch — the main branch is completely unaffected. If an experiment fails, remove the worktree with a single command to discard all changes.

### Workflow

<Steps>
<Step>**Create worktree** — establish an isolated workspace on a new branch</Step>
<Step>**Work in isolation** — all agent file changes are committed to this branch</Step>
<Step>**Review results** — inspect changes, decide to merge or discard</Step>
<Step>**Merge or discard** — if satisfied, merge; if not, `remove` the entire worktree</Step>
</Steps>

### Example

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// Create isolated worktree
await session.callTool('git_worktree', {
  command: 'create',
  branch: 'agent/refactor-auth',
  path: '/tmp/agent-worktrees/refactor-auth',
  newBranch: true,
  base: 'main',
});

// Create session pointing at the worktree
const isolatedSession = agent.session('/tmp/agent-worktrees/refactor-auth');

// All changes on isolated branch
const result = await isolatedSession.send('Refactor auth module to use JWT');

// List all worktrees
const list = await session.callTool('git_worktree', { command: 'list' });

// Discard if not satisfied
await session.callTool('git_worktree', {
  command: 'remove',
  path: '/tmp/agent-worktrees/refactor-auth',
});
```
</Tab>
<Tab value="Python">
```python
# Create isolated worktree
session.call_tool("git_worktree", {
    "command": "create",
    "branch": "agent/refactor-auth",

# Discard entire worktree if not satisfied
session.call_tool("git_worktree", {
    "command": "remove",
    "path": "/tmp/agent-worktrees/refactor-auth",
})
```

## MicroVM Sandbox

With the `sandbox` Cargo feature enabled, `bash` tool commands are routed through an A3S Box MicroVM instead of the host `std::process::Command`. The workspace directory is mounted read-write at `/workspace` inside the VM.

<Callout type="info">
**Requires the `sandbox` Cargo feature.** Add to `Cargo.toml`:
```toml
a3s-code-core = { version = "...", features = ["sandbox"] }
```
</Callout>

### Configuration

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">

</Callout>

## Choosing an Isolation Strategy

| Scenario | Recommended approach |
|----------|---------------------|
| Prevent reading files outside the project | Workspace boundary (always on) |
| Undo agent file writes | File version history |
| Agent works on an experimental branch | Git worktree |
| Isolate system side-effects of bash commands | MicroVM sandbox |
| High-security production environment | Git worktree + MicroVM sandbox |
