# Tools

A3S Code ships with **16 built-in tools** (always available) and **2 plugin tools** (opt-in). All tools execute in-process via native Rust implementations.

<Callout type="info">
`agentic_search` and `agentic_parse` are **plugin tools** — not loaded by default. Mount them via `SessionOptions.plugins`. Each plugin also automatically registers its companion skill into the session's skill registry.
</Callout>

## File Operations

### `read`

Read file contents with line numbers. Supports offset and limit for large files.

```json
{
  "file_path": "src/main.rs",
  "offset": 0,
  "limit": 100
}
```

### `write`

Create or overwrite a file. Automatically creates parent directories.

```json
{
  "file_path": "src/config.rs",
  "content": "pub struct Config {\n    pub port: u16,\n}\n"
}
```

### `edit`

Replace a specific string in a file. Useful for targeted modifications.

```json
{
  "file_path": "src/main.rs",
  "old_string": "fn main() {",
  "new_string": "fn main() -> Result<()> {"
}
```

### `patch`

Apply a unified diff patch. Best for complex multi-line edits.

```json
{
  "file_path": "src/lib.rs",
  "diff": "--- a/src/lib.rs\n+++ b/src/lib.rs\n@@ -1,3 +1,4 @@\n+use anyhow::Result;\n use std::io;\n"
}
```

## Search

### `grep`

Search file contents using ripgrep patterns. Supports regex, globs, and context lines.

```json
{
  "pattern": "fn.*async",
  "path": "src/",
  "glob": "*.rs",
  "context": 2
}
```

### `glob`

Find files matching a glob pattern.

```json
{
  "pattern": "src/**/*.rs"
}
```

### `ls`

List directory contents with file sizes and types.

```json
{
  "path": "src/"
}
```

## Shell

## Plugin Tools

Plugin tools are opt-in — mount them explicitly via `SessionOptions.plugins`. When a plugin loads, it registers both its **tool** (callable by the LLM) and its **companion skill** (injected into the system prompt automatically).

### Mounting Plugins

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
import { Agent, AgenticSearch, AgenticParse } from '@a3s-lab/code';

const session = agent.session('.', {
  plugins: [
    new AgenticSearch(),
    new AgenticParse(),
  ],
});
```
</Tab>

```python
from a3s_code import SessionOptions, AgenticSearch, AgenticParse

opts = SessionOptions()
opts.plugins = [AgenticSearch(), AgenticParse()]
session = agent.session('.', opts)
```

### `agentic_search`

Multi-phase semantic code search. Extracts keywords from natural language, searches in parallel, and ranks results using IDF-weighted scoring.

```json
{
  "query": "how does authentication work",
  "max_results": 10,
  "mode": "fast"
}
```

**Modes:**
- `fast` (default) — keyword extraction + parallel search + relevance ranking
- `deep` — Monte Carlo sampling around high-scoring regions for comprehensive analysis
- `filename_only` — fast file discovery by name

### `agentic_parse`

LLM-enhanced document parsing. Applies structural extraction and optionally an LLM pass for semantic QA.

```json
{
  "path": "report.pdf",
  "query": "What are the key findings?",
  "strategy": "auto",
  "max_chars": 8000
}
```

**Parse strategies:** `auto` (inferred from extension/content) · `structured` (JSON/YAML/TOML/HCL) · `narrative` (Markdown/text) · `tabular` (CSV/TSV) · `code` (source files)

Omit `query` for a fast structural overview without LLM cost.

### Supported Formats

`agentic_parse` supports plain-text formats out of the box: source code, Markdown, JSON, TOML, YAML, HCL, CSV, and more. Binary formats (PDF, Word, Excel) require a custom `DocumentParser` implementation in Rust — not currently exposed in the TypeScript or Python SDKs.

## Direct Tool Execution

Call tools directly on the session without going through the LLM:

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// Convenience wrappers
const content = await session.readFile('src/main.rs');
const output = await session.bash('cargo test --lib');
const files = await session.glob('**/*.rs');
const matches = await session.grep('fn main');

// Generic tool call
const result = await session.tool('edit', {
  file_path: 'src/main.rs',
  old_string: 'fn main() {',
  new_string: 'fn main() -> Result<()> {',
});
console.log(`[${result.exitCode}] ${result.output}`);
```
</Tab>
<Tab value="Python">
```python
# Convenience wrappers
content = session.read_file("src/main.rs")
output = session.bash("cargo test --lib")
files = session.glob("**/*.rs")
matches = session.grep("fn main")

# Generic tool call with dict args
result = session.tool("edit", {
    "file_path": "src/main.rs",
    "old_string": "fn main() {",
    "new_string": "fn main() -> Result<()> {"
})
print(f"[{result['exit_code']}] {result['output']}")
```
</Tab>
</Tabs>

## Git

### `git_worktree`

Manage Git worktrees for parallel workspace isolation. Supports `status`, `create`, `list`, and `remove` subcommands.

```json
{
  "command": "create",
  "branch": "feature-auth",
  "path": "/repo/wt-feature-auth"
}
```

## Subagent Delegation

### `task`

Delegate work to a specialized child agent. The child agent runs in its own session with its own conversation history. See [Subagents](/docs/code/subagents) for details.

```json
{
  "description": "Write unit tests for the auth module",
  "prompt": "Create comprehensive tests for src/auth.rs covering all edge cases"
}
```

### `parallel_task`

Delegate multiple tasks to subagents concurrently. Each task runs in parallel with its own agent instance.

```json
{
  "tasks": [
    {
      "agent": "general",
      "prompt": "Analyze security in auth module"
    },
    {
      "agent": "general",
      "prompt": "Review performance in database layer"
    }
  ]
}
```

### `run_team`

Coordinate a Lead → Worker → Reviewer team to tackle goals that require decomposition, execution, and quality review. The Lead breaks the goal into a task list; Workers claim and execute tasks in parallel; the Reviewer approves or rejects each result.

```json
{
  "goal": "Audit the authentication module for security issues",
  "lead_agent": "general",
  "worker_agent": "general",
  "reviewer_agent": "general",
  "max_steps": 10
}
```

All agent type fields default to `"general"`. Only `goal` is required.

### `Skill`

Invoke skills as first-class tools with skill-based tool access control. When a skill is invoked, the agent temporarily gains access to the skill's allowed tools and receives the skill's instruction prompt.

```json
{
  "skill_name": "code-review",
  "args": "Review src/auth.rs for security issues"
}
```

This enforces skill-based access patterns and prevents agents from bypassing skills to directly access underlying tools. See [Skills](/docs/code/skills) for details.

## Sandbox Routing (Optional)

There is no standalone `sandbox` tool. Instead, `bash` transparently routes through an A3S Box MicroVM when the session has a `BashSandbox` handle configured:

```rust
SessionOptions::new().with_sandbox_handle(Arc::new(my_box_impl))
```

The workspace is mounted at `/workspace` inside the VM. See [Isolation → MicroVM Sandbox](/docs/code/isolation) for `SandboxConfig` fields and the full `BashSandbox` trait definition.

## File Version History

All file-modifying tools (`write`, `edit`, `patch`) automatically capture a snapshot before making changes. You can:

- View the diff between versions
- Restore any previous version
- Track all modifications made during a session

The file history maintains up to 500 snapshots per session.

## Extending with Custom Tools

Beyond built-in tools, A3S Code supports external tool integration via:

- **[MCP](/docs/code/mcp)** — Connect external tool servers via Model Context Protocol (JSON-RPC 2.0, stdio + HTTP+SSE)
- **Custom Tool Trait** — Implement the `Tool` trait in Rust and register with `ToolRegistry`

See [MCP](/docs/code/mcp) for external tool integration examples.
