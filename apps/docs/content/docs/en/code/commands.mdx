# Commands

A3S Code is primarily SDK-driven in v2.1.0. Product CLIs and UIs usually map commands to session APIs rather than depending on legacy public lifecycle commands.

## Recommended Mappings

| User action | Session API |
| --- | --- |
| Send a prompt | `session.send(prompt)` |
| Stream a prompt | `session.stream(prompt)` |
| Ask a side question | `session.btw(question)` |
| Run a deterministic tool | `session.tool(name, args)` |
| Read history | `session.history()` |
| Save state | `session.save()` |
| Resume state | `agent.resumeSession(id, options)` |
| Inspect tools | `session.toolNames()` / `session.toolDefinitions()` |
| Verify work | `session.verifyCommands(subject, commands)` |
| Replay run state | `session.runs()` / `session.runEvents(runId)` |
| Cancel active work | `session.currentRun()` / `session.cancelRun(runId)` |

## Building Slash Commands

If your app exposes slash commands, keep them thin:

```ts
if (input === '/tools') console.log(session.toolNames());
if (input === '/tool-schemas') console.log(session.toolDefinitions());
if (input === '/verify') {
  await session.verifyCommands('current task', session.verificationPresets());
}
```

Avoid hidden loops or background schedulers. Make recurring automation explicit in your host application so it can be audited and cancelled.
