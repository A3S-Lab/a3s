# AGENTS.md

`AGENTS.md` files give the harness stable project instructions without forcing every prompt to repeat them. A3S Code reads them as context and applies the closest relevant instructions to the workspace.

## What To Put In AGENTS.md

```md
# Project Instructions

- Use `cargo test -p a3s-code-core` for core changes.
- Never commit secrets from `.a3s/config.acl`.
- Prefer `rg` for search.
- Release checks must include package metadata and provider smoke tests.
```

Keep instructions operational: build commands, safety rules, code style, directories, and release practices.

## Precedence

Session options, explicit user instructions, skill constraints, security policy, and AGENTS.md are composed by the harness. For side effects, the effective permission/security decision still wins.

## Good Practice

- Keep project instructions short and testable.
- Put secrets in environment variables, not in AGENTS.md.
- Use nested AGENTS.md files only when subdirectories truly differ.
- Update AGENTS.md when release or test commands change.
