# Tasks

In v2.1.0 the routine multi-agent path is the `task` and `parallel_task` tools. They replace older public team runner APIs and keep child context isolated from the parent conversation.

## Single Task

Ask the parent agent to delegate a bounded job:

```text
Use task to ask an explore agent to inspect the auth module.
Return files inspected, findings, risks, and confidence.
```

A child agent should return a compact contract:

- summary
- files inspected or changed
- evidence references
- risks and unknowns
- confidence
- trace reference

The parent should not ingest the full child transcript.

If the host already knows the task boundary, call the same core tool directly:

```ts
await session.delegateTask({
  agent: 'explore',
  description: 'Inspect auth module',
  prompt: 'Return files inspected, findings, risks, and confidence.',
});
```

## Parallel Task

Use `parallel_task` when independent work can run concurrently:

```text
Run parallel_task with three lanes:
1. inspect provider config parsing
2. inspect Node SDK declarations
3. inspect release scripts

Merge the results into one release-readiness report.
```

```ts
await session.parallelTask([
  { agent: 'explore', description: 'Inspect config', prompt: 'Check provider parsing.' },
  { agent: 'verification', description: 'Verify SDK', prompt: 'Check SDK declarations.' },
]);
```

## Agent Directories

Load custom agent definitions through `agentDirs` or register them on a live session:

```ts
const session = agent.session('/repo', { agentDirs: ['./agents'] });
const loaded = session.registerAgentDir('./more-agents');
```

Legacy lifecycle control-plane APIs are removed. Applications that need UI state should consume streaming events, run replay, and `cancelRun(runId)` / `cancel_run(run_id)`.
