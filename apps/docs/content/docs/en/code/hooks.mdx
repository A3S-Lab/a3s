# Lifecycle Hooks

A3S Code provides 11 lifecycle hooks for intercepting and reacting to events during agent execution. Hooks enable logging, security enforcement, telemetry, and custom behavior without modifying the core agent loop.

## How Hooks Work

```
Agent Loop
  ├── GenerateStart hooks fire
  ├── LLM call
  ├── GenerateEnd hooks fire
  └── For each tool_use:
      ├── PreToolUse hooks fire
      ├── Permission check
      ├── HITL check
      ├── Tool execution
      └── PostToolUse hooks fire
```

## Sentinel Hook System

The **sentinel hook** is the mechanism that automatically propagates the parent session's `HookEngine` to every sub-agent spawned by `task` and `parallel_task`, at arbitrary nesting depth — with no manual re-registration required.

### How propagation works

```
Parent session
  ├── register_hook("guard", PreToolUse)    ← registered on HookEngine
  ├── register_hook_handler("guard", ...)
  │
  └── task tool spawns Child AgentLoop
        ├── AgentConfig.hook_engine = sentinel_hook   ← inherited
        ├── ToolContext.sentinel_hook = sentinel_hook  ← set for grandchildren
        │
        └── task tool spawns Grandchild AgentLoop
              ├── AgentConfig.hook_engine = sentinel_hook  ← inherited again
              └── ToolContext.sentinel_hook = sentinel_hook ← depth-unlimited
```

**Implementation path** (`core/src/`):

1. `agent_api.rs` — when a session is built, if `hook_engine` exists it is written into `ToolContext.sentinel_hook`
2. `tools/task.rs` — `TaskTool::execute()` reads `ctx.sentinel_hook` and sets it as `child AgentConfig.hook_engine`
3. `tools/task.rs` — the child's own `ToolContext` also receives `sentinel_hook`, so any grandchild task tool will repeat the same propagation

<Callout type="info">
  **Security implication**: A `PreToolUse` blocking hook registered once on the parent enforces the same policy across every sub-agent at every depth. This is the recommended pattern for multi-agent security enforcement.
</Callout>

### Using sentinel hooks

<Tabs items={['Rust', 'TypeScript', 'Python']}>
<Tab value="Rust">

Rust gives full control: register the hook definition, then register a `HookHandler` that can `block`, `modify`, or `continue` the operation. The handler automatically applies to all sub-agents.

```rust
use a3s_code_core::{
    Agent, SessionOptions,
    hooks::{Hook, HookConfig, HookEvent, HookEventType, HookHandler, HookMatcher, HookResponse},
};
use std::sync::Arc;

// 1. Define a blocking handler
struct DangerousCommandGuard;

impl HookHandler for DangerousCommandGuard {
    fn handle(&self, event: &HookEvent) -> HookResponse {
        let HookEvent::PreToolUse(e) = event else {
            return HookResponse::continue_();
        };
        let cmd = e.args.to_string();
        if cmd.contains("rm -rf") || cmd.contains("DROP TABLE") {
            return HookResponse::block(format!(
                "Blocked by sentinel: dangerous pattern in '{}'", e.tool
            ));
        }
        HookResponse::continue_()
    }
}

// 2. Create session and register sentinel hook once on the parent
let agent = Agent::new("agent.hcl").await?;
let session = agent.session(".", Some(
    SessionOptions::new()
        .with_builtin_skills()
        .with_permissive_policy()
))?;

session.register_hook(
    Hook::new("sentinel-guard", HookEventType::PreToolUse)
        .with_matcher(HookMatcher::tool("bash"))
        .with_config(HookConfig { priority: 2, ..Default::default() })
);
session.register_hook_handler("sentinel-guard", Arc::new(DangerousCommandGuard));

// 3. Any task-spawned sub-agent automatically inherits the sentinel
let result = session.send(
    "Analyze the repo with multiple sub-agents and summarize findings",
    None,
).await?;

println!("{}", result.text);
```

To add a logging-only hook alongside the blocking hook:

```rust
struct AuditLogger;

impl HookHandler for AuditLogger {
    fn handle(&self, event: &HookEvent) -> HookResponse {
        if let HookEvent::PostToolUse(e) = event {
            println!("[audit] tool={} exit={}", e.tool, e.result.exit_code);
        }
        HookResponse::continue_()
    }
}

session.register_hook(
    Hook::new("sentinel-audit", HookEventType::PostToolUse)
        .with_config(HookConfig { priority: 10, ..Default::default() })
);
session.register_hook_handler("sentinel-audit", Arc::new(AuditLogger));
```

</Tab>
<Tab value="TypeScript">

TypeScript supports hook registration (metadata + matcher + priority). Hooks fire inside Rust for every tool call in the parent **and all sub-agents**. Observe parent-level tool activity through `tool_start` / `tool_end` stream events.

```typescript
import { Agent } from '@a3s-lab/code';

const agent = await Agent.create('agent.hcl');
const session = agent.session('/project', {
  builtinSkills: true,
  permissive: true,
});

// Register sentinel hooks — priority 2 runs before standard hooks (10+)
session.registerHook('sentinel-guard', 'pre_tool_use', null, { priority: 2 });
session.registerHook('sentinel-audit', 'post_tool_use', null, { priority: 10 });
session.registerHook('sentinel-gen', 'generate_start', null, { priority: 10 });

console.log(`Sentinel hooks registered: ${session.hookCount()}`);
// → 3

// Hooks propagate into every sub-agent spawned by the task tool.
// Observe parent-level tool calls via the stream:
const stream = await session.stream(
  'Analyze the repo with multiple sub-agents and summarize findings'
);

while (true) {
  const { value: event, done } = await stream.next();
  if (done || !event) break;

  if (event.type === 'tool_start') {
    console.log(`[sentinel] tool called: ${event.toolName}`);
  } else if (event.type === 'tool_end') {
    console.log(`[sentinel] tool done:   ${event.toolName} exit=${event.exitCode}`);
  } else if (event.type === 'end') {
    console.log(`Done. Tokens: ${event.totalTokens}`);
    break;
  }
}

// Clean up when the session is no longer needed
session.unregisterHook('sentinel-guard');
session.unregisterHook('sentinel-audit');
session.unregisterHook('sentinel-gen');
```

Filter the sentinel to a specific tool with a matcher:

```typescript
// Only intercept bash calls
session.registerHook('sentinel-bash', 'pre_tool_use', { tool: 'bash' }, { priority: 2 });
```

<Callout type="info">
  **Blocking from TypeScript**: TypeScript and Python support hook registration only. To implement blocking or modifying logic, implement a `HookHandler` in Rust.
</Callout>

</Tab>

```python
    elif event.event_type == 'tool_end':

    elif event.event_type == 'end':
        print(f"Done. Tokens: {event.total_tokens}")
        break

# Clean up when the session is no longer needed
session.unregister_hook('sentinel-guard')
session.unregister_hook('sentinel-audit')
session.unregister_hook('sentinel-gen')
```

```python

                      matcher={'tool': 'bash'}, config={'priority': 2})
```

## Hook Registration

<Tabs items={['Rust', 'TypeScript', 'Python']}>
<Tab value="Rust">

```rust
use a3s_code_core::hooks::{
    Hook, HookConfig, HookEngine, HookEvent, HookEventType,
    HookHandler, HookResult, HookResponse, HookMatcher,
};
use std::sync::Arc;

let engine = HookEngine::new();

// 1. Register a hook definition (what to listen for)
engine.register(
    Hook::new("log-tools", HookEventType::PreToolUse)
        .with_config(HookConfig { priority: 10, ..Default::default() })
);

// 2. Register a handler (what to do when fired)
struct LogHandler;
impl HookHandler for LogHandler {
    fn handle(&self, event: &HookEvent) -> HookResponse {
        if let HookEvent::PreToolUse(e) = event {
            println!("Tool: {} Args: {:?}", e.tool, e.args);
        }
        HookResponse::continue_()
    }
}
engine.register_handler("log-tools", Arc::new(LogHandler));

// Block dangerous patterns
engine.register(
    Hook::new("block-dangerous", HookEventType::PreToolUse)
        .with_matcher(HookMatcher::tool("bash"))
        .with_config(HookConfig { priority: 5, ..Default::default() })
);

struct BlockHandler;
impl HookHandler for BlockHandler {
    fn handle(&self, event: &HookEvent) -> HookResponse {
        if let HookEvent::PreToolUse(e) = event {
            if e.args.to_string().contains("rm -rf") {
                return HookResponse::block("Dangerous command blocked");
            }
        }
        HookResponse::continue_()
    }
}
engine.register_handler("block-dangerous", Arc::new(BlockHandler));

// Fire a hook (done automatically by the agent loop)
let result = engine.fire(&event).await;
match result {
    HookResult::Continue(None) => { /* proceed */ }
    HookResult::Continue(Some(modified)) => { /* proceed with modified data */ }
    HookResult::Block(reason) => { /* stop execution */ }
    _ => {}
}
```

## Hook Priority

Hooks execute in priority order. Lower numbers run first.

<TypeTable
  type={{
    "1": {
      description: "SecurityGuard (built-in, always runs first)",
    },
    "2-9": {
      description: "Critical custom hooks (validation, blocking)",
    },
    "10-49": {
      description: "Standard hooks (logging, telemetry)",
    },
    "50+": {
      description: "Low-priority hooks (analytics, optional processing)",
    },
  }}
/>

If any hook returns `HookResult::Block`, the operation is cancelled and subsequent hooks for that event are skipped.
