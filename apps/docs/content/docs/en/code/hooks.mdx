# Hooks

Hooks observe or intercept lifecycle events inside a session. They are useful for policy enforcement, audit logs, metrics, and release gates.

## Events

Supported event types are:

```text
pre_tool_use
post_tool_use
generate_start
generate_end
session_start
session_end
skill_load
skill_unload
pre_prompt
post_response
on_error
```

## Blocking Example

```ts
session.registerHook(
  'release-publish-approval',
  'pre_tool_use',
  { tool: 'bash', commandPattern: 'npm publish|twine upload|cargo publish' },
  { priority: 50, timeoutMs: 1000 },
  event => ({
    action: 'block',
    reason: 'publishing must be performed by release automation',
  }),
);
```

A handler returns `{ action: 'continue' }`, `{ action: 'skip' }`, `{ action: 'block', reason }`, or `null`/`undefined` to continue.

## Propagation

Hooks registered on a parent session automatically propagate to delegated child runs created by `task`, including deeper descendants. This keeps security and audit logic consistent across a delegation tree.

## Management

```ts
console.log(session.hookCount());
session.unregisterHook('release-publish-approval');
```
