# Planning

```ts
const session = agent.session('/repo', {
  planningMode: 'enabled',
  goalTracking: true,
  maxToolRounds: 24,
});

const result = await session.send('Plan and complete the release-readiness review');
console.log(result.text);
```

`planningMode: 'auto'` is the default structured pre-analysis path,
`'enabled'` forces planning, and `'disabled'` turns it off for low-latency
requests. Planning state is attached to the run so a host UI can render a
TaskList and update task completion from run events.

Verification commands still provide the completion evidence.
