# Agent Orchestrator

Agent Orchestrator is a main-sub agent coordinator built on a3s-event, providing a unified event bus for real-time monitoring of all sub-agent behaviors, planning, and execution, with dynamic control capabilities.

<Callout type="info">
  **Version Requirement**: v1.1.0+
</Callout>

## Core Features

- **Real AgentLoop Execution** - SubAgents run real LLM + tool loops via `create_with_agent()` ⭐ New
- **Real-time Monitoring** - Monitor all SubAgent lifecycles, state changes, progress, and tool executions
- **Task List Query** - View all SubAgent task lists and current activities in real-time
- **Activity Tracking** - Track each SubAgent's current activity (calling tools, requesting LLM, waiting for control)
- **Dynamic Control** - Pause, resume, cancel SubAgents, adjust parameters, inject new prompts
- **Event-Driven** - Unified event bus based on a3s-event
- **Pluggable Communication** - Default memory-based events, supports custom NATS provider
- **Concurrency Management** - Control maximum concurrent SubAgent count

## Architecture

```text
AgentOrchestrator (Main Agent)
    ↓ Subscribe Events
┌───────────────────────────────┐
│   a3s-event EventBus          │
│ (Memory / NATS / Custom)      │
└───────────────────────────────┘
    ↑ Publish Events    ↓ Control Signals
SubAgentWrapper (Sub Agents)
```

## Quick Start

<Tabs items={['Rust', 'Python']}>
<Tab value="Rust">

```rust
use a3s_code_core::orchestrator::{AgentOrchestrator, SubAgentConfig};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    // Create orchestrator (default memory provider)
    let orchestrator = AgentOrchestrator::new_memory();

    // Subscribe to all events
    let mut events = orchestrator.subscribe_all();

    // Spawn a SubAgent
    let config = SubAgentConfig::new("general", "Use glob to find Python files")
        .with_description("Find Python files")
        .with_permissive(true)
        .with_max_steps(10);

    let handle = orchestrator.spawn_subagent(config).await?;

    // Monitor events
    tokio::spawn(async move {
        while let Ok(event) = events.recv().await {
            println!("Event: {:?}", event.event_name());
        }
    });

    // Control SubAgent
    tokio::time::sleep(tokio::time::Duration::from_secs(2)).await;
    handle.pause().await?;

    tokio::time::sleep(tokio::time::Duration::from_secs(1)).await;
    handle.resume().await?;

    // Wait for completion
    let result = handle.wait().await?;
    println!("Result: {}", result);

    Ok(())
}
```

## Real Execution ⭐ New

By default, `Orchestrator.create()` spawns subagents using an internal stub — useful for tests
and coordination demos. To run **real LLM + tool execution**, use `create_with_agent()` (Python /
TypeScript) or `with_execution_context()` (Rust). This wires the orchestrator to your agent's LLM
client, agent registry, and workspace so every spawned subagent runs a full AgentLoop.

<Tabs items={['Python', 'TypeScript', 'Rust']}>
<Tab value="Python">

```python
from a3s_code import Agent, Orchestrator, SubAgentConfig

# 1. Load your agent config (any provider — Kimi, Claude, OpenAI, …)
agent = Agent.create("agent.hcl")

# 2. Wire the orchestrator to real AgentLoop execution
orch = Orchestrator.create_with_agent(agent, workspace=".")

# 3. Spawn and run — uses real LLM + tool calls
handle = orch.spawn_subagent(SubAgentConfig(
    agent_type="general",
    prompt="Use glob to find all Python files. Report how many you found.",
    permissive=True,
    max_steps=5,
))

result = handle.wait()
print(result)
```

use a3s_code_core::{agent_api::Agent, orchestrator::AgentOrchestrator};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let agent = Agent::new("agent.hcl").await?;

    // Extract LLM client + registry from the agent
    let ctx = agent.make_execution_context(".");

    // Wire the orchestrator
    let orch = AgentOrchestrator::new_memory()
        .with_execution_context(ctx.llm_client, ctx.registry, ctx.workspace);

    let handle = orch.spawn_subagent(
        SubAgentConfig::new("general", "Find all Rust source files.")
            .with_permissive(true)
            .with_max_steps(5),
    ).await?;

    let result = handle.wait().await?;
    println!("{result}");
    Ok(())
}

### Execution Mode Comparison

| Mode | API | Subagent runs |
|------|-----|---------------|
| Stub (default) | `Orchestrator.create()` | Internal placeholder — for tests & demos |
| Real execution | `Orchestrator.create_with_agent(agent, ws)` | Full AgentLoop with real LLM + tools |

### Architecture with Real Execution

```text
AgentOrchestrator
    └── SubAgentWrapper  (ExecutionContext attached)
            ├── AgentRegistry   — looks up agent type definition
            ├── ToolExecutor    — built-in tools only (no Task tool to prevent nesting)
            └── AgentLoop       — real LLM turns + tool execution
                    │ events
                    ▼
        OrchestratorEvent::SubAgentInternalEvent { id, event }
```

<Callout type="info">
  **Timeout support**: Set `timeout_ms` in `SubAgentConfig` to abort long-running subagents
  automatically. The spawned `AgentLoop` task is cancelled and an error is returned.
</Callout>

<Callout type="info">
  **Pause behavior**: Pause marks the state as `Paused` and stops accepting new control signals,
  but the underlying `AgentLoop` continues until the current LLM turn finishes. Cancel aborts the
  task immediately.
</Callout>

<Callout type="info">
  **`duration_ms` in `SubAgentCompleted`**: Reports real elapsed wall-clock time from spawn to
  completion, not a hardcoded zero.
</Callout>

## Event Types

The Orchestrator publishes the following event types:

| Event | Description |
|-------|-------------|
| `SubAgentStarted` | SubAgent started |
| `SubAgentCompleted` | SubAgent completed |
| `SubAgentStateChanged` | State changed |
| `SubAgentProgress` | Progress update |
| `SubAgentInternalEvent` | Internal event (from AgentLoop) |
| `PlanningStarted` | Planning started |
| `PlanningCompleted` | Planning completed |
| `ToolExecutionStarted` | Tool execution started |
| `ToolExecutionCompleted` | Tool execution completed |
| `ControlSignalReceived` | Control signal received |
| `ControlSignalApplied` | Control signal applied |

## SubAgent States

SubAgents go through the following states during execution:

```rust
pub enum SubAgentState {
    Initializing,                    // Initializing
    Running,                         // Running
    Paused,                          // Paused
    Completed { success, output },   // Completed
    Cancelled,                       // Cancelled
    Error { message },               // Error
}
```

## Real-time Monitoring API ⭐ v1.1.0 New

The main agent can view all SubAgent task lists and current activities in real-time.

<Tabs items={['Rust', 'Python', 'TypeScript']}>
<Tab value="Rust">

```rust
// Get all SubAgent information list
let subagents = orchestrator.list_subagents().await;
for info in subagents {
    println!("SubAgent: {}", info.id);
    println!("  Type: {}", info.agent_type);
    println!("  State: {}", info.state);
    println!("  Description: {}", info.description);

    if let Some(activity) = info.current_activity {
        println!("  Current Activity: {:?}", activity);
    }
}

// Get specific SubAgent details
if let Some(info) = orchestrator.get_subagent_info("subagent-1").await {
    println!("Created at: {}", info.created_at);
    println!("Updated at: {}", info.updated_at);
}

// Get all active SubAgent activities
let activities = orchestrator.get_active_activities().await;
for (id, activity) in activities {
    match activity {
        SubAgentActivity::CallingTool { tool_name, args } => {
            println!("{} is calling tool: {}", id, tool_name);
        }
        SubAgentActivity::RequestingLlm { message_count } => {
            println!("{} is requesting LLM ({} messages)", id, message_count);
        }
        SubAgentActivity::WaitingForControl { reason } => {
            println!("{} is waiting: {}", id, reason);
        }
        SubAgentActivity::Idle => {
            println!("{} is idle", id);
        }
    }
}

// Get all SubAgent states
let states = orchestrator.get_all_states().await;
for (id, state) in states {
    println!("{}: {:?}", id, state);
}

// Get active count
let count = orchestrator.active_count().await;
println!("Active SubAgents: {}", count);
```

### SubAgent Activity Types

SubAgents have the following activity types during execution:

| Activity Type | Description | Data |
|--------------|-------------|------|
| `Idle` | Idle state | - |
| `CallingTool` | Calling a tool | Tool name and arguments |
| `RequestingLlm` | Requesting LLM | Message count |
| `WaitingForControl` | Waiting for control signal | Reason |

### Monitoring API List

| API | Description |
|-----|-------------|
| `list_subagents()` | Get all SubAgent information list |
| `get_subagent_info(id)` | Get specific SubAgent details |
| `get_active_activities()` | Get all active SubAgent activities |
| `get_all_states()` | Get all SubAgent states |
| `active_count()` | Get active SubAgent count |
| `pause_subagent(id)` | Pause specified SubAgent |
| `resume_subagent(id)` | Resume specified SubAgent |
| `cancel_subagent(id)` | Cancel specified SubAgent |
| `wait_all()` | Wait for all SubAgents to complete |

## Configuration Options

### OrchestratorConfig

```rust
pub struct OrchestratorConfig {
    /// Event channel buffer size
    pub event_buffer_size: usize,           // Default: 1000

    /// Control signal channel buffer size
    pub control_buffer_size: usize,         // Default: 100

    /// Maximum concurrent SubAgents
    pub max_concurrent_subagents: usize,    // Default: 50

    /// Event subject prefix
    pub subject_prefix: String,             // Default: "agent"
}
```

### SubAgentConfig

```rust
pub struct SubAgentConfig {
    /// Agent type (general, explore, plan, etc.)
    pub agent_type: String,

    /// Task description
    pub description: String,

    /// Execution prompt
    pub prompt: String,

    /// Enable permissive mode (bypass HITL)
    pub permissive: bool,

    /// Maximum execution steps
    pub max_steps: Option<usize>,

    /// Execution timeout (milliseconds)
    pub timeout_ms: Option<u64>,

    /// Parent SubAgent ID (for nesting)
    pub parent_id: Option<String>,

    /// Custom metadata
    pub metadata: serde_json::Value,
}
```

## Advanced Usage

### Subscribe to Specific SubAgent Events

```rust
let mut events = orchestrator.subscribe_subagent(&handle.id);

while let Ok(event) = events.recv().await {
    match event {
        OrchestratorEvent::SubAgentProgress { step, total_steps, message, .. } => {
            println!("Progress: {}/{} - {}", step, total_steps, message);
        }
        OrchestratorEvent::ToolExecutionStarted { tool_name, .. } => {
            println!("Tool started: {}", tool_name);
        }
        _ => {}
    }
}
```

### Concurrency Limits

```rust
let mut config = OrchestratorConfig::default();
config.max_concurrent_subagents = 5;

let orchestrator = AgentOrchestrator::new(config);

// Attempting to spawn 6th SubAgent will fail
for i in 0..6 {
    let result = orchestrator.spawn_subagent(config.clone()).await;
    if result.is_err() {
        println!("Max concurrent limit reached");
    }
}
```

### Nested SubAgents

```rust
// Parent SubAgent
let parent_config = SubAgentConfig::new("general", "Main task");
let parent_handle = orchestrator.spawn_subagent(parent_config).await?;

// Child SubAgent
let child_config = SubAgentConfig::new("explore", "Subtask")
    .with_parent_id(&parent_handle.id);
let child_handle = orchestrator.spawn_subagent(child_config).await?;
```

## Best Practices

1. **Use `create_with_agent` for production** - Always wire a real agent for actual LLM execution; `create()` alone only runs stubs
2. **Set Reasonable Concurrency Limits** - Configure `max_concurrent_subagents` based on system resources
3. **Use Event Filtering** - Subscribe to specific SubAgent events to reduce noise
4. **Handle Error States** - Listen to `SubAgentStateChanged` events and handle errors
5. **Set Timeouts** - Configure `timeout_ms` for long-running tasks; the underlying task is aborted automatically on timeout
6. **Clean Up Resources** - Use `cancel()` to clean up unneeded SubAgents
