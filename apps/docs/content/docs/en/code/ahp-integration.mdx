# AHP Integration

A3S Code integrates with AHP 2.4 so an external harness can observe and guide the agent loop. AHP is for supervision: policy, context injection, idle decisions, audit, and organization-level safety.

## Event Surface

AHP events include handshake, session start/end, pre-action, post-action, pre-prompt, post-response, error, query, heartbeat, and idle.

AHP decisions are advisory inputs to the same harness path as local policy. They
can add context, request confirmation, deny or allow an action, influence idle
handling, and attach audit metadata, but they do not bypass context budgeting or
the final effective permission decision.

## Transports

Node sessions accept AHP transport objects:

```ts
import {
  HttpTransport,
  StdioTransport,
  WebSocketTransport,
  UnixSocketTransport,
} from '@a3s-lab/code';

const session = agent.session('/repo', {
  ahpTransport: new HttpTransport('http://localhost:8080/ahp', 'token'),
});
```

Available transports:

- `StdioTransport(program, args)`
- `HttpTransport(url, authToken)`
- `WebSocketTransport(url, authToken)`
- `UnixSocketTransport(path)` on Unix targets only

Unix sockets are not available on Windows targets.

## ACL

```text
ahp = {
  enabled = true
  url     = "http://harness:8080/ahp"
  idle_ms = 10000
}
```

## Recommended Use

Keep AHP decisions small and explicit. Let the coding agent continue doing local reasoning, while the harness answers safety, context, approval, and idle-control questions.
