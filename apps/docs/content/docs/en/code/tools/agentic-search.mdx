# Agentic Search Tool

A3S Code includes the `agentic_search` tool, a Sirchmunk-inspired intelligent code search tool that implements a multi-phase search pipeline.

<Callout type="info">
Unlike traditional `grep`, `agentic_search` accepts natural language queries, automatically extracts keywords, and ranks results using IDF-weighted relevance scoring.
</Callout>

## Core Features

- **Natural language queries** — Input "JWT token validation" instead of regex
- **Automatic keyword extraction** — Removes stop words, extracts multi-granularity keywords
- **Parallel search** — Searches file contents and structure simultaneously
- **Relevance ranking** — IDF-weighted scoring, files matching more keywords rank higher
- **File type awareness** — Code files get 1.2x boost, config 1.0x, docs 0.9x
- **Monte Carlo sampling** — DEEP mode uses Gaussian importance sampling for comprehensive analysis

## Document Parsing

By default, `agentic_search` only reads plain text files. To enable parsing of binary documents (PDF, DOCX, etc.), configure a `DocumentParserRegistry` in `SessionOptions`.

<Callout type="info">
Document parsing is optional — the tool works out of the box for code search without any configuration.
</Callout>

## Search Pipeline

### FAST Mode (Default)

```
Phase 1: Keyword Extraction
  ↓ Remove stop words, extract 2-3 word phrases and individual words

Phase 2: Parallel Search
  ├─ Content search: ripgrep-style regex matching
  └─ Structure search: filename and path matching

Phase 3: Relevance Scoring
  ↓ relevance = (match_count / sqrt(file_lines)) * idf_boost * file_type_boost

Phase 4: Result Formatting
  ↓ Return matching lines with context
```

### DEEP Mode (Monte Carlo Sampling)

```
Phase 1: Initial Search
  ↓ Get 2x max_results as sampling pool

Phase 2: Monte Carlo Evidence Sampling
  ├─ Act 1: Fuzzy anchoring — identify high-scoring regions
  ├─ Act 2: Gaussian importance sampling — sample around anchors (decaying sigma)
  └─ Act 3: Top-K selection — return highest evidence score regions

Phase 3: Evidence Synthesis
  ↓ Organize sampled results into coherent evidence regions

Phase 4: Enhanced Formatting
  ↓ Display evidence scores and Gaussian weights
```

### Search Modes

#### FAST Mode (Default)

Most common mode, returns results in 2-5 seconds:

```typescript
agentic_search({
  query: "how does authentication work",
  max_results: 5,
  context_lines: 2
})
```

#### DEEP Mode (Monte Carlo Sampling)

Comprehensive analysis using Gaussian importance sampling, 10-30 seconds:

```typescript
agentic_search({
  query: "explain the entire authentication flow",
  mode: "deep",
  max_results: 3
})
```

**DEEP mode features:**
- Initial search gets 2x results as sampling pool
- Gaussian distribution samples around high-scoring matches
- sigma adapts based on match density (5.0 → 2.0)
- Evidence score: `relevance * sum(weights) / sqrt(sampled_lines)`
- Organizes sampled lines into coherent evidence regions

#### FILENAME_ONLY Mode

Quick file discovery, searches filenames only:

```typescript
agentic_search({
  query: "auth",
  mode: "filename_only",
  max_results: 20
})
```

## Relevance Scoring Algorithm

```rust
// Base score: match count / sqrt(total lines)
let base_score = (match_count as f32) / (total_lines as f32).sqrt();

// IDF boost: unique keywords matched / total keywords
let idf_boost = (unique_keywords_matched as f32) / (total_keywords as f32);

// File type boost
let type_boost = match file_type {
    Code => 1.2,
    Config => 1.0,
    Documentation => 0.9,
    Other => 0.8,
};

// Final relevance
let relevance = base_score * idf_boost * type_boost;
```

This ensures:
- Files matching more keywords rank higher
- Smaller, focused files rank higher than large files
- Code files are prioritized over documentation

## Comparison: agentic_search vs grep

| Feature | agentic_search | grep |
|---------|----------------|------|
| Input | Natural language | Regex pattern |
| Keyword extraction | Automatic | Manual |
| Relevance ranking | Yes (IDF-weighted) | No (file order) |
| File type awareness | Yes | No |
| Stop word filtering | Yes | No |
| Best for | Exploratory search | Precise pattern matching |

**Usage recommendation:**
- Use `agentic_search` for initial discovery
- Use `grep` for precise follow-up searches

## Best Practices

### 1. Use Natural Language

```typescript
// ✅ Good
agentic_search({ query: "how does authentication work" })

// ❌ Bad
agentic_search({ query: "auth" })
```

### 2. Be Specific

```typescript
// ✅ Good
agentic_search({ query: "JWT token validation logic" })

// ❌ Bad
agentic_search({ query: "tokens" })
```

### 3. Combine with Other Tools

```typescript
// 1. Use agentic_search to discover relevant files
const search = await agentic_search({ query: "authentication" });

// 2. Use read to examine the most relevant files
const content = await read({ path: "src/auth.rs" });

// 3. Use grep for precise searches within discovered files
const matches = await grep({
  pattern: "verify_token",
  path: "src/auth.rs"
});
```

### 4. Adjust Context Lines

```typescript
// Overview: 0-1 lines
agentic_search({ query: "auth", context_lines: 0 })

// Standard: 2-3 lines (default)
agentic_search({ query: "auth", context_lines: 2 })

// Detailed: 4-5 lines
agentic_search({ query: "auth", context_lines: 5 })
```

### 5. Use Include Patterns

```typescript
// Search Rust files only
agentic_search({
  query: "database",
  include: "*.rs"
})

// Search config files
agentic_search({
  query: "port",
  include: "*.{toml,yaml,json}"
})
```

## Troubleshooting

### Search is too slow

1. Use `include` pattern to limit file types
2. Reduce `max_results`
3. Use `filename_only` mode for quick discovery

### Expected results not found

1. Try more specific keywords
2. Check if files are excluded by `.gitignore`
3. Verify file size is < 1MB (large files are skipped)
4. Try `grep` for exact matching

### Too many results

1. Make query more specific
2. Use `include` pattern to filter file types
3. Reduce `max_results`
4. Reduce `context_lines`
