# Limits

Limits keep the harness predictable under long-running work, noisy tools, and provider failures.

## Session Options

```ts
const session = agent.session('/repo', {
  maxToolRounds: 24,
  maxParseRetries: 3,
  toolTimeoutMs: 120000,
  circuitBreakerThreshold: 4,
  autoCompact: true,
  autoCompactThreshold: 0.75,
  continuationEnabled: true,
  maxContinuationTurns: 3,
});
```

## What Each Limit Does

- `maxToolRounds` bounds the number of model-tool iterations in a turn.
- `maxParseRetries` bounds malformed tool-call recovery.
- `toolTimeoutMs` stops long-running tools.
- `circuitBreakerThreshold` aborts after repeated provider failures.
- `autoCompact` summarizes history and evidence before context pressure becomes unsafe.
- `continuationEnabled` allows controlled follow-up turns for incomplete work.

## Practical Defaults

Use strict limits for CI, release, and user-facing automation. Use larger budgets for exploratory local coding sessions, but keep verification commands explicit and required when the task has side effects.
