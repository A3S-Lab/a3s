# Skills

Skills are reusable instructions that the harness can discover, rank, and inject when relevant. They are not always dumped into the prompt; v2.1.0 treats skills as budgeted context.

## Enable Built-Ins

```ts
const session = agent.session('/repo', {
  builtinSkills: true,
});
```

Built-in skills include code search, code review, explanation, and bug finding. Larger role-specific behavior should live in project skill files or agent definitions.

## Skill Files

Store Markdown files with frontmatter in a skill directory:

```md
---
name: release-review
description: Review release blockers and verification evidence
allowed-tools: "grep, read, bash(cargo test*)"
---

Check package metadata, changelog, release scripts, and CI status.
Return blockers first.
```

```ts
const session = agent.session('/repo', {
  skillDirs: ['./skills'],
});
```

## Inline Skills

```ts
const session = agent.session('/repo', {
  inlineSkills: [
    {
      name: 'strict-release-review',
      kind: 'instruction',
      content: 'Always separate blockers from nice-to-have improvements.',
    },
  ],
});
```

## Skill Tool

`search_skills` finds relevant skills. `Skill` applies a named skill for a turn. Skill administration is now an SDK or filesystem concern rather than a model-visible management tool.
