# A3S Documentation

A3S is infrastructure for running autonomous AI agents in production. The stack
starts from the agent's real properties: it generates actions, acts on the real
world, cannot be trusted to instrument itself honestly, and needs an external
runtime veto when an action is unsafe.

## Architecture

```
you -> a3s CLI / SDKs
       |
       v
    A3S Code  -- tools, memory, planning, MCP, subagents, verification
       |
       +--> A3S Flow     -- durable workflows, waits, hooks, workers
       +--> A3S Box      -- isolated Linux OCI execution
       +--> A3S Power    -- model serving
       +--> A3S Gateway  -- ingress, proxying, middleware
              +--> Gateway Middleware -- auth, rate-limit, transform, network policy
       +--> Lane/Search/Memory/Event

kernel/user boundary:
    A3S Observer -> A3S Sentry -> Observer guards -> kernel denies

shared contracts:
    ACL config · AHP supervision · Common transport/privacy/tool types

distribution:
    Homebrew tap · release archives · formula checks · cask app bundles
```

The spine is drive -> orchestrate -> isolate -> serve/route -> observe ->
govern. Observer and Sentry wrap the rest: whatever the agent does through Code,
inside Flow workers, inside Box, or out through Gateway should be visible and
answerable to an external control plane.
