# CLI Commands

`a3s` is intentionally small: it routes users to the right A3S surface and keeps
the heavier product behavior in the component crates.

## Public Commands

| Command | Behavior |
| --- | --- |
| `a3s code` | Launch the interactive coding-agent terminal UI. |
| `a3s code research [--local-only\|--web] QUERY` | Run the host-managed DeepResearch pipeline without the full-screen TUI and write local `report.md` and `index.html` artifacts. `deepresearch` and `deep-research` remain aliases; retired runtime-selection flags are rejected. |
| `a3s code resume <id>` | Resume a saved session by id. |
| `a3s code resume` | Resume the newest saved session in the current workspace. |
| `a3s code update` | Alias for the CLI self-update path. |
| `a3s web` | Start the local A3S Code API and A3S Web workspace in the foreground. |
| `a3s web -d` | Start or reuse the healthy workspace service in the background and print its URL, PID, and log path. |
| `a3s web -d --replace` | Gracefully stop and restart only an identity-verified A3S-managed instance; never terminate an unrelated port owner. |
| `a3s login [token]` / `a3s logout` | Sign in to or out from the configured A3S OS account. Browser OAuth is used when no token is supplied. |
| `a3s account list\|status` | Inspect Claude Code, Codex, and A3S OS login availability without exposing product credentials. |
| `a3s account login\|logout <source>` | Delegate Claude Code and Codex authentication to their product CLIs, or manage the shared A3S OS session. |
| `a3s model list\|current` | Read the unified model catalog and active A3S Code route. |
| `a3s model use <route>` | Persist the active model route separately from `config.acl`. Account routes use `claude-code/`, `codex/`, or `a3s-os/` prefixes. |
| `a3s model reset\|config` | Return to `default_model` or print the active `config.acl` path. |
| `a3s box <args...>` | Run `a3s-box <args...>`, bootstrapping Box when missing. |
| `a3s use <args...>` | Resolve `a3s-use` and forward Browser, Office, OCR, or installed external-domain arguments and exit status. |
| `a3s list [--json]` | Report catalog components, delegated Use capabilities, ownership, health, and unregistered local `a3s-*` tools. It does not use the network by default. |
| `a3s install <component>...` | Install or repair registered components such as `use`, `use/browser`, and `use/office`. With no component, list the typed catalog without mutation. |
| `a3s uninstall <component>...` | Remove only files owned by component receipts; child and user-data removal require explicit options. |
| `a3s top` | Open the live monitor for boxes, coding agents, sessions, events, and processes. |
| `a3s update [component]` | Keep the Code self-update alias with no target, or update selected managed components explicitly. |
| `a3s --version` | Print the CLI version. |
| `a3s --help` | Print the built-in usage text. |

Unknown top-level commands exit with an error instead of guessing another
binary name.

## Dispatch Model

`a3s code` stays in-process and calls the TUI module. `a3s web` runs the Web
service in the current process, while `a3s web -d` spawns an independent
process group and redirects output to the local Web log. `a3s box ...` shells
out to the `a3s-box` binary after discovery/bootstrap. `a3s use ...` forwards
to the resolved `a3s-use` binary. Component lifecycle commands use the trusted
catalog and ownership receipts; unregistered PATH tools are reported but never
executed by discovery. Product helpers remain separate binaries rather than
being linked into the umbrella CLI.

Web startup is idempotent. A healthy managed instance for the workspace is
reused; a healthy foreground or legacy A3S Web instance on the requested
address can be discovered without granting stop authority. Port ownership is
checked before assets, configuration, and saved sessions are restored, so a
repeat invocation does not emit unrelated session warnings first. Only explicit
`--replace` uses the private control endpoint to gracefully replace a managed
instance.

## Runtime Boundaries

- The CLI owns command selection, the trusted component catalog, ownership-safe
  lifecycle operations, first-use glue, and self-update UX.
- A3S Use owns typed Browser, native Office, and OCR domains plus explicitly installed ACL
  packages with native CLI, standard MCP, and/or `SKILL.md` surfaces.
- A3S Code owns the agent session, model/tool execution, workspace services, and
  persistence.
- A3S TUI owns terminal rendering and input handling.
- A3S Box owns sandbox/container runtime behavior.
