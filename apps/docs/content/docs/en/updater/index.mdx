# A3S Updater

`a3s-updater` is a small library used by A3S command-line tools to check GitHub
Releases, download the matching platform archive, extract the target binary,
and replace the currently running binary.

The current crate version is `a3s-updater` `0.3.0`.

## Capability Map

| Area | Current capability |
| --- | --- |
| Release lookup | Fetches the latest GitHub release for a configured owner/repository through the GitHub Releases API. |
| Versioning | Parses semver tags with an optional `v` prefix and compares against the current binary version. |
| Platform match | Normalizes supported targets to `darwin` or `linux` and `arm64` or `x86_64`. |
| Asset naming | Looks for archives named `<binary>-<version>-<os>-<arch>.tar.gz`. |
| Archive handling | Downloads `.tar.gz`, extracts only regular files into a temporary directory, and selects the requested binary by file name. |
| Installation | Renames the current executable to a backup, copies the new binary into place, sets Unix executable permissions, and restores on copy failure. |
| Fleet metadata | Verifies Ed25519-signed component identity, version, SHA-256 digest, protocol range, release channel, and rollback constraints. |
| Fleet activation | Stages node-agent or Gateway binaries and systemd units, persists every state transition, and gates success on health and exact applied version. |
| Recovery | Replays interruption-safe receipts, automatically restores the prior binary after activation or health failure, and keeps downloads outbound from the node. |

## Boundary

The original GitHub Release path targets CLI-style replacement. The fleet path
targets Linux services but deliberately does not schedule rollouts or provide
remote shell access. Cloud wiring remains an explicit host responsibility.
