# Skills System

Skills are Markdown files that extend agent behavior. They can inject system prompts, define custom tools, or create specialized agent personas. Compatible with Claude Code Skills format.

## Skill Format

```markdown
# ~/.a3s/skills/deploy.md
---
name: deploy
description: Deploy to production
allowed_tools: Bash(kubectl:*)
---
You are a deployment specialist. Run kubectl apply to deploy the application.
Always verify the deployment status after applying changes.
```

### Frontmatter Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | yes | Unique skill identifier |
| `description` | yes | What the skill does |
| `allowed_tools` | no | Restrict which tools this skill can use (glob patterns) |

### Skill Kinds

Skills are automatically classified into three kinds:

| Kind | Description |
|------|-------------|
| `instruction` | Injects system prompt context (no tools) |
| `tool` | Defines custom tool backends (binary, HTTP, script) |
| `agent` | Creates a specialized agent persona |

## Loading Skills

```typescript
// Load a single skill by name
await session.loadSkill('deploy');

// Load all skills from a directory
await session.loadSkills('~/.a3s/skills');

const skills = await session.listSkills();
```

## Skill Discovery & Installation

The agent has three native tools for discovering and managing skills at runtime:

| Tool | Description |
|------|-------------|
| `search_skills` | Search for skills on GitHub via the GitHub Search API |
| `install_skill` | Download and install a skill from a GitHub repository |
| `load_skill` | Load an installed skill into the current session |

### Searching Skills

The `search_skills` tool queries GitHub for repositories containing A3S-compatible skill files. No authentication required.

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
const { skills } = await client.searchSkills({
  sessionId,
  query: 'kubernetes deployment',
});
// Returns matching skills with name, description, repo URL
```
</Tab>
<Tab value="Python">
```python
result = await client.search_skills(
    session_id=session_id,
    query="kubernetes deployment",
)
```
</Tab>
</Tabs>

### Installing Skills

Install a skill from GitHub by specifying `owner/repo` or `owner/repo@skill-name`:

<Tabs groupId="lang" items={['TypeScript', 'Python']}>
<Tab value="TypeScript">
```typescript
// Install all skills from a repo
await client.installSkill({
  source: 'a3s-lab/skills',
  global: true,  // Install to ~/.a3s/skills/ (false = project-local)
});

// Install a specific skill
await client.installSkill({
  source: 'a3s-lab/skills@deploy',
  global: false,
});
```
</Tab>
<Tab value="Python">
```python
await client.install_skill(source="a3s-lab/skills", global_install=True)
await client.install_skill(source="a3s-lab/skills@deploy", global_install=False)
```
</Tab>
</Tabs>

### Autonomous Discovery

The agent can autonomously discover and load skills when it encounters tasks beyond its built-in capabilities. For example, if asked to deploy to Kubernetes and no deployment skill is loaded, the agent may:

1. Call `search_skills("kubernetes deploy")`
2. Find a matching skill
3. Call `install_skill("owner/repo@k8s-deploy")`
4. Call `load_skill("k8s-deploy")`
5. Proceed with the deployment task

## Skill Tool Filters

When a skill specifies `allowed_tools`, the agent loop enforces these restrictions. A skill with `allowed_tools: Bash(kubectl:*)` can only use bash commands starting with `kubectl`.

This provides per-skill sandboxing — a deployment skill can't accidentally run `rm -rf /`, even if the session's permission policy would allow it.

## Skill Directories

Configure skill directories in `config.json`:

```json
{
  "skillDirs": ["~/.a3s/skills", "/shared/team-skills"],
  "agentDirs": ["~/.a3s/agents"]
}
```

Skills in these directories are available for loading but not automatically active — they must be explicitly loaded into a session.
