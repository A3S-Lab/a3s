# Todo Tracking

Each session maintains a todo list for tracking multi-step tasks. The agent creates, updates, and completes todos as it works through complex requests.

## Todo Structure

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier |
| `content` | string | Task description |
| `status` | enum | `pending`, `in_progress`, `completed`, `blocked` |
| `priority` | enum | `high`, `medium`, `low` |
| `created_at` | timestamp | When the todo was created |
| `completed_at` | timestamp | When the todo was completed |

## Context Integration

The todo list is included in the agent's context, so it always knows what work remains. This prevents the agent from losing track of subtasks in long-running sessions.
